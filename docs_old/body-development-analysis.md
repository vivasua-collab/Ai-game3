## 2️⃣ Физические характеристики: система развития

### 2.1 Принцип линейного развития

**Из лора:** "нет стадии куколки, чтобы резко делать скачки"

**Решение:** Развитие физических характеристик (сила, ловкость):
1. **Постепенное** — прирост 0.001 за действие (10 действий = +0.01)
2. **Накопительное** — виртуальная дельта накапливается
3. **Закрепляемое** — сон конвертирует виртуальную дельту в реальную
4. **С порогами** — чем выше стат, тем больше дельты нужно (аналогия с ядром)

### 2.5 Максимальное закрепление за сон

**Финальная формула:**
```typescript
// Базовый кап за сон
const MAX_CONSOLIDATION_PER_SLEEP = 0.20;

// 4 часа = +0.067
// 6 часов = +0.133
// 8 часов = +0.20 (кап)

function calculateMaxConsolidation(sleepHours: number): number {
  if (sleepHours < 4) return 0;
  const base = 0.067;
  const perHour = 0.033;
  return Math.min(MAX_CONSOLIDATION_PER_SLEEP, base + (sleepHours - 4) * perHour);
}
```

### 2.6 Система Порогов Развития

**Идея:** Аналогия с ядром культивации — чем больше характеристика, тем больше "виртуальной дельты" нужно для повышения.

```typescript
/**
 * Расчёт порога развития для следующей единицы характеристики
 * Чем выше стат, тем больше нужно накопить для повышения на 1
 */
function calculateStatThreshold(currentStat: number): number {
  // Базовый порог: 1.0 для стата 10-19
  // Затем: floor(stat / 10)
  // Аналогия с ядром: чем больше сосуд, тем больше нужно для наполнения
  return Math.max(1.0, Math.floor(currentStat / 10));
}

// Примеры:
// Стат 10: threshold = 1.0  → нужно 1.0 виртуальной дельты для 11
// Стат 20: threshold = 2.0  → нужно 2.0 виртуальной дельты для 21
// Стат 30: threshold = 3.0  → нужно 3.0 виртуальной дельты для 31
// Стат 50: threshold = 5.0  → нужно 5.0 виртуальной дельты для 51
// Стат 100: threshold = 10.0 → нужно 10.0 виртуальной дельты для 101

/**
 * Процесс повышения при сне
 */
function processStatAdvancement(stat: StatDevelopment): void {
  const threshold = calculateStatThreshold(stat.current);
  
  // Сколько раз можно повысить (если накоплено много)
  while (stat.virtualDelta >= threshold) {
    stat.current += 1;
    stat.virtualDelta -= threshold;
    // Новый порог будет выше!
  }
}
```

**Преимущества:**
- Нет жёсткого капа
- Естественное замедление развития
- Аналогия с ядром культивации
- Каждое повышение значимо

**Расчёт достижимости за периоды:**

| Период | Накоплено дельты | Достижимый стат | Время на +1
|--------|------------------|-----------------|------------|
| 1000 дней | 150.0 | ~55 | 21-28 дней |
| 3000 дней | 450.0 | ~80 | 42-56 дней |
| 10000 дней | 1500.0 | ~125 | 70+ дней |

### 4.2 Бонус за изучение новой техники

**Решение:** Одноразовый бонус за изучение, зависит от уровня техники.

```typescript
// Бонус за изучение техники
const TECHNIQUE_LEARNING_BONUS = {
  BASE_BONUS: 0.02,           // Базовый бонус
  LEVEL_MULTIPLIER: 0.01,     // За каждый уровень техники
  DIMINISHING_FACTOR: 0.5,    // Снижение для высоких характеристик
};

// Пример: Изучение техники L3
// bonus = 0.02 + 3 * 0.01 = 0.05 интеллекта

// При высоком интеллекте (>50) бонус снижается на 50%
```

**Примечание:** Коэффициенты будут выявляться после реализации всех механик и альфа-тестирования. Должны быть легко корректируемые в коде.

### 4.3 Буфер или отбрасывание для техник?

**Расшифровка вопроса:**
При использовании техник прирост +0.001. Как обрабатывать нецелые значения?

**Решение:** Буфер + сон
```
┌─────────────────────────────────────────────────────────────────┐
│                    БУФЕР ИНТЕЛЛЕКТА                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   Накопление:                                                    │
│   ────────────                                                   │
│   Техника 1:  buffer += 0.001 (итого: 0.001)                    │
│   Техника 2:  buffer += 0.001 (итого: 0.002)                    │
```

### 5.2 Влияние интеллекта на ВСЕ техники Ци

**Решение:** Интеллект даёт базовый бонус ко всем техникам Ци. НЕ влияет на проводимость.

```typescript
// Базовый бонус интеллекта к Ци-техникам
const INTELLECT_QI_BONUS = {
  BASE_FACTOR: 0.005,  // +0.5% за каждую единицу интеллекта выше 10
  CAP: 0.30,           // Максимум +30%
};

function calculateIntellectQiBonus(intelligence: number): number {
  if (intelligence <= 10) return 0;
  return Math.min(
    INTELLECT_QI_BONUS.CAP,
    (intelligence - 10) * INTELLECT_QI_BONUS.BASE_FACTOR
  );
}

// Примеры:
// Intelligence 20: +5% к эффективности Ци-техник
// Intelligence 30: +10%
// Intelligence 50: +20%
// Intelligence 70: +30% (кап)
```

**Примечание:** Коэффициенты определяются в процессе тестов и симуляций.

## 6️⃣ Система усталости

### 6.1 Проблема

Текущая система: 10 ударов = смертельно устал.

### 6.2 Решение (ПЕРЕСМОТР)

```typescript
// Параметры усталости
const FATIGUE_SYSTEM = {
  // Физическая усталость
  PHYSICAL: {
    MAX: 100,
    RECOVERY_RATE: 0.5,     // В секунду при отдыхе
    ATTACK_COST: 0.5,       // За удар (было больше)
    DODGE_COST: 0.3,        // За уклонение

  // Ментальная усталость
  MENTAL: {
    MAX: 100,
    RECOVERY_RATE: 0.3,     // В секунду при отдыхе
    TECHNIQUE_COST: 1.0,    // За использование техники
    MEDITATION_COST: 0.1,   // За секунду медитации
    
    // Влияние интеллекта
    INTELLIGENCE_REDUCTION: 0.005, // -0.5% за каждую int выше 10
  },

// Пример: 10 ударов
// Физическая усталость: 10 * 0.5 = 5 из 100 (5%)
// При vitality 30: 5 * 0.8 = 4 (20% снижение)
```

## 7️⃣ Нагрузка на CPU для NPC

### 7.1 Проблема

NPC много (десятки/сотни в зоне). Просчёт развития каждого — накладно.

### 7.2 Решение: Множители для Temp NPC

| Тип NPC | Обновление | Хранение |
|---------|------------|----------|
| Temp (временные) | Множители при генерации | Только в памяти |
| Plot (сюжетные) | Полный расчёт | БД |
| Главы сект | Полный расчёт + периодический | БД |

### 7.3 Оценка нагрузки

| Операция | CPU время | Для 100 NPC |
|----------|-----------|-------------|
| Генерация Temp NPC | ~1-5 мс | ~100-500 мс (одноразово) |
| Расчёт одного действия | ~0.01 мс | ~1 мс |
| Закрепление при сне | ~0.1 мс | ~10 мс |

**Вывод:** Приемлемо для игрового процесса.

## 8️⃣ Применение vitalityMultiplier ко всем персонажам

### 8.1 Формула

```typescript
// Глобальная функция для всех персонажей
function calculateMaxHP(
  baseHP: number,
  sizeMultiplier: number,
  vitality: number,
  cultivationLevel: number
): number {
  const vitalityMultiplier = 1 + (vitality - 10) * 0.05;
  const cultivationBonus = STAT_MULTIPLIERS_BY_LEVEL[cultivationLevel] || 1.0;
  
  return Math.floor(baseHP * sizeMultiplier * vitalityMultiplier * cultivationBonus);
}
```

### 8.2 Нагрузка при применении

| Операция | CPU время | Примечание |
|----------|-----------|------------|
| Расчёт HP одной части | ~0.001 мс | Простая арифметика |
| Расчёт HP 10 частей | ~0.01 мс | Для одного NPC |
| Пересчёт 100 NPC | ~1 мс | При изменении vitality |

**Вывод:** Нагрузка минимальна, можно применять глобально.

## 9️⃣ Принятые решения

### 9.1 Утверждено

| Решение | Значение |
|---------|----------|
| Прирост за действие | **0.001** |
| Кап за сон | **+0.20** (8 часов) |
| Множитель культивации | **УБРАТЬ** |
| Система развития | **Пороги развития** (аналогия с ядром) |
| Бонус за изучение техник | Да, зависит от уровня |
| Буфер интеллекта | Да + сон |
| Интеллект → Ци-техники | Базовый бонус |
| Потеря виртуальной дельты | **НЕТ** (кап на дельту) |
| Принудительный сон | RestDialog или кнопка |

### 9.2 Система порогов развития

```typescript
// Формула порога для повышения характеристики
threshold = Math.max(1.0, Math.floor(currentStat / 10));

// Примеры:
// Стат 10 → 11: нужно 1.0 виртуальной дельты
// Стат 20 → 21: нужно 2.0 виртуальной дельты
// Стат 50 → 51: нужно 5.0 виртуальной дельты
// Стат 100 → 101: нужно 10.0 виртуальной дельты
```

**Аналогия с ядром культивации:**
- Ядро: чем выше уровень → больше Ци для прорыва
- Стат: чем выше значение → больше дельты для повышения
- Принцип: "чем больше сосуд, тем больше наполнять"

### 9.3 Достижимость за периоды

| Период | Достижимый стат | Время на +1 |
|--------|-----------------|-------------|
| 1000 дней | ~55 | 21-28 дней |
| 3000 дней | ~80 | 42-56 дней |
| 10000 дней | ~125 | 70+ дней |

### 9.4 Для тестирования
