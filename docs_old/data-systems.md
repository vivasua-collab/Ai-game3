## 📋 Обзор

Документ объединяет три ключевые системы:
1. **Архитектура хранения** — гибридный подход SQLite + JSON

## 1️⃣ Архитектура хранения

### Гибридный подход (рекомендуется)

```
┌─────────────────────────────────────────────────────────────┐
│  ДИНАМИЧЕСКИЕ → SQLite (ACID, транзакции)                   │
│  ├── Character (Ци, HP, усталость — часто)                  │
│  ├── Inventory (количество — часто)                         │
│  └── TechniquePool (временные)                              │
├─────────────────────────────────────────────────────────────┤
│  СТАТИЧЕСКИЕ → JSON файлы (кэш в памяти)                    │
│  ├── Пресеты техник (~2046 шт)                              │
│  ├── Пресеты предметов (~2046 шт)                           │
│  └── Шаблоны NPC (~50 шт)                                   │
├─────────────────────────────────────────────────────────────┤
│  ИСТОРИЯ → JSON файлы (ротация)                             │
│  ├── Messages                                               │
│  └── SystemLogs                                             │
└─────────────────────────────────────────────────────────────┘
```

### Лимиты JSON файлов

| Параметр | Рекомендуемый | Жёсткий |
|----------|---------------|---------|
| Макс. объектов в файле | 10,000 | 20,000 |
| Макс. размер файла | 5 MB | 10 MB |

### Итоговый размер базы данных

| Компонент | Количество | Размер |
|-----------|------------|--------|
| Пресеты (техники + предметы + NPC) | ~4,100 | ~650 KB |
| Игроки (макс. 100) | 100 | ~4.1 MB |
| NPC (minimal) | 100,000 | 15-70 MB |
| Мир (локации, события) | ~1,000 | ~900 KB |
| **ИТОГО** | | **20-75 MB** |

## 2️⃣ Система идентификаторов

### Форматы ID

**1. cUID2 (Prisma auto-generated)**
- Используется для всех записей в БД
- Формат: `cmmaewq7h000knxux5ckll002`
- Преимущества: уникальность, безопасность, сортировка по времени

**2. Префикс + счётчик (генерируемые объекты)**
- Формат: `{PREFIX}_{NUMBER:06d}`
- Пример: `MS_0512`, `WP_0042`, `NPC_000042`

### Префиксы по типам

#### Техники

| Тип | Префикс | Пример |
|-----|---------|--------|
| Удар телом | MS | MS_0512 |
| Оружейная | MW | MW_0042 |
| Дальняя | RG | RG_0123 |
| Защитная | DF | DF_0042 |
| Культивация | CU | CU_0010 |
| Поддержка | SP | SP_0042 |

#### Предметы

| Тип | Префикс | Пример |
|-----|---------|--------|
| Оружие | WP | WP_0042 |
| Броня | AR | AR_0123 |
| Аксессуар | AC | AC_0007 |
| Расходник | CS | CS_0512 |
| Материал | MT | MT_0089 |
| Камень Ци | QS | QS_0033 |
| Зарядник | CH | CH_0015 |

#### NPC

| Тип | Префикс | Пример |
|-----|---------|--------|
| Сгенерированный | NP | NP_000042 |
| Preset | NPC_PRESET | NPC_PRESET_00001 |
| Временный | TEMP | TEMP_083452 |

### Система счётчиков

```json
// presets/counters.json
{
  "counters": {
    "MS": 1024,
    "MW": 512,
    "WP": 100,
    "NPC": 100
  }
}
```

**Реализация:** `src/lib/generator/id-service.ts`

## 3️⃣ Система пресетов

### Структура файлов

```
/src/data/presets/
├── techniques/
│   ├── index.json              # Индекс (id → file)
│   ├── level-1.json            # 1024 техники (~150 KB)
│   ├── level-2.json            # 512 техник (~75 KB)
│   └── ...                     # Убывает по уровням
├── items/
│   ├── index.json
│   └── level-{1-9}.json
├── npc/
│   └── templates.json          # ~50 шаблонов
└── counters.json               # Счётчики ID
```

### Концепция Base + Modifiers

Вместо хранения полных объектов:

```typescript
// Базовый объект (фиксированные характеристики)
interface BaseTechnique {
  id: string;
  name: string;
  type: TechniqueType;
  element: Element;
  level: number;
  baseDamage: number;
  baseQiCost: number;
}

// Модификаторы (флаги + значения)
interface TechniqueModifiers {
  effects: {
    burning?: boolean;
    freezing?: boolean;
    stun?: boolean;
    // ...
  };
  effectValues: {
    burningDamage?: number;
    stunDuration?: number;
    // ...
  };
  penalties: {
    qiCostMultiplier?: number;
  };
  bonuses: {
    damageMultiplier?: number;
  };
}
```

### Распределение по уровням

```
Уровень    Количество    Формула
────────────────────────────────────
   1         1024        base
   2          512        base / 2
   3          256        base / 4
   ...
   9            4        base / 256
────────────────────────────────────
ИТОГО:      2046 техник
```

### Размер хранения

| Подход | 1 техника | 2046 техник |
|--------|-----------|-------------|
| Полный JSON | ~800 байт | ~1.6 MB |
| Base + Modifiers | ~150 байт | ~300 KB |
| **Экономия** | | **81%** |

## 5️⃣ Стратегии хранения

### Хранить полностью (БД)

- Игровые персонажи (макс. 100)
- Каталог техник игрока
- Инвентарь игроков
- Состояния сессий

### Хранить по ID

- NPC и монстры (до 100,000)
- Техники NPC (массив ID)
- Экипировка NPC (массив ID)

### Генерировать заранее (JSON)

- Пресеты техник (~2046 шт)
- Пресеты предметов (~2046 шт)
- Шаблоны NPC (~50 шт)
