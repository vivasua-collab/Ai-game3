## 🎯 Фундаментальное правило

```
┌─────────────────────────────────────────────────────────────────┐
│   1 TICK = 1 СЕКУНДА РЕАЛЬНОГО ВРЕМЕНИ (FIXED, NEVER CHANGES)  │
│                                                                 │
│   Переменная: СКОЛЬКО ИГРОВОГО ВРЕМЕНИ проходит за тик          │
└─────────────────────────────────────────────────────────────────┘
```

Это правило **НЕИЗМЕННО**. Скорость игры регулируется только через `minutesPerTick`.

**Важно для мультиплеера:** При появлении многопользовательского режима все игроки будут жить в одном времени со статичной скоростью тиков. Сейчас игра однопользовательская.

## 🚀 СИСТЕМА МАСШТАБИРОВАНИЯ ВРЕМЕНИ (Time Scaling)

### ✅ Решения пользователя (2026-03-24)

#### 1. Разная скорость для разных действий?
**Ответ: ДА**

| Действие | Режим времени | Обоснование |
|----------|--------------|-------------|
| Путешествие (выход с локации) | `fast` / `ultra` | Ускорение перемещения |
| Исследование локации | `normal` | Стандартная скорость |
| Бой | `superSuperSlow` | Тактический контроль |
| Медитация | `ultra` | Максимальное ускорение |

#### 2. Должна ли медитация (ultra) делать всё мгновенным?
**Ответ: ДА**

- В режиме медитации время ускоряется до `ultra` (60 мин/тик)
- Все действия масштабируются соответственно
- Игрок "проживает" часы за секунды

#### 3. Нужен ли "боевой режим" времени?
**Ответ: ДА**

- Автоматическое переключение на `superSuperSlow` при начале боя
- Возврат к предыдущей скорости после боя
- Возможность ручного переключения во время боя

#### 4. Многопользовательский режим?
**Ответ: НЕТ (пока)**

- Игра **однопользовательская**
- При появлении мультиплеера: все живут в одном времени со статичной скоростью тиков
- Клиент-серверная архитектура сейчас — для будущего перехода на другой графический движок

### Принцип работы Time Scaling

**Концепция: "Время как ресурс"**

**Принцип**: Все действия измеряются в **игровых минутах**, а не в реальных секундах.

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    ПОДХОД: ВРЕМЯ КАК РЕСУРС                                │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  Действие = "занимает X игровых минут"                                      │
│  Реальная длительность = X / minutesPerTick секунд                          │
│                                                                             │
│  Примеры:                                                                   │
│  ─────────────────────────────────────────────────────────────────────────  │
│  Атака: занимает 1 игровую минуту                                           │
│  ├─ При normal (5 мин/тик):  1/5 = 0.2 секунды (быстро!)                   │
│  ├─ При slow (1 мин/тик):    1/1 = 1 секунда                               │
│  └─ При superSlow (0.5):     1/0.5 = 2 секунды                             │
│                                                                             │
│  Путешествие 100px: занимает 0.5 игровой минуты                             │
│  ├─ При normal (5 мин/тик):  0.5/5 = 0.1 секунды                           │
│  ├─ При slow (1 мин/тик):    0.5/1 = 0.5 секунды                           │
│  └─ При superSlow (0.5):     0.5/0.5 = 1 секунда                           │
│                                                                             │
│  Плюсы:                                                                     │
│  ✓ Время = ресурс (культивация тратит время)                               │
│  ✓ Баланс: быстрое время = быстро "тратится"                               │
│  ✓ Подходит для RPG культивации                                            │
│  ✓ "Тайм-менеджмент" как игровая механика                                  │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### Базовые длительности действий

```typescript
ACTION_DURATIONS = {
  // === Combat ===
  BASIC_ATTACK: 1,        // 1 игровая минута
  TECHNIQUE_FAST: 0.5,    // 30 игровых секунд
  TECHNIQUE_NORMAL: 2,    // 2 игровые минуты
  TECHNIQUE_ULTIMATE: 5,  // 5 игровых минут
  
  // === Movement ===
  STEP_BASE: 0.1,         // 6 игровых секунд (базовый шаг)
  RUN_SPEED: 1,           // 1 минута на "клетку"
  DASH: 0.05,             // 3 игровых секунды
  
  // === Cultivation ===
  MEDITATION_TICK: 60,    // 1 час (медитация за тик)
  QI_ABSORPTION: 5,       // 5 минут
  BREAKTHROUGH: 480,      // 8 часов
  
  // === NPC ===
  NPC_IDLE: 0.25,         // 15 секунд
  NPC_PATROL_STEP: 0.5,   // 30 секунд
  NPC_ATTACK: 1,          // 1 минута
}
```

### Формулы конвертации

```typescript
// Игровые минуты → реальные миллисекунды
realMs = (gameMinutes / minutesPerTick) * 1000

// Скорость движения
actualSpeed = baseSpeed * minutesPerTick

// Кулдаун
cooldown = (baseGameMinutes / minutesPerTick) * 1000
```

## ⚠️ ИСПРАВЛЕНИЕ: Инверсия скорости (2026-03-24)

### Проблема

При использовании функции `scaleMovementSpeed()` наблюдалась **инверсия скорости**:
- При `slow` времени игрок двигался слишком быстро
- При `fast` времени игрок двигался слишком медленно

### Причина

Неправильная формула масштабирования:
```typescript
// НЕПРАВИЛЬНО (инверсия):
scaledSpeed = baseSpeed * TIME_SCALING_FACTORS[speed];
// slow: 200 * 5.0 = 1000 px/sec (очень быстро!)
// fast: 200 * 0.333 = 66 px/sec (очень медленно!)
```

### Решение

Использовать функцию `scaleMovementSpeedInverse()`:
```typescript
// ПРАВИЛЬНО:
scaledSpeed = baseSpeed / TIME_SCALING_FACTORS[speed];
// slow: 200 / 5.0 = 40 px/sec (медленно)
// fast: 200 / 0.333 = 600 px/sec (быстро)
```

### Результат

| Speed | Factor | Base 200 | Scaled | Восприятие |
|-------|--------|----------|--------|------------|
| superSuperSlow | 20.0 | 200 | 10 | Очень медленное (бой) |
| slow | 5.0 | 200 | 40 | Медленное |
| normal | 1.0 | 200 | 200 | Обычное |
| fast | 0.333 | 200 | 600 | Быстрое (путешествие) |
| ultra | 0.083 | 200 | 2400 | Очень быстрое (медитация) |

## ⚡ Batch обработка

### QiTickProcessor

- Накапливает тики в памяти
- Отправляет на сервер каждые 10 тиков
- Обрабатывает: пассивное накопление Ци, рассеивание

### TruthSystem batch save

- Сохраняет каждые 60 тиков (1 минута реального времени)
- Принудительное сохранение при паузе
- Критические события сохраняются немедленно

## 🔧 Известные проблемы и решения

### Проблема 1: timer:start НЕ обрабатывается

**Симптом**: При движении игрока время не начинается

**Причина**: 
- `tickTimer.start()` эмитит `timer:start`
- LocationScene слушает только `timer:resume` и `timer:pause`

**Решение**: Добавить слушатель `timer:start` или эмитить оба события

### Проблема 2: isGamePaused не синхронизирован

**Симптом**: Движение заблокировано после старта

**Причина**: Локальный флаг `isGamePaused` не обновляется при `timer:start`

**Решение**: Проверять store напрямую: `useTimeStore.getState().isPaused`

### Проблема 3: Разные источники времени

**Симптом**: React показывает 06:00, Phaser показывает 07:00

**Причина**:
- React: `time.store.gameTime` (инициализируется хардкодом)
- Phaser: `game.store.worldTime` (инициализируется с сервера)

**Решение**: Синхронизация через `_initFromServer()` при старте
