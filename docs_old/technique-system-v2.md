## 1️⃣ GRADE SYSTEM (Система Качества)

### 1.1 Уровни Grade

> **Важно:** После внедрения "Матрёшка" (2026-03-15) техники используют единую систему Grade.

```typescript
// src/types/grade.ts
type TechniqueGrade = 'common' | 'refined' | 'perfect' | 'transcendent';
```

### 1.2 Множители Grade

| Grade | Урон | Бонусов | Шанс эффекта |
|-------|------|---------|--------------|
| **Common** | ×1.0 | 0 | 0% |
| **Refined** | ×1.2 | 1 | 20% |
| **Perfect** | ×1.4 | 2 | 50% |
| **Transcendent** | ×1.6 | 3 | 80% |

> **Стоимость Ци всегда ×1.0** — не зависит от Grade.

### 1.3 Распределение Grade

```typescript
// НЕ зависит от уровня!
const UNIVERSAL_TECHNIQUE_DISTRIBUTION = {
  common: 60,       // 60%
  refined: 28,      // 28%
  perfect: 10,      // 10%
  transcendent: 2,  // 2% — даже на L1!
};
```

## 2️⃣ СТРУКТУРНАЯ ЁМКОСТЬ (Capacity)

### 2.1 Принцип

**Ёмкость техники — максимальное базовое Ци, которое техника может обработать.**

```
capacity = baseCapacity(type) × 2^(level-1) × masteryBonus
```

Где:
- `baseCapacity(type)` — базовая ёмкость по типу техники
- `2^(level-1)` — множитель уровня
- `masteryBonus` — бонус мастерства (1.0 ~ 1.5)

### 2.2 Базовая ёмкость по типам техники

```typescript
// Файл: src/lib/constants/technique-capacity.ts
export const BASE_CAPACITY_BY_TYPE: Record<TechniqueType, number | null> = {
  formation: 80,     // Максимальная — долгая установка
  defense: 72,       // Очень высокая — резервуар для блокирования
  support: 56,       // Повышенная — мягкое воздействие
  healing: 56,       // Повышенная — мягкое воздействие
  combat: 48,        // Базовое, переопределяется по подтипу
  movement: 40,      // Ниже средней — выталкивание тела
  curse: 40,         // Ниже средней — пробитие защиты
  poison: 40,        // Ниже средней — скрытое внедрение
  sensory: 32,       // Низкая — расхождение веером
  cultivation: null, // Пассивная — не использует capacity
};

export const BASE_CAPACITY_BY_COMBAT_SUBTYPE: Record<CombatSubtype, number> = {
  melee_strike: 64,      // Высокая — Ци в теле
  melee_weapon: 48,      // Средняя — через оружие
  ranged_projectile: 32, // Низкая — снаряд летит
  ranged_beam: 32,       // Низкая — луч рассеивается
  ranged_aoe: 32,        // Низкая — область
};
```

### 2.3 Примеры ёмкости

**melee_strike L5, mastery 0%:**
```
capacity = 64 × 2^4 × 1.0 = 1024 базового Ци
```

**melee_strike L3, mastery 0%:**
```
capacity = 64 × 2^2 × 1.0 = 256 базового Ци
```

**ranged_projectile L9, mastery 100%:**
```
capacity = 32 × 256 × 1.5 = 12288 базового Ци
```

## 3️⃣ ДЕСТАБИЛИЗАЦИЯ

### 3.1 Принцип неделимости Ци

**Кратность выпуска Ци практиком = 1**

Практик выпускает Ци минимальными единицами своего уровня культивации.

```
1 единица Ци практика L9 = 256 единиц базового Ци
1 единица Ци практика L5 = 16 единиц базового Ци
1 единица Ци практики L1 = 1 единица базового Ци
```

### 3.2 Механика переполнения

При использовании техники практик вливает в неё своё Ци.

**Если вливаемое Ци > ёмкости техники → ПЕРЕПОЛНЕНИЕ**

```
Пример: Практик L9 использует технику L2 (melee_strike)

  Ёмкость техники: capacity = 64 × 2 = 128 базового Ци
  1 единица Ци практика L9 = 256 базового Ци
  
  Переполнение: 256 - 128 = 128 единиц
  
  Результат:
  1. Излишки Ци (128) рассеиваются
  2. Практик получает урон = 50% от излишков = 64 урона
  3. Урон по цели (только melee_strike/melee_weapon!) = 50% от влитого = 128 урона
```

### 3.3 Правило переполнения

> **Переполнение ВОЗМОЖНО, но с последствиями.**

```
┌─────────────────────────────────────────────────────────────────────┐
│                    ПРАВИЛО ПЕРЕПОЛНЕНИЯ                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  При переполнении техники:                                          │
│                                                                      │
│  1. ИЗЛИШКИ ЦИ РАССЕИВАЮТСЯ                                         │
│     - excessQi = inputQi - capacity                                 │
│     - Рассеивается в окружающее пространство                        │
│                                                                      │
│  2. УРОН ПРАКТИКУ                                                    │
│     - backlashDamage = excessQi × 0.5 (50% от излишков)            │
│                                                                      │
│  3. УРОН ПО ЦЕЛИ — ТОЛЬКО ДЛЯ БЛИЖНЕГО БОЯ!                         │
│     - melee_strike / melee_weapon: effectiveDamage = inputQi × 0.5 │
│     - ranged_*: Ци разлетается во все стороны, урона по цели НЕТ   │
│                                                                      │
│  Примечание:                                                         │
│  Практик не дурак — он знает, какие техники ему подходят.          │
│  Дестабилизация происходит только при ошибках или вынужденных      │
│  ситуациях (например, нет других техник под рукой).                │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### 3.4 Примеры дестабилизации

**Сценарий 1: Ближний бой с переполнением**
```
Практик L9 (qiDensity = 256) использует L2 melee_strike (capacity = 128)

  Вливаемое Ци: 256 базового
  Ёмкость: 128
  Переполнение: 128
  
  Результат:
  - Практик получает: 128 × 0.5 = 64 урона
  - Урон по цели: 256 × 0.5 = 128 урона (ближний бой!)
```

**Сценарий 2: Дальняя атака с переполнением**
```
Практик L9 (qiDensity = 256) использует L1 ranged_projectile (capacity = 32)

  Вливаемое Ци: 256 базового
  Ёмкость: 32
  Переполнение: 224
  
  Результат:
  - Практик получает: 224 × 0.5 = 112 урона
  - Урон по цели: НЕТ! Ци разлетается во все стороны
```

**Сценарий 3: Нормальное использование**
```
Практик L9 (qiDensity = 256) использует L3 melee_strike (capacity = 256)

  Вливаемое Ци: 256 базового
  Ёмкость: 256
  
  Результат: Ёмкость = вливаемому Ци
  - Техника работает корректно
  - Урон = 256 × gradeMult
```

## 4️⃣ ОГРАНИЧЕНИЯ ПО УРОВНЮ

### 4.1 Почему нельзя использовать техники ВЫШЕ своим уровнем?

> **ЛОР: Резонанс Ци**

```
┌─────────────────────────────────────────────────────────────────────┐
│                    ЛОР: РЕЗОНАНС ЦИ                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  Техника — это структурированный канал для Ци.                      │
│  Каждый уровень техники требует определённой плотности Ци           │
│  для поддержания структуры канала.                                   │
│                                                                      │
│  Практик L5 (qiDensity = 16):                                       │
│  ─────────────────────────────────────                              │
│  - L5 техника: Требует плотность 16 → РЕЗОНАНС ✓                    │
│  - L6 техника: Требует плотность 32 → НЕТ резонанса ✗              │
│  - L7 техника: Требует плотность 64 → НЕТ резонанса ✗              │
│                                                                      │
│  При попытке использовать технику выше уровнем:                     │
│  - Ци практика "не вытягивает" структуру техники                   │
│  - Канал коллапсирует при попытке активации                        │
│  - Техника просто НЕ активируется                                   │
│                                                                      │
│  Аналогия:                                                           │
│  "Нельзя заставить слабый источник света                            │
│   пройти через линзу, рассчитанную на мощный лазер."                │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### 4.2 Таблица совместимости

| Уровень практика | qiDensity | Мин. L техники | Макс. L техники |
|-----------------|-----------|----------------|-----------------|
| L1 | 1 | L1 | L1 |
| L2 | 2 | L1 | L2 |
| L3 | 4 | L1 | L3 |
| L4 | 8 | L2 | L4 |
| L5 | 16 | L2 | L5 |
| L6 | 32 | L3 | L6 |
| L7 | 64 | L3 | L7 |
| L8 | 128 | L4 | L8 |
| L9 | 256 | L4 | L9 |

**Правило:**
- **Минимальный уровень техники** = достаточно ёмкости для qiDensity практика
- **Максимальный уровень техники** = уровень практика (резонанс)

|--------------------|--------------|------------|
| L1 | 1 | 1 |
| L2 | 2 | 2 |

| L6 | 32 | 32 |
| L7 | 64 | 64 |

## 6️⃣ РАСЧЁТ УРОНА

### 6.1 Формула урона

> **Урон = Ёмкость × Grade множитель**

```
finalDamage = capacity × gradeMult

где:
  capacity = затраченное Ци (ограничено ёмкостью техники)
  gradeMult = множитель Grade (×1.0 ~ ×1.6)
```

### 6.2 Пример расчёта

**Практик L5 использует melee_strike L5, Grade Perfect:**

```
Базовая ёмкость: baseCapacity = 64
Множитель уровня: 2^4 = 16
Мастерство: 0% → masteryBonus = 1.0

capacity = 64 × 16 × 1.0 = 1024 базового Ци
finalDamage = 1024 × 1.4 = 1433 урона
```

### 6.3 Множители Grade

| Grade | Множитель урона |
|-------|-----------------|
| Common | ×1.0 |
| Refined | ×1.2 |
| Perfect | ×1.4 |
| Transcendent | ×1.6 |

## 8️⃣ СТИХИИ

### 8.1 Единый перечень стихий (8 элементов)

> **Важно:** Набор стихий ЕДИН для всех систем игры.

```typescript
// Файл: src/types/technique-types.ts
export type TechniqueElement = 
  | 'fire'      // 🔥 Огонь
  | 'water'     // 💧 Вода
  | 'earth'     // 🪨 Земля
  | 'air'       // 💨 Воздух
  | 'lightning' // ⚡ Молния
  | 'void'      // 🌑 Пустота
  | 'neutral'   // ⚪ Нейтральный
  | 'poison';   // ☠️ Яд (особая стихия!)
```

### 8.2 Характеристики стихий

| Стихия | ID | Emoji | Цвет | Характер | Домен |
|--------|-----|-------|------|----------|-------|
| **Огонь** | `fire` | 🔥 | Красный | Разрушение, горение | Урон, DoT |
| **Вода** | `water` | 💧 | Синий | Плавность, течение | Контроль |
| **Земля** | `earth` | 🪨 | Коричневый | Прочность, устойчивость | Защита, стан |
| **Воздух** | `air` | 💨 | Серый | Скорость, лёгкость | Перемещение |
| **Молния** | `lightning` | ⚡ | Жёлтый | Скорость, разряд | Крит, цепи |
| **Пустота** | `void` | 🌑 | Фиолетовый | Антимагия, разложение | Пробитие |
| **Нейтральный** | `neutral` | ⚪ | Серый | Чистый Ци | Базовый урон |
| **Яд** | `poison` | ☠️ | Зелёный | Отравление, разложение | DoT, дебаффы |

### 8.3 ⚠️ ОГРАНИЧЕНИЯ ПО ТИПАМ ТЕХНИК

> **Критически важно:** Не все стихии доступны для всех типов техник!

| Тип техники | Допустимые стихии | Примечание |
|-------------|-------------------|------------|
| **Combat** | fire, water, earth, air, lightning, void, neutral | Все основные |
| **Defense** | fire, water, earth, air, lightning, void, neutral | Все основные |
| **Support** | fire, water, earth, air, lightning, void, neutral | Все основные |
| **Movement** | fire, water, earth, air, lightning, void, neutral | Все основные |
| **Sensory** | fire, water, earth, air, lightning, void, neutral | Все основные |
| **Curse** | fire, water, earth, air, lightning, void, neutral | Все основные |
| **Healing** | ⚪ **neutral ТОЛЬКО** | БЕЗ стихийных бонусов |
| **Cultivation** | ⚪ **neutral ТОЛЬКО** | БЕЗ стихий |
| **Poison** | ☠️ **poison ТОЛЬКО** | Особая стихия |

### 8.4 Применение стихий

Стихии применяются в:
- **Техники** — элемент техники (TechniqueElement)
- **Оружие** — урон стихией (включая poison)
- **Броня** — сопротивление стихии (включая poison)
- **Формации** — тип формации
- **NPC** — врождённая стихия

> **Важно:** Камни Ци НЕ имеют стихийного окраса. Это чистая энергия без элемента.

### 8.5 Мультипликаторы стихий (БЕЗ комбинирования)

> **Правило: Одна техника = одна стихия**

## 9️⃣ БОНУСЫ ТЕХНИК

### 9.1 Проблема разнообразия

При ограничениях по уровню:
- Практик L9 может использовать только техники L4-L9
- Техники L1-L3 недоступны для высоких практиков
- Теряется смысл в изучении нескольких техник одного уровня

### 9.2 Решение: Три уровня бонусов

```
┌─────────────────────────────────────────────────────────────────┐
│                    СИСТЕМА БОНУСОВ                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  БОНУС 1: Сила эффекта от Grade                                  │
│  ────────────────────────────                                   │
│  - Common: 0% (нет эффекта)                                     │
│  - Refined: 50% силы эффекта                                    │
│  - Perfect: 100% силы эффекта                                   │
│  - Transcendent: 150% силы эффекта                              │
│                                                                  │
│  БОНУС 2: Эффект от стихии (тип техники)                        │
│  ───────────────────────────────────                            │
│  Определяется стихией техники и её типом:                       │
│  - Атакующие: Дополнительный эффект стихии                      │
│  - Защитные: Сопротивление стихии                               │
│  - Поддержка: Временные модификаторы (НЕ изменяют stats!)       │
│                                                                  │
│  БОНУС 3: Transcendent-эффект (только для Transcendent)         │
│  ───────────────────────────────────────────────                │
│  Уникальный бонус, появляющийся только на Transcendent Grade:   │
│  - Усиление одного из эффектов (длительность +50%)              │
│  - Или уникальное свойство стихии                               │
│                                                                  │
│  ⚠️ ИСКЛЮЧЕНИЯ:                                                  │
│  - Healing: БЕЗ стихийных бонусов                               │
│  - Cultivation: БЕЗ стихий                                      │
│  - Poison: Особая механика (несколько дебаффов по Grade)        │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 1️⃣2️⃣ ПОЛНЫЙ РАСЧЁТ

### 12.1 Иерархия расчёта

```
1. Проверка совместимости по уровню
   └── canUse = techniqueLevel <= practitionerLevel (резонанс)
   └── Если false → техника НЕ активируется

2. Проверка ёмкости
   └── canHold = techniqueCapacity >= qiDensity
   └── Если false → дестабилизация
   
3. При дестабилизации:
   └── Излишки Ци рассеиваются
   └── Урон практику = excessQi × 0.5
   └── Урон по цели:
       ├── melee_strike / melee_weapon: inputQi × 0.5
       └── ranged_*: НЕТ урона (Ци разлетается)

4. При нормальном использовании:
   └── Расчёт ёмкости: capacity = baseCapacity × 2^(level-1) × masteryBonus
   └── Расчёт урона: damage = capacity × gradeMult
   └── Применение бонусов стихии (согласно ограничениям!)
```

### 12.2 Ключевой принцип

> **Урон = Ёмкость × Grade**
> 
> При переполнении:
> - Урон практику = 50% от излишков Ци
> - Урон по цели = 50% от влитого Ци (только ближний бой!)
> - Для дальних атак — Ци разлетается, урона по цели НЕТ

## 1️⃣4️⃣ ⭐ ПОДАВЛЕНИЕ УРОВНЕМ (Level Suppression)

### 14.1 Проблема

Qi Buffer с поглощением 90% позволяет практике L1 наносить урон практике L8 (10% от урона проходит).

### 14.2 Решение

Множитель подавления на основе разницы уровней культивации:

```typescript
// src/lib/constants/level-suppression.ts

export const LEVEL_SUPPRESSION_TABLE = {
  0: { normal: 1.0, technique: 1.0, ultimate: 1.0 },    // Тот же уровень
  1: { normal: 0.5, technique: 0.75, ultimate: 1.0 },   // +1 уровень
  2: { normal: 0.1, technique: 0.25, ultimate: 0.5 },   // +2 уровня
  3: { normal: 0.0, technique: 0.05, ultimate: 0.25 },  // +3 уровня
  4: { normal: 0.0, technique: 0.0, ultimate: 0.1 },    // +4 уровня
  5: { normal: 0.0, technique: 0.0, ultimate: 0.0 },    // +5+ уровней
};
```

### 14.3 Типы атак

| Тип | Описание | Пробитие |
|-----|----------|----------|
| `normal` | Обычная атака без техники | Минимальное |
| `technique` | Атака техникой | `technique.level` влияет |
| `ultimate` | Ultimate-техника (`isUltimate: true`) | Максимальное |

### 14.4 Формула

```typescript
function calculateLevelSuppression(
  attackerLevel: number,
  defenderLevel: number,
  attackType: 'normal' | 'technique' | 'ultimate',
  techniqueLevel?: number
): number {
  // Для техник: можно использовать technique.level для пробития
  let effectiveAttackerLevel = attackerLevel;
  if (attackType === 'technique' && techniqueLevel) {
    effectiveAttackerLevel = Math.max(attackerLevel, techniqueLevel);
  }
  
  const levelDiff = Math.max(0, defenderLevel - effectiveAttackerLevel);
  const clampedDiff = Math.min(5, levelDiff);
  
  return LEVEL_SUPPRESSION_TABLE[clampedDiff][attackType];
}
```

### 14.5 Примеры

| Атакующий | Защитник | Тип атаки | Множитель | Урон 39,321 → |
|-----------|----------|-----------|-----------|---------------|
| L9 | L9 | technique | 1.0 | 39,321 |
| L7 | L9 | normal | 0.0 | 0 |
| L7 | L9 | technique (L7) | 0.05 | 1,966 |
| L7 | L9 | technique (L8) | 0.25 | 9,830 |
| L5 | L9 | ultimate | 0.1 | 3,932 |

### 14.6 AOE атаки

При AOE атаке подавление применяется **ИНДИВИДУАЛЬНО** к каждой цели.

## 1️⃣4️⃣ ⭐ ПОДАВЛЕНИЕ УРОВНЕМ (НОВОЕ v3.0)

### 14.1 Проблема

> **Сырая Ци поглощает только 90% урона!**
> 
> Практик L8 получает 10% урона даже от L1 — противоречит жанру сянься.

### 14.2 Решение

**Техника уровня N пробивает до N уровней разницы.**

```

→ Пробивает до 5 уровней разницы
→ Может нанести урон практику L9 (diff=1) — ×0.75
→ НЕ может нанести урон практику L10 (diff=2) — но L10 не существует!
```

### 14.3 Таблица подавления

| effectiveDiff | normal | technique | ultimate |
|---------------|--------|-----------|----------|
| 0 | ×1.0 | ×1.0 | ×1.0 |
| +1 | ×0.5 | ×0.75 | ×1.0 |
| +2 | ×0.1 | ×0.25 | ×0.5 |
| +3 | ×0 | ×0.05 | ×0.25 |
| +4 | ×0 | ×0 | ×0.1 |
| +5+ | ×0 | ×0 | ×0 |

### 14.4 Формула

```
levelDiff = defenderLevel - attackerLevel
breakthrough = techniqueLevel
effectiveDiff = max(0, levelDiff - breakthrough)

attackType = isUltimate ? 'ultimate' : technique ? 'technique' : 'normal'
multiplier = SUPPRESSION_TABLE[effectiveDiff][attackType]
```

| Подавление +3 | ×0.25 | ×0.05 |
| Подавление +4 | ×0.1 | ×0 |
| Редкость | <1% | Обычная |

### 15.4 Интеграция в расчёт урона

```
1. Проверка совместимости по уровню (резонанс)
   └── techniqueLevel <= practitionerLevel
   
2. Расчёт ёмкости и базового урона
   └── damage = capacity × gradeMult
   
3. ⭐ ПОДАВЛЕНИЕ УРОВНЕМ (НОВОЕ!)
   └── attackType = isUltimate ? 'ultimate' : 'technique'
   └── multiplier = getSuppressionMultiplier(levelDiff, techniqueLevel, attackType)
   └── damage ×= multiplier
   └── if (multiplier === 0) return IMMUNE
   
4. Проверка дестабилизации (если применимо)
   
5. Применение защиты цели
```

### 16.5 AOE и подавление

**Важно:** Подавление рассчитывается ИНДИВИДУАЛЬНО для каждой цели!

```typescript
function processAOEAttack(
  attacker: Entity,
  technique: Technique,
  targets: Entity[]
) {
  const baseDamage = calculateBaseDamage(attacker, technique);
  
  for (const target of targets) {
    // AOE множитель (затухание от центра)
    const aoeMult = calculateAOEFalloff(distance);
    
    // ⭐ Подавление ИНДИВИДУАЛЬНО для каждой цели
    const suppressionMult = calculateLevelSuppression(
      attacker.cultivationLevel,
      target.cultivationLevel,
      technique.level,
      technique.isUltimate
    );
    
    const finalDamage = baseDamage * aoeMult * suppressionMult;
    
    if (suppressionMult > 0) {
      applyDamage(target, finalDamage);
    } else {
      log(`${target.name}: ИММУНИТЕТ (разница уровней)`);
    }
  }
}
```
