# 📊 Единая Система Бонусов и Штрафов

**Версия:** 2.0
**Дата:** 2026-03-16
**Статус:** 📋 Проектирование (итерация 2)

---

## 📋 Обзор

Документ описывает единую систему бонусов и штрафов, где **одни и те же переменные** используются для расчёта:
- **Бонусы** → положительные значения переменных
- **Штрафы** → отрицательные значения переменных

### Ключевые принципы

1. **Единые переменные** — бонус и штраф это знак (+/-) значения одной переменной
2. **Суммирование перед применением** — сначала сумма всех модификаторов, потом применение
3. **Мягкие капы** — защита от чрезмерного разгона (+200% скорости)
4. **Порядок расчёта** — плоские → сумма → проценты

## 1️⃣ Формула Расчёта

### 1.1 Базовый алгоритм

```
┌─────────────────────────────────────────────────────────────────────┐
│                    АЛГОРИТМ РАСЧЁТА                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  ШАГ 1: Сбор модификаторов                                          │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │  flat_modifiers = [+10, +5, -3, +8]                         │    │
│  │  percent_modifiers = [+10%, +5%, -15%, +20%]                │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                      │
│  ШАГ 2: Суммирование                                                │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │  flat_sum = 10 + 5 + (-3) + 8 = 20                          │    │
│  │  percent_sum = 10 + 5 + (-15) + 20 = 20%                    │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                      │
│  ШАГ 3: Применение                                                  │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │  result = (base_value + flat_sum) × (1 + percent_sum/100)   │    │
│  │  result = (100 + 20) × 1.20 = 144                           │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                      │
│  ШАГ 4: Мягкий кап                                                  │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │  result = apply_soft_cap(result, variable_id)               │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### 1.2 Формула в коде

```typescript
/**
 * Расчёт итогового значения переменной
 * 
 * @param baseValue - Базовое значение (от характеристик тела)
 * @param modifiers - Массив модификаторов
 * @param variableId - ID переменной (для определения капа)
 * @returns Итоговое значение
 */
function calculateFinalValue(
  baseValue: number,
  modifiers: Modifier[],
  variableId: string
): number {
  // 1. Разделение на плоские и процентные
  const flatModifiers = modifiers.filter(m => m.type === 'flat');
  const percentModifiers = modifiers.filter(m => m.type === 'percent');
  
  // 2. Сумма плоских модификаторов
  const flatSum = flatModifiers.reduce((sum, m) => sum + m.value, 0);
  
  // 3. Сумма процентных модификаторов (аддитивно!)
  // +20% и +10% = +30%, НЕ (+20% × 1.1 = +32%)
  const percentSum = percentModifiers.reduce((sum, m) => sum + m.value, 0);
  
  // 4. Применение: сначала плоские, потом проценты
  let result = baseValue + flatSum;
  result = result * (1 + percentSum / 100);
  
  // 5. Мягкий кап
  result = applySoftCap(result, baseValue, variableId);
  
  // 6. Жёсткие ограничения
  result = Math.max(result, 0); // Не может быть отрицательным
  
  return result;
}
```

### 1.3 Пример расчёта

```
Переменная: speed
Базовое значение: 100%

Экипировка:
  • Сапоги скорости: +10% скорости, +5 плоской скорости
  • Тяжёлая броня: -15% скорости
  • Пояс ловкости: +8% скорости
  • Проклятие медлительности: -20% скорости

Расчёт:
  1. Плоские: +5
  2. Проценты: +10% - 15% + 8% - 20% = -17%
  
  3. result = (100 + 5) × (1 - 0.17)
            = 105 × 0.83
            = 87.15
            
  4. Мягкий кап для скорости: min(87.15, 150) = 87.15
  5. Жёсткий минимум: max(87.15, 0) = 87.15

Итоговая скорость: 87.15%
```

## 2️⃣ Мягкие Капы (Soft Caps)

### 2.1 Зачем нужны

Без капов можно получить:
- +200% скорости → персонаж движется в 3 раза быстрее
- +150% урона → убивает врагов с одного удара
- -80% стоимости Ци → техники почти бесплатны

Мягкие капы позволяют получать бонусы сверх лимита, но с **убывающей эффективностью**.

### 2.2 Система Диминишинга (Diminishing Returns)

```typescript
/**
 * Применение мягкого капа
 * 
 * Формула: effective_bonus = cap × (1 - e^(-bonus/cap))
 * 
 * При бонусе = капу: ~63% эффективности
 * При бонусе = 2× кап: ~86% эффективности
 * При бонусе = 3× кап: ~95% эффективности
 */
function applySoftCap(
  finalValue: number,
  baseValue: number,
  variableId: string
): number {
  const capConfig = SOFT_CAPS[variableId];
  if (!capConfig) return finalValue;
  
  const { capType, capValue, decayRate } = capConfig;
  
  // Вычисляем общий бонус
  const totalBonus = finalValue - baseValue;
  
  // Если бонус в пределах капа — применяем полностью
  if (capType === 'positive' && totalBonus <= capValue) {
    return finalValue;
  }
  if (capType === 'absolute' && Math.abs(totalBonus) <= capValue) {
    return finalValue;
  }
  
  // Применяем диминишинг
  const sign = totalBonus > 0 ? 1 : -1;
  const absoluteBonus = Math.abs(totalBonus);
  
  // Экспоненциальное затухание
  const effectiveBonus = capValue * (1 - Math.exp(-absoluteBonus / (capValue * decayRate)));
  
  return baseValue + sign * effectiveBonus;
}
```

## 3️⃣ Порядок Применения: Плоские → Проценты

### 3.1 Почему именно так

**Вариант A (неверный):** Проценты от базы
```
База: 100
Плоские: +30
Проценты: +50%

result = 100 × 1.5 + 30 = 180
```

**Вариант B (правильный):** Проценты от суммы
```
База: 100
Плоские: +30
Проценты: +50%

result = (100 + 30) × 1.5 = 195
```

Вариант B даёт более интуитивный результат: проценты усиливают ВСЮ базу, включая плоские бонусы.

### 3.2 Почему проценты суммируются аддитивно

**Вариант A (мультипликативный):** 
```
База: 100
+20%, +20%, +20%

result = 100 × 1.2 × 1.2 × 1.2 = 172.8
```
Проблема: 3 предмета по +20% дают +72.8%, а не +60%

**Вариант B (аддитивный):**
```
База: 100
+20%, +20%, +20%

result = 100 × (1 + 0.6) = 160
```
Решение: 3 предмета по +20% дают ровно +60%

### 3.3 Финальная формула

```typescript
/**
 * Итоговая формула расчёта
 */
function calculateFinalValue(
  baseValue: number,
  modifiers: Modifier[],
  variableId: string
): number {
  // Шаг 1: Сумма плоских модификаторов
  const flatSum = modifiers
    .filter(m => m.type === 'flat')
    .reduce((sum, m) => sum + m.value, 0);
  
  // Шаг 2: Сумма процентных модификаторов (аддитивно!)
  const percentSum = modifiers
    .filter(m => m.type === 'percent')
    .reduce((sum, m) => sum + m.value, 0);
  
  // Шаг 3: Применение (плоские → проценты)
  let result = (baseValue + flatSum) * (1 + percentSum / 100);
  
  // Шаг 4: Мягкий кап
  result = applySoftCap(result, baseValue, variableId);
  
  // Шаг 5: Жёсткие ограничения
  result = Math.max(result, getMinValue(variableId));
  result = Math.min(result, getMaxValue(variableId));
  
  return result;
}
```

## 4️⃣ Модифицируемые Переменные

### 4.1 Переменные с двусторонними модификаторами

Эти переменные могут иметь как положительные, так и отрицательные значения.

| ID переменной | Описание | Единица | Мягкий кап | Жёсткий мин |
|---------------|----------|---------|------------|-------------|
| `speed` | Скорость передвижения | % | ±50% | 10% |
| `attack_speed` | Скорость атаки | % | +75% | 20% |
| `damage` | Наносимый урон | % | +100% | 0% |
| `defense` | Защита | % | +80% | 0% |
| `qi_cost` | Стоимость техник в Ци | % | -50% | 10% |
| `qi_efficiency` | Эффективность Ци | % | +50% | 0% |
| `stamina_cost` | Стоимость в выносливости | % | ±50% | 10% |
| `cooldown` | Время перезарядки | % | -60% | 20% |
| `crit_chance` | Шанс крита | % | +50% | 0% |
| `crit_damage` | Множитель крита | % | +150% | 100% |
| `stealth` | Скрытность | % | ±100% | 0% |
| `perception` | Восприятие | % | +100% | 0% |
| `healing_received` | Получаемое лечение | % | ±80% | 0% |
| `weight_capacity` | Переносимый вес | % | +100% | 20% |

### 4.2 Переменные с модификаторами через проклятия/эффекты

Эти переменные **обычно положительные**, но могут стать отрицательными через:
- Проклятия
- Порчу экипировки
- Специальные эффекты
- Потребление ресурса экипировкой

| ID переменной | Обычный источник | Отрицательный источник | Мягкий кап |
|---------------|------------------|------------------------|------------|
| `qi_regen` | Техники, медитация | **Ци-паразит** (проклятие) | +100% / -50% |
| `qi_conductivity` | Развитие меридиан | **Повреждение меридиан** | +50% / -30% |
| `hp_regen` | Техники, покой | **Гниющая плоть** (проклятие) | +50% / -30% |
| `life_steal` | Зачарования | **Кровавая клятва** (отдаёт HP) | +30% / N/A |
| `thorns` | Зачарования | — | +50% / N/A |
| `luck` | Артефакты | **Проклятие неудачи** | +50% / -30% |
| `exp_bonus` | Благословения | **Проклятие застоя** | +50% / -30% |

### 4.3 Переменные только с положительными модификаторами

Эти переменные **физически не могут** иметь отрицательные значения.

| ID переменной | Описание | Причина |
|---------------|----------|---------|
| `armor` | Броня | Броня — физический параметр, не может быть отрицательной |
| `elemental_fire` | Сила огня | Это "дополнительный" параметр, начинается с 0 |
| `elemental_water` | Сила воды | Аналогично |
| `elemental_earth` | Сила земли | Аналогично |
| `elemental_air` | Сила воздуха | Аналогично |
| `elemental_lightning` | Сила молнии | Аналогично |

### 4.4 Переменные только с отрицательными модификаторами

Эти переменные **отрицательные по определению** — их "улучшение" = уменьшение.

| ID переменной | Описание | Примечание |
|---------------|----------|------------|
| `damage_taken` | Получаемый урон | Положительное = хуже для игрока |
| `debuff_duration` | Длительность дебаффов | Положительное = хуже |
| `fall_damage` | Урон от падения | Положительное = больше урона |
| `hunger_rate` | Скорость голода | Положительное = быстрее голод |
| `durability_loss` | Скорость износа | Положительное = быстрее ломается |

### 4.5 Необратимые переменные (НЕ модифицируются бонусами)

Эти переменные **развиваются только естественным путём** согласно лору мира.

| ID переменной | Описание | Как развивается |
|---------------|----------|-----------------|
| `strength` | Сила | Тренировка и закалка мышц |
| `agility` | Ловкость | Тренировка связок и суставов |
| `intelligence` | Интеллект | Изучение и осознание |
| `vitality` | Живучесть | Развитие организма и культивация |
| `core_capacity` | Ёмкость ядра | Прорывы уровней культивации |
| `qi_density` | Плотность Ци | Уровень культивации (2^(level-1)) |

**Обоснование из лора (start_lore.md):**
> Базовые характеристики тела развиваются только через физическую и ментальную практику.
> Экипировка не может дать "бонус к силе" — она может лишь усилить эффект силы через Ци.

**Исключение:** Базовая проводимость меридиан (qi_conductivity) может быть **временно** модифицирована:
- Повреждение меридиан (проклятие, травма) → снижение
- Техники "очищения каналов" → временное повышение

## 7️⃣ Интеграция с Кодом

### 7.1 Типы

```typescript
// src/types/modifier.ts

/**
 * Тип модификатора
 */
export type ModifierType = 'flat' | 'percent';

/**
 * Источник модификатора
 */
export type ModifierSource = 
  | 'material'      // Материал экипировки
  | 'grade'         // Качество предмета
  | 'enchant'       // Зачарование
  | 'technique'     // Техника Ци
  | 'condition'     // Состояние (бафф/дебафф)
  | 'environment'   // Окружение
  | 'curse'         // Проклятие
  | 'blessing'      // Благословение
  | 'weight'        // Вес экипировки
  | 'durability';   // Состояние прочности

/**
 * Модификатор переменной
 */
export interface Modifier {
  variableId: string;
  type: ModifierType;
  value: number;      // Положительное = бонус, отрицательное = штраф
  source: ModifierSource;
  sourceId?: string;  // ID конкретного источника (предмет, техника)
  duration?: number;  // Для временных модификаторов (мс)
  stackable?: boolean; // Можно ли стакать
}

/**
 * Результат расчёта
 */
export interface CalculationResult {
  baseValue: number;
  flatSum: number;
  percentSum: number;
  rawResult: number;
  cappedResult: number;
  finalValue: number;
  appliedCap: boolean;
  capDetails?: {
    capValue: number;
    effectiveBonus: number;
  };
}
```

### 7.2 Калькулятор

```typescript
// src/lib/game/modifier-calculator.ts

import type { Modifier, CalculationResult } from '@/types/modifier';

export class ModifierCalculator {
  /**
   * Расчёт итогового значения переменной
   */
  calculate(
    baseValue: number,
    modifiers: Modifier[],
    variableId: string
  ): CalculationResult {
    // 1. Разделение и суммирование
    const flatSum = this.sumFlat(modifiers);
    const percentSum = this.sumPercent(modifiers);
    
    // 2. Применение формулы
    const rawResult = (baseValue + flatSum) * (1 + percentSum / 100);
    
    // 3. Мягкий кап
    const { cappedResult, appliedCap, capDetails } = this.applySoftCap(
      rawResult, 
      baseValue, 
      variableId
    );
    
    // 4. Жёсткие ограничения
    const finalValue = this.applyHardLimits(cappedResult, variableId);
    
    return {
      baseValue,
      flatSum,
      percentSum,
      rawResult,
      cappedResult,
      finalValue,
      appliedCap,
      capDetails,
    };
  }
  
  private sumFlat(modifiers: Modifier[]): number {
    return modifiers
      .filter(m => m.type === 'flat')
      .reduce((sum, m) => sum + m.value, 0);
  }
  
  private sumPercent(modifiers: Modifier[]): number {
    return modifiers
      .filter(m => m.type === 'percent')
      .reduce((sum, m) => sum + m.value, 0);
  }
  
  // ... остальная реализация
}

export const modifierCalculator = new ModifierCalculator();
```
