## ✅ ФИНАЛЬНЫЙ ВАРИАНТ: Система Порогов Развития

> **Статус:** ПРИНЯТ
> 
> **Обоснование:**
> 1. Естественное замедление без жёстких границ
> 2. Полная аналогия с ядром культивации
> 3. Каждое повышение значимо и ощутимо
> 4. Нет "стен" и "потолков" — развитие бесконечно

### Концепция

**Аналогия с ядром:** Чем больше характеристика, тем больше виртуальной дельты нужно для повышения.

```typescript
// Формула порога

// Стат 10-19: threshold = 1.0 (нужно 1.0 для +1)
// Стат 20-29: threshold = 2.0 (нужно 2.0 для +1)

// Стат 50-59: threshold = 5.0 (нужно 5.0 для +1)
```

## 7️⃣ Формула сна с капом

### Кап +0.20 (финальный)

```typescript
const MAX_CONSOLIDATION_PER_SLEEP = 0.20;

function calculateMaxConsolidation(sleepHours: number): number {
  if (sleepHours < 4) return 0;
  // 4 часа = +0.067, 6 часов = +0.133, 8 часов = +0.20
  const base = 0.067;
  const perHour = 0.033;
  return Math.min(MAX_CONSOLIDATION_PER_SLEEP, base + (sleepHours - 4) * perHour);
}
```

## 9️⃣ Финальная формула (Система Порогов)

```typescript
// Базовый кап за сон
const MAX_CONSOLIDATION_PER_SLEEP = 0.20;

// Система порогов развития
function calculateStatThreshold(currentStat: number): number {
  // Чем выше стат, тем больше нужно виртуальной дельты для +1
  return Math.max(1.0, Math.floor(currentStat / 10));
}

// Процесс повышения при сне
function processStatAdvancement(stat: StatDevelopment): void {
  const threshold = calculateStatThreshold(stat.current);
  
  // Сколько раз можно повысить (если накоплено много)
  while (stat.virtualDelta >= threshold) {
    stat.current += 1;
    stat.virtualDelta -= threshold;
    // Новый порог будет выше!
  }
}
```

### Преимущества системы порогов

1. **Нет жёсткого капа** — развитие бесконечно
2. **Естественное замедление** — каждое +1 становится сложнее
3. **Аналогия с ядром** — чем больше сосуд, тем больше наполнять
4. **Каждое повышение значимо** — нет обесценивания прогресса
