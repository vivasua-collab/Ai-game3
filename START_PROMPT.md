## ПРАВИЛА РАБОТЫ

### 1. Дата и время
```bash
date '+%Y-%m-%d %H:%M:%S %Z'
```
Использовать для всех операций.

### 2. Комментарии в файлах ⚠️ ОБЯЗАТЕЛЬНО

**КАЖДЫЙ раз при редактировании файла — СТАВИТЬ ОТМЕТКУ ДАТЫ И ВРЕМЕНИ. Без исключений. Без оправданий.**

**Создание:**
```csharp
// Создано: YYYY-MM-DD HH:MM:SS UTC
```

**Редактирование:**
```csharp
// Редактировано: YYYY-MM-DD HH:MM:SS UTC
```

**⚠️ ЭТО ПРАВИЛО — НЕ РЕКОМЕНДАЦИЯ, А ЖЁСТКОЕ ТРЕБОВАНИЕ:**
- Отметка `// Редактировано: ...` ставится В КАЖДОМ файле, который был изменён
- Отметка ставится НЕМЕДЛЕННО при редактировании, не «потом»
- Без отметки — работа считается НЕВЫПОЛНЕННОЙ
- Если файл редактировался несколько раз — обновлять отметку на последнюю

**⚠️ КОММЕНТАРИИ ТОЛЬКО НА РУССКОМ ЯЗЫКЕ:**
- Все комментарии в коде (`// ...`) пишутся ТОЛЬКО на русском языке
- Исключения: имена классов/методов/переменных (они на английском по правилам C#)
- Технические термины можно на английском, но пояснения — на русском

### 3. Актуальность документации
Файл с датой редактирования ближе к текущей — актуальный.

### 6. GitHub токен

### 7. Замороженные скрипты и архитектура ⚠️

**Оркестратор FullSceneBuilder.cs ЗАМОРОЖЕН.** Архитектура «оркестратор + фазовые файлы» НЕ подлежит изменению.
- НЕ переписывать оркестратор в монолит
- НЕ объединять фазовые файлы обратно
- НЕ менять интерфейс IScenePhase

Подробности → `docs_asset_setup/SCENE_BUILDER_ARCHITECTURE.md`

**Правило:** Изменения сцены → редактировать нужный `PhaseNNXxx.cs`. FullSceneBuilder.cs — только добавление строки в `PHASES` для новой фазы (с подтверждением). ScenePatchBuilder.cs — УСТАРЕЛ, НЕ добавлять патчи.
