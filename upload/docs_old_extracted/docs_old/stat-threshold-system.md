## 🎯 Концепция

**Аналогия с ядром культивации:** Чем больше ядро, тем больше Ци нужно для прорыва. Аналогично — чем выше характеристика, тем больше "виртуальной дельты" нужно для её повышения.

**Название:** **Порог Развития** (Development Threshold) или **Ёмкость Характеристики** (Stat Capacity)

## 1️⃣ Механика системы

### 1.1 Базовый принцип

```
┌─────────────────────────────────────────────────────────────────┐
│                  СИСТЕМА ПОРОГОВ РАЗВИТИЯ                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   Аналогия с ядром:                                              │
│   ──────────────────                                             │
│   Ядро L1 → L2: нужно заполнить ядро 10 раз                     │
│   Ядро L2 → L3: нужно заполнить ядро 20 раз (+10)               │
│   Ядро L9 → L10: нужно заполнить ядро 90 раз                    │
│                                                                  │
│   Характеристика:                                                │
│   ─────────────────                                              │
│   Стат 10 → 11: нужно накопить 1.0 виртуальной дельты           │
│   Стат 20 → 21: нужно накопить 2.0 виртуальной дельты           │
│   Стат 50 → 51: нужно накопить 5.0 виртуальной дельты           │
│   Стат 100 → 101: нужно накопить 10.0 виртуальной дельты        │
│                                                                  │
│   Формула:                                                       │
│   ─────────                                                      │
│   requiredDelta = floor(currentStat / 10)                       │
│   (Минимум 1.0)                                                  │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 Формула порога

```typescript
/**
 * Расчёт порога развития для следующей единицы характеристики
 * 
 * @param currentStat Текущее значение характеристики
 * @returns Требуемая виртуальная дельта для повышения на 1
 */
function calculateStatThreshold(currentStat: number): number {
  // Базовый порог: 1.0 для стата 10-19
  // Затем: floor(stat / 10)
  return Math.max(1.0, Math.floor(currentStat / 10));
}

// Примеры:
// Стат 10: threshold = 1.0  → нужно 1.0 для 11
// Стат 15: threshold = 1.0  → нужно 1.0 для 16
// Стат 20: threshold = 2.0  → нужно 2.0 для 21
// Стат 25: threshold = 2.0  → нужно 2.0 для 26
// Стат 30: threshold = 3.0  → нужно 3.0 для 31
// Стат 50: threshold = 5.0  → нужно 5.0 для 51
// Стат 100: threshold = 10.0 → нужно 10.0 для 101
```

```
│                 ПРОЦЕСС ПОВЫШЕНИЯ ХАРАКТЕРИСТИКИ                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   Пример: Сила 20 → 21                                          │
│                                                                  │
│   Шаг 1: Накопление виртуальной дельты                          │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │ Текущий стат: 20                                         │   │
│   │ Порог: floor(20/10) = 2.0                               │   │
│   │                                                         │   │
│   │ Виртуальная дельта:                                     │   │
│   │ День 1: +0.20 → total 0.20                              │   │
│   │ День 2: +0.20 → total 0.40                              │   │
│   │ ...                                                      │   │
│   │ День 10: +0.20 → total 2.00 ← ПОРОГ ДОСТИГНУТ!         │   │
│   └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│   Шаг 2: Повышение (автоматически при сне)                      │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │ При сне:                                                 │   │
│   │ - Проверка: virtualDelta >= threshold                   │   │
│   │ - Если да: stat += 1, virtualDelta -= threshold         │   │

│   │                                                         │   │
│   │ Результат: Сила 21, virtualDelta: 0.0                   │   │
│   └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│   Пример: Сила 29 → 30                                          │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │ Порог для 29→30: floor(29/10) = 2.0                     │   │
│   │ После повышения: Сила 30                                │   │
│   │ Новый порог: floor(30/10) = 3.0 ← УВЕЛИЧИЛСЯ!           │   │
│   └─────────────────────────────────────────────────────────┘   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 2️⃣ Структура данных

### 2.1 Интерфейс характеристики

```typescript
interface StatDevelopment {
  // Текущее значение
  current: number;           // Например: 25.0
  
  // Виртуальная дельта (накопленная)
  virtualDelta: number;      // Например: 1.47
  
  // Порог для следующего повышения (вычисляемый)
  threshold: number;         // floor(25/10) = 2.0
  
  // Прогресс до следующего повышения
  progress: number;          // virtualDelta / threshold = 0.735 (73.5%)
}

interface CharacterStatsDevelopment {
  strength: StatDevelopment;
  agility: StatDevelopment;
  intelligence: StatDevelopment;
  vitality: StatDevelopment;
}
```

### 2.2 Функции расчёта

```typescript
/**
 * Получить порог для характеристики
 */
function getStatThreshold(currentStat: number): number {
  return Math.max(1.0, Math.floor(currentStat / 10));
}

/**
 * Получить прогресс до повышения
 */
function getStatProgress(stat: StatDevelopment): number {
  return stat.virtualDelta / stat.threshold;
}

/**
 * Проверить, можно ли повысить характеристику
 */
function canAdvanceStat(stat: StatDevelopment): boolean {
  return stat.virtualDelta >= stat.threshold;
}

/**
 * Повысить характеристику (при сне)
 */
function advanceStat(stat: StatDevelopment): { newStat: number; remainingDelta: number } | null {
  if (!canAdvanceStat(stat)) return null;
  
  const newStat = stat.current + 1;
  const remainingDelta = stat.virtualDelta - stat.threshold;
  
  return {
    newStat,
    remainingDelta,
  };
}
```

## 3️⃣ Расчёт развития во времени

### 3.1 Допущения

- **Кап закрепления за сон:** +0.20 (8 часов)
- **Активных дней:** 75%
- **Распределение:** 1 характеристика (фокусная тренировка)

### 3.2 Расчёт времени повышения

```typescript
// Функция расчёта дней для повышения с X на X+1
function daysToAdvance(
  currentStat: number,
  dailyConsolidation: number,  // +0.20
  activeDaysRatio: number = 0.75
): number {
  const threshold = getStatThreshold(currentStat);
  const effectiveDaily = dailyConsolidation * activeDaysRatio;
  return Math.ceil(threshold / effectiveDaily);
}
```

### 3.4 Расчёт за периоды

**За 1000 дней (750 активных):**
```
Накопление виртуальной дельты: 750 × 0.20 = 150.0

Расчёт достижимости:
- 10 → 20: 10 повышений × 1.0 = 10.0 виртуальной дельты
- 20 → 30: 10 повышений × 2.0 = 20.0 виртуальной дельты
- 30 → 40: 10 повышений × 3.0 = 30.0 виртуальной дельты
- 40 → 50: 10 повышений × 4.0 = 40.0 виртуальной дельты
- 50 → 55: 5 повышений × 5.0 = 25.0 виртуальной дельты

Итого: 10 + 20 + 30 + 40 + 25 = 125.0 виртуальной дельты
Остаток: 150.0 - 125.0 = 25.0

Можно достичь: ~55-56 стата за 1000 дней
```

**За 10000 дней (7500 активных):**
```
Накопление: 7500 × 0.20 = 1500.0 виртуальной дельты

Расчёт:
- 10 → 20: 10 × 1.0 = 10
- 20 → 30: 10 × 2.0 = 20
- 30 → 40: 10 × 3.0 = 30
- 40 → 50: 10 × 4.0 = 40
- 50 → 60: 10 × 5.0 = 50
- 60 → 70: 10 × 6.0 = 60
- 70 → 80: 10 × 7.0 = 70
- 80 → 90: 10 × 8.0 = 80
- 90 → 100: 10 × 9.0 = 90
- 100 → 110: 10 × 10.0 = 100
- 110 → 120: 10 × 11.0 = 110
- 120 → 125: 5 × 12.0 = 60

Сумма до 100: 10+20+30+40+50+60+70+80+90 = 450
Сумма 100→125: 100+110+60 = 270
Итого до 125: 720.0 виртуальной дельты

Можно достичь: ~125 стата за 10000 дней
```

## 5️⃣ Преимущества системы порогов

### 5.1 Естественное замедление

- Нет искусственных стен
- Игрок всегда видит прогресс
- Каждое повышение значимо

### 5.2 Связь с ядром культивации

```
┌─────────────────────────────────────────────────────────────────┐
│                ПАРАЛЛЕЛЬ С ЯДРОМ КУЛЬТИВАЦИИ                     │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   Ядро:                         Характеристика:                 │
│   ──────                        ────────────────                 │
│   L1 → L2: 10 заполнений        10 → 11: 1.0 виртуальной        │
│   L2 → L3: 20 заполнений        20 → 21: 2.0 виртуальной        │
│   L3 → L4: 30 заполнений        30 → 31: 3.0 виртуальной        │
│   ...                           ...                              │
│                                                                  │
│   Общая логика:                                                  │
```

## 7️⃣ Рекомендация

**Принять базовую формулу:**

```typescript
threshold = Math.max(1.0, Math.floor(currentStat / 10));
```

**Обоснование:**
1. Простая и понятная
2. Аналогия с ядром (кратность 10)
3. Предсказуемый рост
4. Естественное замедление без стен
5. Баланс для длинных сессий
