## 🎯 Концепция: "Облачная игра для одного"

### Метафора Трёх Уровней: Божество → Облако → Земля

```
┌──────────────────────────────────────────────────────────────────────────────┐
│                   МЕТАФОРА: БОЖЕСТВО → ОБЛАКО → ЗЕМЛЯ                         │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│   👁️ БОЖЕСТВО (Игрок)                                                       │
│   ├── Управляет аватаром в мире                                              │
│   ├── Принимает решения (атака, движение, отдых)                            │
│   ├── Видит мир через "облако" (экран)                                      │
│   └── Его воля = действия персонажа                                         │
│                                                                              │
│   ☁️ ОБЛАКО (Браузер / Thin Client)                                         │
│   ├── Посредник между божеством и миром                                     │
│   ├── Отображает мир (Phaser рендеринг)                                     │
│   ├── Передаёт волю божества на землю (HTTP requests)                       │
│   └── НЕ принимает решений - только транслирует                             │
│                                                                              │
│   🌍 ЗЕМЛЯ (Сервер - TruthSystem)                                           │
│   ├── Реальный мир, где всё происходит                                      │
│   ├── Хранит состояние (HP, Qi, NPC, мир)                                   │
│   ├── Исполняет расчёты (урон, культивация, AI)                            │
│   └── 1 TICK = 1 СЕКУНДА реального времени                                  │
│                                                                              │
│   ═════════════════════════════════════════════════════════════════════════  │
│                                                                              │
│   ПОТОК:                                                                     │
│   👁️ Божество → "Хочу атаковать" → ☁️ Облако → HTTP POST → 🌍 Земля        │
│   🌍 Земля → расчёт урона, HP, Qi → результат → ☁️ Облако → 👁️ Божество    │
│                                                                              │
│   Аналогия:                                                                  │
│   ├── Божество = игрок за экраном, направляет аватара                       │
│   ├── Облако = браузер, показывает мир и передаёт команды                   │
│   └── Земля = сервер, где живёт настоящий мир и его правила                 │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘
```

## 🔄 Принцип взаимодействия

### "Тонкий клиент" (Thin Client)

```
┌──────────────────────────────────────────────────────────────────────────────┐
│                      ПРИНЦИП: ТОНКИЙ КЛИЕНТ                                   │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│   Клиент (Browser) ──────────────────────────────────────────────────────▶  │
│                                                                              │
│   1. ПОЛУЧАЕТ данные для отображения                                        │
│      ├── GET /api/game/state → { character, location, npcs, time }          │
│      └── Клиент только рендерит                                             │
│                                                                              │
│   2. ОТПРАВЛЯЕТ действия на сервер                                          │
│      ├── POST /api/combat → { attackerId, targetId, techniqueId }           │
│      ├── Сервер рассчитывает урон                                           │
│      └── Сервер возвращает результат → Клиент отображает                    │
│                                                                              │
│   3. НИКОГДА не принимает решений                                           │
│      ├── ❌ "Достаточно ли Ци?" → Сервер решает                             │
│      ├── ❌ "Сколько урона?" → Сервер рассчитывает                          │
│      ├── ❌ "Мёртв ли NPC?" → Сервер определяет                             │
│      └── ✅ Только отображает результат                                      │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘
```

### Пример: Атака техникой

```
┌──────────────────────────────────────────────────────────────────────────────┐
│                      ПРИМЕР: ПОТОК АТАКИ                                      │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│   CLIENT (Phaser)                                                           │
│   │                                                                          │
│   ├── 1. Игрок нажимает кнопку атаки                                        │
│   │   │                                                                      │
│   │   └── Клиент: "Игрок хочет атаковать"                                   │
│   │       └── НЕ проверяет Qi, НЕ рассчитывает урон                         │
│   │                                                                          │
│   ├── 2. Отправка запроса                                                   │
│   │   │                                                                      │
│   │   └── POST /api/combat {                                                │
│   │         action: 'attack',                                               │
│   │         attackerId: 'player_1',                                         │
│   │         targetId: 'npc_wolf_01',                                        │
│   │         techniqueId: 'fireball_001'                                     │
│   │       }                                                                  │
│   │                                                                          │
│   │   ┌─────────────────────────────────────────────────────────────────┐   │
│   │   │                      SERVER PROCESSING                          │   │
│   │   │                                                                 │   │
│   │   │   1. Проверка Qi: character.qi >= technique.qiCost              │   │
│   │   │   2. Списание Qi: character.qi -= qiCost                        │   │
│   │   │   3. Расчёт урона:                                              │   │
│   │   │      baseDamage = technique.baseDamage                          │   │
│   │   │      masteryBonus = 1 + (mastery * 0.5)                         │   │
│   │   │      conductivityBonus = 1 + (conductivity * 0.3)               │   │
│   │   │      finalDamage = baseDamage * masteryBonus * conductivityBonus│   │
│   │   │   4. Применение урона к NPC:                                    │   │
│   │   │      npc.hp -= finalDamage                                      │   │
│   │   │      if (npc.hp <= 0) → npc.state = 'dead'                      │   │
│   │   │   5. Генерация лута (если NPC умер)                             │   │
│   │   │   6. Сохранение изменений в TruthSystem                         │   │
│   │   │                                                                 │   │
│   │   └─────────────────────────────────────────────────────────────────┘   │
│   │                                                                          │
│   ├── 3. Получение ответа                                                   │
│   │   │                                                                      │
│   │   └── {                                                                 │
│   │         success: true,                                                  │
│   │         damage: 45,                                                     │
│   │         currentQi: 150,                                                 │
│   │         targetHp: 0,                                                    │
│   │         targetDead: true,                                               │
│   │         loot: [{ id: 'qi_stone_001', type: 'qi_stone', value: 50 }]    │
│   │       }                                                                  │
│   │                                                                          │
│   └── 4. Отображение результата                                             │
│       ├── Анимация атаки                                                    │
│       ├── Число урона "-45"                                                 │
│       ├── Смерть NPC                                                        │
│       └── Выпадение лута                                                    │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘
```

## ⚡ Преимущества HTTP-Only архитектуры

### Для однопользовательской игры

```
┌──────────────────────────────────────────────────────────────────────────────┐
│                         ПРЕИМУЩЕСТВА                                          │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│   ✅ ПРОСТОТА                                                                │
│      ├── Один протокол (HTTP)                                               │
│      ├── Нет WebSocket соединений                                           │
│      ├── Проще debug                                                        │
│      └── Меньше точек отказа                                                │
│                                                                              │
│   ✅ НАДЁЖНОСТЬ                                                              │
│      ├── HTTP запросы атомарны                                              │
│      ├── Нет состояния соединения                                           │
│      ├── Автоматический retry через браузер                                 │
│      └── Потеря сети = повторный запрос                                     │
│                                                                              │
│   ✅ БЕЗОПАСНОСТЬ                                                            │
│      ├── Вся логика на сервере                                              │
│      ├── Клиент не может "накрутить"                                        │
│      ├── Валидация на сервере                                               │
│      └── Невозможность читерства через клиент                               │
│                                                                              │
│   ✅ ПРОИЗВОДИТЕЛЬНОСТЬ                                                      │
│      ├── 1 TICK = 1 СЕКУНДА                                                 │
│      ├── HTTP latency (10-100мс) << 1000мс                                  │
│      ├── Вполне достаточно для turn-based механики                          │
│      └── Кэширование на клиенте для отображения                             │
│                                                                              │
│   ✅ МАСШТАБИРУЕМОСТЬ                                                        │
│      ├── Легко добавить WebSocket в будущем                                 │
│      ├── Готово для мультиплеера                                            │
│      └── API уже отделён от клиента                                         │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘
```

## 🔀 Сравнение: HTTP vs WebSocket

### Когда использовать что

```
┌──────────────────────────────────────────────────────────────────────────────┐
│                    HTTP vs WebSocket: ВЫБОР                                   │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│   HTTP (Текущий выбор) ───────────────────────────────────────────────────▶  │
│                                                                              │
│   ✅ Подходит для:                                                           │
│      ├── Однопользовательская игра                                          │
│      ├── 1 TICK = 1 СЕКУНДА                                                 │
│      ├── Turn-based механика                                                │
│      ├── Действия по запросу игрока                                         │
│      └── Периодические обновления состояния                                 │
│                                                                              │
│   ❌ НЕ подходит для:                                                        │
│      ├── Real-time PvP бои                                                  │
│      ├── Массовые сражения (100+ юнитов)                                    │
│      ├── Высокочастотные обновления (<100мс)                                │
```

```typescript
async function useTechnique(techniqueId: string, targetId: string) {

  }
}
```

### Пример 3: Серверная обработка боя

```typescript
// SERVER: /api/combat/route.ts
export async function POST(request: Request) {
  const { action, techniqueId, targetId } = await request.json();

  // 1. Получаем состояние из TruthSystem
  const session = TruthSystem.getSession(sessionId);
  const character = session.character;
  const technique = await db.technique.findUnique(techniqueId);

  // 2. Проверки
  if (character.qi < technique.qiCost) {
    return Response.json({ success: false, error: 'Недостаточно Ци' });
  }

  // 3. Списание Qi
  TruthSystem.spendQi(sessionId, technique.qiCost);

  // 4. Расчёт урона
  const damage = calculateDamage(character, technique, target);

  // 5. Применение урона
  target.hp -= damage;

  // 6. Проверка смерти
  let loot = null;
  if (target.hp <= 0) {
    target.state = 'dead';
    loot = await generateLoot(target);
  }

  // 7. Сохранение изменений
  TruthSystem.updateNPC(sessionId, targetId, target);

  // 8. Возврат результата
  return Response.json({
    success: true,
    damage,
    currentQi: character.qi,
    targetHp: target.hp,
    targetDead: target.hp <= 0,
    loot
  });
}
```

### Решение: Гибридный подход

```
┌──────────────────────────────────────────────────────────────────────────────┐
│                      ПРИНЯТОЕ РЕШЕНИЕ                                         │
├──────────────────────────────────────────────────────────────────────────────┤

│                                                                              │
│   ├── /api/combat          → Бой (высокочастотный)                           │
│   ├── /api/ai/sync         → AI синхронизация (tick-based)                   │
│   ├── /api/game/move       → Движение                                        │
│   └── /api/rest            → Отдых, медитация                                │
│                                                                              │

│                                                                              │
│   ├── inventory:events     → Инвентарь (низкочастотный)                      │
│   ├── environment:events   → Окружение                                       │
│   └── body:events          → Состояние тела                                  │

│   ПРИЧИНА:                                                                    │
│   ├── Combat и AI требуют максимальной производительности                    │
│   ├── Инвентарь и окружение менее критичны к latency                         │
│   └── Event Bus удобен для сложных событий с middleware                      │
│                                                                              │
```

## 📋 Чек-лист архитектуры

### Требования к коду

| Правило | Описание |
|---------|----------|
| ✅ Сервер - источник истины | Все расчёты на сервере |
| ✅ Клиент - только отображение | Никакой бизнес-логики в браузере |
| ✅ HTTP для всех операций | POST/GET к API routes |
| ✅ TruthSystem для состояния | Память первична, БД вторична |
| ✅ Валидация на сервере | Нельзя доверять клиенту |
| ✅ Атомарные операции | Запрос → Расчёт → Ответ |
| ❌ Не использовать localStorage | Заблокирован в sandbox |
| ❌ Не хранить состояние на клиенте | Только кэш для отображения |
