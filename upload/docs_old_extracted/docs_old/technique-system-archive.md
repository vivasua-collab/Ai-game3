## 🎯 ИДЕЯ

Вместо множителя уровня техники — использовать **кап по базовому Ци**.

### Лоровое обоснование:

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    КОНЦЕПЦИЯ "БАЗОВОГО ЦИ"                               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  Практик L1:                                                             │
│  - Ци "разрежённое", качество = 1                                        │
│  - 1 единица Ци практики = 1 единица базового Ци                         │
│  - Может использовать технику L1 без проблем                             │
│                                                                          │
│  Практик L9:                                                             │
│  - Ци "сжатое", качество = 256 (qiDensity)                              │
│  - 1 единица Ци практики = 256 единиц базового Ци                        │
│  - Техника L1 (ёмкость 50) переполняется от 1 единицы!                   │
│                                                                          │
│  ВЫВОД: Техника низкого уровня не может обработать плотное Ци мастера!  │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

---

## 📐 ФОРМУЛА

### Предлагаемая (ПРИНЯТА):

```typescript
// БАЗОВОЕ Ци = Ци практики × qiDensity
const baseQiInput = qiCost * qiDensity;
const stability = checkDestabilization(baseQiInput, capacity);
```

**Ключевое изменение:** Входящее Ци умножается на qiDensity!

## 📊 ТАБЛИЦА ПЕРЕПОЛНЕНИЯ

### Практик L1 (qiDensity = 1), техника L1 (capacity = 50)

| Ци практики | Базовое Ци | Ёмкость | Статус |
|-------------|------------|---------|--------|
| 10 | 10 | 50 | ✅ OK |
| 50 | 50 | 50 | ✅ OK (лимит) |
| 60 | 60 | 50 | ⚠️ Дестабилизация |

### Практик L5 (qiDensity = 16), техника L1 (capacity = 50)

| Ци практики | Базовое Ци | Ёмкость | Статус |
|-------------|------------|---------|--------|
| 1 | 16 | 50 | ✅ OK |
| 3 | 48 | 50 | ✅ OK |
| 4 | 64 | 50 | ⚠️ Дестабилизация |
| 10 | 160 | 50 | ❌ СИЛЬНАЯ дестабилизация |

### Практик L9 (qiDensity = 256), техника L1 (capacity = 50)

| Ци практики | Базовое Ци | Ёмкость | Статус |
|-------------|------------|---------|--------|
| 1 | 256 | 50 | ❌ Дестабилизация! |

**ВЫВОД: Практик L9 НЕ может использовать технику L1!**
Даже 1 единица Ци практики = 256 базового Ци, что превышает ёмкость 50!

---

## 🎯 ПРЕИМУЩЕСТВА

### 1. Лоровое обоснование

> *"Техника низкого уровня имеет примитивную структуру, которая разрушается от плотного Ци мастера. Как пытаться пропустить Niagara через соломинку!"*

### 2. Без множителей

- Не нужно добавлять `techniqueLevelMult`
- Работает через существующую систему дестабилизации
- Естественное ограничение

### 3. Мягкое ограничение

- Техника L1 не запрещена для L9
- Просто невыгодна: 1 Ци → дестабилизация
- Можно использовать... но себе во вред!

### 4. "Пушка по воробьям" решена

- Низкие техники опасны для мастеров
- Высокие техники эффективны
- Естественный баланс

---

## 🎯 РЕКОМЕНДАЦИЯ (ВЫПОЛНЕНО)

### Принять вариант "Кап по базовому Ци"!

**Причины:**
1. ✅ Лоровое обоснование (структура техники vs плотность Ци)
2. ✅ Естественное ограничение без искусственных множителей
3. ✅ Работает через существующую дестабилизацию
4. ✅ "Ультимативная" техника L1 невозможна для L9
5. ✅ Поощряет изучение высоких техник

**Формула:**
```typescript
const baseQiInput = qiCost * qiDensity;
const stability = checkDestabilization(baseQiInput, capacity);
const damage = stability.effectiveQi * statMult * masteryMult * rarityMult;
```

## 🔴 КРИТИЧЕСКАЯ ПРОБЛЕМА: Техника L1 как "Ультимативная"

### ❌ ПРОБЛЕМА

**Практик L9 может использовать технику L1!**

Это приводит к:
1. Дестабилизации при вложении >55 Ци
2. Backlash damage
3. Но технически ВОЗМОЖНО!

### Согласно документации (technique-system-v2.md):

```
Практик L9 использует технику L1 со 100 Ци:
  
  techniqueCapacity = 50 Ци
  qiInput = 100 Ци
  
  Результат:
  - Перегрузка: 100 > 55 (110% от 50)
  - efficiency = 50/100 = 0.5 (50%)
  - backlashDamage = (100-50) × 0.5 = 25 урона себе!
  - Итоговый урон: только 50 × qiDensity
```

## 🎯 РЕШЕНИЕ (ПРИНЯТО)

Использовать систему **базового Ци + ёмкость** вместо множителя уровня.

Это решает проблему естественным образом:
- L9 практик с L1 техникой получает дестабилизацию
- Урон ограничен ёмкостью техники
- Backlash наказывает за использование слабых техник

## 📋 НАЙДЕННЫЕ ОГРАНИЧЕНИЯ

### 1. minCultivationLevel (нижняя граница)

**Генерация в technique-generator.ts:**

| Тип техники | Формула | Пример |
|-------------|---------|--------|
| Combat (melee_strike) | `Math.max(1, level - 1)` | L5 техника → minCultivationLevel = 4 |
| Cultivation | `level` | L5 техника → minCultivationLevel = 5 |
| Curse | `level` | L5 техника → minCultivationLevel = 5 |

### 2. Проверка в canUseTechnique

```typescript
if (character.cultivationLevel < technique.minCultivationLevel) {
  return {
    canUse: false,
    reason: `Требуется уровень культивации ${technique.minCultivationLevel}`,
  };
}
```

## 💡 ПРЕДЛОЖЕНИЕ: Множитель уровня техники

### Формула:

```typescript
// НОВОЕ: Множитель от уровня техники
const techniqueLevelMult = Math.pow(2, (techniqueLevel - 1) / 2);

// Урон = effectiveQi × qiDensity × techniqueLevelMult
let damage = effectiveQi * qiDensity * techniqueLevelMult;
```

### Таблица множителя:

| Уровень техники | Множитель | Формула |
|-----------------|-----------|---------|
| L1 | ×1.0 | 2^0 |
| L2 | ×1.4 | 2^0.5 |
| L3 | ×2.0 | 2^1 |
| L4 | ×2.8 | 2^1.5 |
| L5 | ×4.0 | 2^2 |
| L6 | ×5.7 | 2^2.5 |
| L7 | ×8.0 | 2^3 |
| L8 | ×11.3 | 2^3.5 |
| L9 | ×16.0 | 2^4 |

## 📊 НОВАЯ ТАБЛИЦА УРОНА

### Практик L9 (qiDensity = 256), максимум Ци

| Техника | effectiveQi | Множитель | Итоговый урон |
|---------|-------------|-----------|---------------|
| L1 | 50 | ×1.0 | 12,800 |
| L3 | 200 | ×2.0 | 102,400 |
| L5 | 800 | ×4.0 | 819,200 |
| L7 | 3,200 | ×8.0 | 6,553,600 |
| L9 | 12,800 | ×16.0 | 52,428,800 |

---

## ❌ ПРИЧИНА ОТКЛОНЕНИЯ

Данное решение было **отклонено** в пользу системы **базового Ци + ёмкость по типу техники**.

**Причины:**
1. Система capacity решает проблему естественнее через лор
2. Не требует дополнительного множителя
3. Связывает тип техники с ёмкостью ( Formation > Defense > Combat > Ranged )
4. Дестабилизация уже ограничивает слабые техники

## 🎯 ПРИНЯТАЯ КОНЦЕПЦИЯ

```
Базовое Ци = Ци практики × qiDensity

Ёмкость техники = baseCapacityByType × 2^(techniqueLevel - 1)

Техника переполняется, если:
Базовое Ци > Ёмкость × 1.1 (110% запас)
```
