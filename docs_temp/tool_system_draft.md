## 📋 Обзор

Инструменты — экипируемые предметы, расширяющие возможности игрока во взаимодействии с миром:
- Добыча ресурсов (руды, дерева, камня, трав)
- Ремонт экипировки
- Крафт (кузнечный молот, алхимический котёл)
- Строительство (в будущем)

**В текущей итерации (Harvest System)** — только инструменты добычи.

## 🏗️ Место в архитектуре экипировки

### Интеграция с «Матрёшкой»

Инструмент = EquipmentData с `category = ItemCategory.Misc` (или новый `Tool`)  
+ дополнительные поля через наследование или композицию.

**Два варианта интеграции:**

| Вариант | Описание | Плюсы | Минусы |
|---------|----------|-------|--------|
| **A: ToolData : EquipmentData** | Новый SO-класс, наследник EquipmentData | Чистая архитектура, типобезопасность | +1 файл, +1 CreateAssetMenu |
| **B: Поля в EquipmentData** | Добавить tool-поля в существующий EquipmentData | Нет новых файлов, простота | Загрязнение класса, поля не нужны для оружия/брони |

**Рекомендация:** Вариант A (ToolData : EquipmentData).  
Инструменты имеют уникальную структуру (effectiveCategories, damageMultiplier, yieldMultiplier), которая не нужна оружию и броне.

### Слот экипировки

Инструмент экипируется в слот `WeaponMain` — занимает руку.  
Нельзя одновременно держать оружие и инструмент.

**Альтернатива (будущее):** Добавить слот `ToolBelt` для инструмента,  
не блокирующий оружие. Решение — после проработки полной концепции экипировки.

## 🔨 Типы инструментов добычи

### 1. Базовые типы

| Тип | ID | Назначение | Эффективен против |
|-----|----|-----------|-------------------|
| **Кулаки** | `hands` | Добыча без инструмента | Всё (минимальная эффективность) |
| **Топор** | `axe` | Рубка деревьев | Деревья (Tree) |
| **Кирка** | `pickaxe` | Добыча камня и руды | Камень (Rock), Руда (OreVein) |
| **Серп** | `sickle` | Сбор растений | Кусты (Bush), Травы (Herb) |

### 2. Категории объектов добычи

| Категория | ID | Объекты |
|-----------|----|---------|
| **Wood** | `wood` | Tree_Oak, Tree_Pine, Tree_Birch |
| **Stone** | `stone` | Rock_Small, Rock_Medium |
| **Ore** | `ore` | OreVein |
| **Plant** | `plant` | Bush, Bush_Berry, Herb |

## 📊 Параметры инструмента

### Структура ToolData

```
ToolData : EquipmentData
├── toolType: ToolType              // axe, pickaxe, sickle, hands
├── effectiveCategories: string[]   // Категории объектов, против которых эффективен
├── damageMultiplier: float         // Множитель урона по durability (1.0-3.0)
├── yieldMultiplier: float          // Множитель количества лута (1.0-3.0)
├── harvestSpeedMultiplier: float   // Множитель скорости добычи (0.5-2.0)
└── durabilityCostPerHit: int       // Износ инструмента за удар (0-5)
```

### Формулы добычи с инструментом

```
// Урон по durability объекта
actualDamage = baseHarvestDamage × tool.damageMultiplier

// Количество ресурса за тик
actualYield = Floor(baseYieldPerTick × tool.yieldMultiplier)
actualYield = Min(actualYield, totalResource - harvestedSoFar)

// Скорость добычи (длительность анимации)
actualDuration = baseHarvestDuration / tool.harvestSpeedMultiplier
```

### Множители по типам

| Инструмент | damageMultiplier | yieldMultiplier | speedMultiplier | effectiveCategories |
|------------|-----------------|-----------------|-----------------|---------------------|
| **Кулаки** | 1.0 | 1.0 | 1.0 | [] (всё) |
| **Каменный топор** | 1.5 | 1.0 | 1.0 | [wood] |
| **Железный топор** | 2.0 | 1.5 | 1.0 | [wood] |
| **Духовный топор** | 3.0 | 2.0 | 1.2 | [wood] |
| **Каменная кирка** | 1.5 | 1.0 | 1.0 | [stone, ore] |
| **Железная кирка** | 2.0 | 1.5 | 1.0 | [stone, ore] |
| **Духовная кирка** | 3.0 | 2.0 | 1.2 | [stone, ore] |
| **Каменный серп** | 1.5 | 1.0 | 1.0 | [plant] |
| **Железный серп** | 2.0 | 1.5 | 1.0 | [plant] |
| **Духовный серп** | 3.0 | 2.0 | 1.2 | [plant] |

### Правило неэффективного инструмента

Если инструмент **НЕ** эффективен против категории объекта:

```
damageMultiplier = 0.5    // Половина урона
yieldMultiplier = 0.5     // Половина лута
```

Топор по камню = 0.5× урон, 0.5× лут.  
Кирка по дереву = 0.5× урон, 0.5× лут.

## 🏗️ Интеграция с Harvestable

### Поток добычи с инструментом

```
Игрок нажимает F
  │
  ├─ PlayerController.AttemptHarvest()
  │   ├─ Поиск ближайшего Harvestable
  │   ├─ Получение текущего инструмента:
  │   │   ToolData tool = equipmentController.GetEquippedTool()
  │   │   if (tool == null) tool = ToolData.HandsFallback()
  │   │
  │   ├─ Проверка эффективности:
  │   │   bool effective = tool.IsEffectiveFor(harvestable.category)
  │   │   float dmgMult = effective ? tool.damageMultiplier : 0.5f
  │   │   float yieldMult = effective ? tool.yieldMultiplier : 0.5f
  │   │
  │   ├─ Расчёт:
  │   │   int damage = RoundToInt(baseHarvestDamage * dmgMult)
  │   │   int yield = FloorToInt(baseYieldPerTick * yieldMult)
  │   │   float duration = baseHarvestDuration / tool.harvestSpeedMultiplier
  │   │
  │   ├─ Вызов: harvestable.HarvestHit(damage, yield)
  │   │
  │   ├─ Износ инструмента:
  │   │   if (tool.durabilityCostPerHit > 0)
  │   │       equipmentController.ApplyWear(WeaponMain, tool.durabilityCostPerHit)
  │   │
  │   └─ Опыт Strength +0.5 (без инструмента) / +1.0 (с инструментом)
```

### Fallback: Кулаки (без инструмента)

Когда в слоте `WeaponMain` нет инструмента — используется виртуальный `HandsFallback`:

```csharp
public static ToolData HandsFallback()
{
    // Программный объект, не SO — создаётся на лету
    return new ToolData
    {
        toolType = ToolType.Hands,
        damageMultiplier = 1.0f,
        yieldMultiplier = 1.0f,
        harvestSpeedMultiplier = 1.0f,
        effectiveCategories = new string[0], // Эффективен против всего
        durabilityCostPerHit = 0             // Кулаки не изнашиваются
    };
}
```

## 📊 Тиры инструментов (Материал + Грейд)

### Зависимость от материала

Инструмент следует архитектуре «Матрёшка»:

| Тир | Материал | damageMult | yieldMult | speedMult | Прочность | Требуемый уровень |
|-----|----------|-----------|-----------|-----------|-----------|-------------------|
| **T1** | Камень + Дерево | 1.5 | 1.0 | 1.0 | 50 | 0 (Смертный) |
| **T2** | Железо + Дерево | 2.0 | 1.5 | 1.0 | 150 | 1 (Пробуждённое Ядро) |
| **T3** | Сталь + Кожа | 2.5 | 1.8 | 1.1 | 300 | 3 (Пламя Внутреннего) |
| **T4** | Духовное железо | 3.0 | 2.0 | 1.2 | 500 | 5 (Сердце Небес) |
| **T5** | Звёздный металл | 4.0 | 3.0 | 1.5 | 800 | 7 (Вечное Кольцо) |

### Влияние грейда

Грейд влияет на множители инструмента:

| Грейд | damageMult | yieldMult | speedMult |
|-------|-----------|-----------|-----------|
| **Damaged** | ×0.5 | ×0.5 | ×0.8 |
| **Common** | ×1.0 | ×1.0 | ×1.0 |
| **Refined** | ×1.3 | ×1.3 | ×1.1 |
| **Perfect** | ×1.7 | ×1.7 | ×1.2 |
| **Transcendent** | ×2.5 | ×2.5 | ×1.5 |

**Итоговая формула:**
```
effectiveDamageMult = baseDamageMult × gradeMult
effectiveYieldMult = baseYieldMult × gradeMult
effectiveSpeedMult = baseSpeedMult × gradeSpeedMult
```

## 🛠️ Износ инструмента

### Механика

- Каждый удар по harvestable объекту — `durabilityCostPerHit` единиц износа
- Стоимость износа зависит от материала объекта:

| Категория объекта | durabilityCostPerHit |
|-------------------|---------------------|
| Wood (дерево) | 1 |
| Stone (камень) | 2 |
| Ore (руда) | 3 |
| Plant (растение) | 0 |

- Инструмент с прочностью 0 = сломан (Damaged грейд, ×0.5 эффективность)
- Ремонт — через NPC-кузнеца или навык ремонта

## 🔗 Связь с текущей реализацией

### Что меняется в Harvestable (чекпоинт v3)

В спецификации Harvestable (§4.1) уже есть:
- `float toolMultiplier = 1.0f` → **заменяется** на получение множителя из инструмента

**Было (чекпоинт v3):**
```csharp
int baseYieldPerTick        // Базовое количество ресурса за тик
float toolMultiplier = 1.0f // Коэффициент инструмента (пока = 1.0)
```

**Станет:**
```csharp
int baseYieldPerTick        // Базовое количество ресурса за тик
// toolMultiplier больше НЕ поле Harvestable
// Множитель приходит из PlayerController.AttemptHarvest() через параметр
```

### Что меняется в PlayerController.AttemptHarvest()

**Добавляется:**
- Получение ToolData из EquipmentController
- Расчёт множителей эффективности
- Износ инструмента
