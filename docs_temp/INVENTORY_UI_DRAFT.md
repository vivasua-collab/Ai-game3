## 1. Цель

Создать полнофункциональную систему инвентаря для Cultivation World Simulator:
- Панель инвентаря, открываемая по клавише
- Система рюкзака (Diablo-style сетка, размер зависит от рюкзака)
- Кукла тела с упрощёнными слотами экипировки
- Drag & drop перемещение предметов
- Пояс быстрого доступа
- Духовное хранилище (межмировая складка) + кольца хранения
- Каталогизатор для не-сеточных хранилищ

## 3. Архитектура инвентаря

### 3.1 Главный экран инвентаря

Открывается по клавише **I**. Состоит из зон:

```
┌─────────────────────────────────────────────────────────────────┐
│                        ИНВЕНТАРЬ                                │
│                                                                  │
│  ┌──────────┐  ┌──────────────────────────────────────────────┐ │
│  │  КУКЛА   │  │          РЮКЗАК                              │ │
│  │  ТЕЛА    │  │                                              │ │
│  │          │  │   3 × 4 слота (стартовый рюкзак)            │ │
│  │ [Голова] │  │                                              │ │
│  │ [Торс  ] │  │   Предметы 1×1, 2×1, 1×2, 2×2             │ │
│  │ [Пояс  ] │  │                                              │ │
│  │ [Ноги  ] │  │   Drag & Drop                                │ │
│  │ [Ступни] │  │                                              │ │
│  │          │  │   Вес: 12.5 / 30 кг  (−20% от рюкзака)     │ │
│  │ [Рука 1] │  │                                              │ │
│  │ [Рука 2] │  └──────────────────────────────────────────────┘ │
│  │          │                                                    │
│  │          │  ┌──────────────────────────────────────────────┐ │
│  └──────────┘  │  ПОЯС:  [1] [2] [3] [4]                     │ │
│                └──────────────────────────────────────────────┘ │
│                                                                  │
│  ┌────────────────────────────────────────────────────────────┐ │
│  │ [Рюкзак ▼]  │  [Дух. хранилище]  │  [Кольцо хранения]    │ │
│  └────────────────────────────────────────────────────────────┘ │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 3.2 Кукла тела (Body Doll) — УПРОЩЁННАЯ

Для начального этапа — только видимые слоты. Остальные скрыты (заглушки на будущее).

**Видимые слоты:**

| # | Слот | ID | Описание | Что надевается |
|---|------|----|----------|----------------|
| 1 | Голова | `head` | Шлем, шапка, корона | Любой головной убор |
| 2 | Торс | `torso` | Нагрудник, рубашка, роба | Броня или одежда на торс |
| 3 | Пояс | `belt` | Пояс, ремень | Пояс быстрого доступа или зарядник-пояс |
| 4 | Ноги | `legs` | Поножи, штаны | Броня или одежда на ноги |
| 5 | Ступни | `feet` | Сабатоны, сапоги | Обувь |
| 6 | Правая рука | `weapon_main` | Основное оружие | Одноручное или щит |
| 7 | Левая рука | `weapon_off` | Вторичное оружие | Одноручное, щит или инструмент |

**Логика оружия:**
- **Одноручное** — занимает один слот (main или off)
- **Двуручное** — занимает ОБА слота (main + off), второй слот блокируется
- Двуручное нельзя надеть, если off-слот занят, и наоборот

**Скрытые слоты (заглушки, не отображаются):**

| Слот | ID | Почему скрыт |
|------|----|-------------|
| Амулет | `amulet` | Будет добавлен с ювелирной системой |
| Кольцо левое 1 | `ring_left_1` | Будет с системой хранения (см. §3.6) |
| Кольцо левое 2 | `ring_left_2` | Будет с системой хранения |
| Кольцо правое 1 | `ring_right_1` | Будет с системой хранения |
| Кольцо правое 2 | `ring_right_2` | Будет с системой хранения |
| Зарядник | `charger` | Будет с системой зарядников |
| Перчатки | `hands` | Будет с расширением экипировки |
| Плащ/спина | `back` | Будет с расширением экипировки |

**Правила для начального этапа:**
- Каждый видимый слот = 1 предмет
- Armor и Clothing НЕ разделены (пока один слот на зону)
- Слот пояса — особый: пояс быстрого доступа или зарядник-пояс (см. §3.5)

### 3.3 Система рюкзака

Размер сетки инвентаря определяется **надетым рюкзаком**. Разные рюкзаки дают разное количество слотов и бонус на уменьшение веса.

**Рюкзак — это предмет**, экипируемый... нет, рюкзак НЕ надевается на куклу. Рюкзак — это отдельная система, привязанная к персонажу. Игрок может сменить рюкзак, но не может носить два одновременно.

**Модель рюкзака:**

| Параметр | Описание |
|----------|----------|
| id | ID рюкзака |
| name | Название |
| gridWidth | Ширина сетки (слотов) |
| gridHeight | Высота сетки (слотов) |
| weightReduction | Снижение веса (%) |
| maxWeightBonus | Бонус к максимальному весу (кг) |
| slots | Дополнительные слоты пояса (0-4) |

**Стартовый рюкзак:**

| Параметр | Значение |
|----------|----------|
| name | Тканевая сумка |
| gridWidth | 3 |
| gridHeight | 4 |
| weightReduction | 0% |
| maxWeightBonus | 0 |
| beltSlots | 0 |

**Примеры рюкзаков (черновики на будущее):**

| Рюкзак | Сетка | Снижение веса | Бонус веса | Слоты пояса |
|--------|-------|---------------|-----------|-------------|
| Тканевая сумка | 3×4 (12) | 0% | +0 | 0 |
| Кожаный ранец | 4×5 (20) | 10% | +10 | 1 |
| Железный контейнер | 5×5 (25) | 15% | +20 | 2 |
| Духовный мешок | 6×6 (36) | 25% | +30 | 2 |
| Межпространственный сундук | 8×7 (56) | 40% | +50 | 4 |

**Механика:**
- При смене рюкзака: если старый рюкзак не пуст — предметы нельзя вытащить, пока рюкзак снят
- Предметы в рюкзаке сохраняются при его снятии (рюкзак = контейнер)
- Вес рюкзака = собственный вес + вес содержимого × (1 − weightReduction)

### 3.4 Основная сетка (содержимое рюкзака)

- **Размер:** определяется рюкзаком (стартовый: 3×4)
- **Размеры предметов:** 1×1, 2×1, 1×2, 2×2
- **Стак:** стекируемые предметы объединяются (maxStack)
- **Drag & Drop:** перетаскивание между слотами, на куклу, в пояс
- **Вес:** отображается внизу сетки, с учётом снижения от рюкзака

### 3.5 Пояс

Пояс — особый слот экипировки на кукле. Тип пояса определяет его функцию:

**Типы поясов (черновики):**

| Тип | Описание | Слоты |
|-----|----------|-------|
| Обычный ремень | Без функций | 0 |
| Пояс зелий | Быстрый доступ к расходникам | 2-4 |
| Пояс-зарядник | Зарядник Ци в форме пояса | 0 (дает зарядник) |
| Комбинированный | И слоты и зарядник | 2 + зарядник |

**Для начального этапа:**
- Пояс не реализуется как отдельная система
- Слоты быстрого доступа = 0 (нет пояса → нет слотов)
- Когда игрок найдёт/создаст пояс зелий — появятся слоты
- Горячие клавиши: 1..4 (только если слоты есть)

### 3.6 Духовное хранилище + Кольца хранения

**ДВА параллельных механизма** хранения с разной логикой.

---

#### 3.6.1 Духовное хранилище (Межмировая складка)

По сути — «мусорка», куда складывается всё собранное, отодранное и сейчас ненужное. По логике мира — находится в межмировой складке.

**Характеристики:**

| Параметр | Значение |
|----------|----------|
| Размер | Без ограничений |
| Сетка | НЕТ — каталогизатор |
| Доступ | Из любой локации |
| Разблокировка | Уровень культивации 3 (AwakenedCore) |

**Стоимость использования:**

| Ресурс | Формула | Примечание |
|--------|---------|------------|
| Энергия (Ци) | `baseCost + weight × costPerKg` | Зависит от веса объекта |
| Время | Константа (например, 2 сек) | Время на открытие «отверстия» в складку |

- От качества предмета стоимость НЕ зависит
- На ранних этапах ограничивает использование (мало Ци)
- На поздних этапах — уже не существенно

**Каталогизатор:**

Вместо сетки — список с фильтрацией и сортировкой:

```
┌─────────────────────────────────────────────┐
│  ДУХОВНОЕ ХРАНИЛИЩЕ (Межмировая складка)     │
│                                              │
│  [Фильтр: Все ▼]  [Сортировка: Тип ▼]       │
│  [Поиск: _______________]                    │
│                                              │
│  ┌─ Оружие (3) ─────────────────────────┐   │
│  │  Меч Рассвета          x1   2.5 кг  │   │
│  │  Кинжал Тени           x2   0.8 кг  │   │
│  │  Посох Ветра           x1   3.0 кг  │   │
│  ├─ Материалы (12) ─────────────────────┤   │
│  │  Железная руда         x45  22.5 кг │   │
│  │  Древесина             x30  15.0 кг │   │
│  ├─ Расходники (5) ────────────────────┤   │
│  │  Пилюля Здоровья       x8   0.4 кг  │   │
│  └──────────────────────────────────────┘   │
│                                              │
│  Всего: 20 предметов  │  Вес: 44.2 кг       │
│  Извлечение: 15 Ци + 2 сек                   │
│                                              │
│  [Извлечь]  [Извлечь все этого типа]         │
└─────────────────────────────────────────────┘
```

**Фильтры каталогизатора:**
- По типу (оружие, броня, материалы, расходники...)
- По редкости
- По весу (лёгкие / средние / тяжёлые)
- По текстовому поиску

#### 3.6.2 Кольца хранения

Альтернативный/дополнительный механизм. Кольцо хранения — надевается на слот кольца на кукле. Объём ОГРАНИЧЕН.

**Характеристики:**

| Параметр | Значение |
|----------|----------|
| Слоты на кукле | 1-2 (ring_left, ring_right) |
| Объём | Ограничен (зависит от качества кольца) |
| Доступ | Из любой локации (кольцо при себе) |
| Разблокировка | Наличие кольца хранения в инвентаре |

**Стоимость использования:**

| Ресурс | Формула | Примечание |
|--------|---------|------------|
| Энергия (Ци) | `baseCost + volume × costPerUnit` | Зависит от ОБЪЁМА объекта |
| Время | Константа (например, 1.5 сек) | Быстрее чем духовное хранилище |

**Объём предмета** — новая характеристика. Каждый предмет имеет объём (в условных единицах), который определяет, сколько места он занимает в кольце хранения.

| Пример предмета | Объём |
|----------------|-------|
| Кольцо | 0.1 |
| Пилюля | 0.1 |
| Кинжал | 1 |
| Меч | 2 |
| Нагрудник | 3 |
| Двуручный меч | 4 |

**Примеры колец хранения (черновики):**

| Кольцо | Объём | Энергия на извлечение |
|--------|-------|----------------------|
| Кольцо-щель | 5 | 5 + volume × 3 |
| Кольцо-карман | 15 | 5 + volume × 2 |
| Кольцо-кладовая | 30 | 5 + volume × 1 |
| Кольцо-пространство | 60 | 5 + volume × 0.5 |

**Отличие от духовного хранилища:**
- Объём ограничен vs без ограничений
- Стоимость зависит от объёма vs от веса
- Быстрее доступ (1.5 сек vs 2 сек)
- Можно сменить кольцо на другое с другим объёмом
- Нужен слот на кукле → конкурирует с обычными кольцами

#### 3.6.3 Взаимодействие двух систем (Вариант 3)

Обе системы существуют одновременно. Правила вложения:

**Флаг «Вложение» (allowNesting):**

Каждый предмет имеет флаг, разрешающий или запрещающий перемещение в хранилище:

| Значение | Описание |
|----------|----------|
| `none` | Нельзя поместить ни в какое хранилище |
| `spirit` | Можно ТОЛЬКО в духовное хранилище |
| `ring` | Можно ТОЛЬКО в кольцо хранения |
| `any` | Можно в любое хранилище |

**Правила вложения:**

1. **Кольцо хранения → Духовное хранилище:** ЗАПРЕЩЕНО. Кольцо хранения — самостоятельный артефакт, его содержимое не может быть «вложено» в складку. Флаг `allowNesting = none` для колец хранения.

2. **Кольцо хранения → Другое кольцо хранения:** ЗАПРЕЩЕНО. Пространственная нестабильность. Флаг `allowNesting = none`.

3. **Духовное хранилище → Кольцо хранения:** ЗАПРЕЩЕНО. Нельзя извлечь из складки прямо в кольцо — только в рюкзак или руки.

4. **Обычный предмет → Любое хранилище:** Определяется флагом `allowNesting`. По умолчанию `any`.

5. **Живые существа / квестовые предметы:** `allowNesting = none`. Нельзя поместить никуда.

**Цепочка перемещения:**
```
Рюкзак ↔ Кукла ↔ Руки
    ↕              ↕
Дух. хранилище    Кольцо хранения
    
Дух. хранилище ✗→ Кольцо хранения  (запрещено)
Кольцо хранения ✗→ Дух. хранилище  (запрещено)
Кольцо хранения ✗→ Кольцо хранения  (запрещено)
```

## 4. Новые характеристики предметов

Для поддержки системы хранения каждому предмету нужны дополнительные поля:

| Поле | Тип | Описание |
|------|-----|----------|
| `volume` | float | Объём предмета (для кольца хранения) |
| `allowNesting` | enum | `none`, `spirit`, `ring`, `any` — куда можно поместить |

**Значения объёма по умолчанию:**

| Категория | Объём | Примечание |
|-----------|-------|------------|
| Расходники (пилюли, свитки) | 0.1 | Очень маленькие |
| Кольца, амулеты | 0.1-0.3 | Маленькие |
| Еда, травы | 0.5 | Средние |
| Ингредиенты, материалы | 0.5-1 | Зависит от типа |
| Лёгкое оружие (кинжалы) | 1 | Одноручное малое |
| Среднее оружие (мечи) | 2 | Одноручное |
| Броня (перчатки, сапоги) | 1-2 | Средняя |
| Тяжёлое оружие (двуручное) | 3-4 | Большое |
| Тяжёлая броня (нагрудник) | 3-4 | Большая |
| Рюкзак | 2-5 | Зависит от размера |
| Кольцо хранения | 0.3 | Само кольцо (как предмет) |

## 5. Взаимодействие

### 5.1 Открытие/закрытие

- **Клавиша I** — открыть/закрыть инвентарь
- **Клавиша E** — контекстное действие (подобрать предмет, открыть сундук)
- При открытии — игра переходит в состояние `GameState.Inventory`
- Курсор разблокируется (Cursor.lockState = None)

### 5.2 Drag & Drop

- Захват предмета: клик левой кнопкой
- Перемещение: удержание + движение мыши
- Отпускание: если валидная позиция — предмет перемещается
- Если целевой слот занят — предметы меняются местами (swap)
- Если предмет бросить мимо панели — он выпадает на землю
- Между рюкзаком и куклой — прямое перетаскивание
- В духовное хранилище / кольцо — через Shift+клик или контекстное меню

### 5.3 Контекстное меню (правый клик)

| Действие | Условие |
|----------|---------|
| Экипировать | Предмет экипируемый, не надет |
| Снять | Предмет надет на персонаже |
| Использовать | Предмет расходуемый |
| В духовное хранилище | `allowNesting = spirit` или `any`, достаточно Ци |
| В кольцо хранения | `allowNesting = ring` или `any`, есть место, достаточно Ци |
| Разделить стек | Стек > 1 |
| Выбросить | Любой предмет |
| Подробнее | Открыть tooltip |

## 7. Связь с бэкендом

### 7.1 InventoryController (существующий, требует доработки)

**Требуемые изменения:**
- `gridWidth` / `gridHeight` → определяются рюкзаком, не фиксированные
- Добавить `SetBackpack(BackpackData)` — сменить рюкзак
- Добавить расчёт `effectiveWeight = totalWeight × (1 − backpack.weightReduction)`

**Существующие вызовы (без изменений):**
- `AddItem()`, `RemoveItem()`, `MoveItem()`, `SwapSlots()`
- `FindSlotById()`, `HasItem()`, `CountItem()`
- `GetSaveData()`, `LoadSaveData()`
- События: `OnItemAdded`, `OnItemRemoved`, `OnItemStackChanged`, `OnWeightChanged`, `OnInventoryFull`

### 7.2 EquipmentController (существующий, требует доработки)

**Требуемые изменения:**
- Упростить слоты до 7 видимых (head, torso, belt, legs, feet, weapon_main, weapon_off)
- Скрытые слоты — заглушки, не функциональны
- Двуручное оружие: при equip → блокирует weapon_off
- Слот `belt` — особая логика (пояс быстрого доступа / зарядник-пояс)

**Существующие вызовы (без изменений):**
- `EquipItem()`, `UnequipItem()`, `GetEquippedItem()`, `IsSlotOccupied()`

### 7.3 SpiritStorageController (НОВЫЙ)

- `StoreItem(itemId, count)` — переместить в духовное хранилище (списывает Ци)
- `RetrieveItem(entryId, count)` — извлечь (списывает Ци)
- `GetContents()` — получить каталог
- `FilterByCategory()`, `FilterByRarity()`, `Search()`
- `GetStorageCost(weight)` — рассчитать стоимость помещения
- `GetRetrievalCost(weight)` — рассчитать стоимость извлечения

### 7.4 StorageRingController (НОВЫЙ)

- `StoreItem(itemId, count)` — в кольцо (проверяет объём + Ци)
- `RetrieveItem(entryId, count)` — из кольца (списывает Ци)
- `GetContents()` — содержимое кольца
- `GetCurrentVolume()` / `GetMaxVolume()` — текущий/максимальный объём
- `GetStorageCost(volume)` — стоимость помещения
- `CanStore(itemId)` — проверка по флагу `allowNesting` и объёму

### 7.5 Цикл обновления

1. Игрок нажимает I → UI открывает панель
2. UI читает `InventoryController.Slots` → создаёт визуальные слоты рюкзака
3. UI читает `EquipmentController` → заполняет куклу
4. UI проверяет наличие духовного хранилища / кольца → показывает вкладки
5. Игрок перетаскивает предмет → UI вызывает `MoveItem()` или `EquipItem()`
6. Игрок Shift+клик → UI предлагает: духовное хранилище или кольцо (если оба доступны)
7. Событие контроллера → UI обновляет визуал
