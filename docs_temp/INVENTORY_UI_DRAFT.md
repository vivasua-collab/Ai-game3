## 1. Цель

Создать полнофункциональную систему инвентаря для Cultivation World Simulator:
- Диалог/панель инвентаря, открываемая по клавише
- Diablo-style сетка с предметами разного размера
- Drag & drop перемещение предметов
- Экипировка предметов на персонажа
- Пояс быстрого доступа
- Духовное хранилище

### 3.2 Кукла тела (Body Doll)

Визуальное отображение персонажа с слотами экипировки.

**Слоты экипировки (гуманоид):**

| Зона | Armor Slot | Clothing Slot | Описание |
|------|------------|---------------|----------|
| Голова | head_armor | head_clothing | Шлем + шапка |
| Торс | torso_armor | torso_clothing | Нагрудник + рубашка |
| Ноги | legs_armor | legs_clothing | Поножи + штаны |
| Ступни | feet_armor | feet_clothing | Сабатоны + сапоги |
| Кисти | hands_armor | hands_clothing | Перчатки (боевые + обычные) |

**Оружие:**

| Слот | Описание | Конфликт |
|------|----------|----------|
| weapon_main | Основная рука | Блокируется twohanded |
| weapon_off | Вторичная рука | Блокируется twohanded |
| weapon_twohanded | Двуручное | Блокирует main+off |

**Аксессуары:**

| Слот | Лимит |
|------|-------|
| amulet | 1 |
| ring_left_1 | 1 |
| ring_left_2 | 1 |
| ring_right_1 | 1 |
| ring_right_2 | 1 |
| charger | 1 |

**Правила:**
- Armor + Clothing = раздельные слоты, не конфликтуют
- Максимум 4 кольца (по 2 на руку)
- 1 зарядник Ци (форм-фактор не влияет на слот)
- Двуручное оружие занимает оба слота рук

### 3.3 Основная сетка

- **Размер:** 8×6 (48 слотов), конфигурируемый через InventoryController
- **Размеры предметов:** 1×1, 2×1, 1×2, 2×2
- **Стак:** стекируемые предметы объединяются (maxStack)
- **Drag & Drop:** перетаскивание между слотами, на куклу, в пояс

### 3.4 Пояс быстрого доступа

- 4 слота внизу экрана (видны всегда, без открытия инвентаря)
- Только расходники: таблетки, эликсиры, еда, свитки
- Горячие клавиши: CTRL+1..CTRL+4 (или просто 1..4)
- При использовании — предмет удаляется

### 3.5 Духовное хранилище

- Вне-пространственное хранилище, доступное из любой локации
- Разблокируется на уровне культивации 3 (AwakenedCore)
- Вместимость: 20 слотов (расширяется)
- Отдельная вкладка/кнопка в интерфейсе инвентаря

## 4. Взаимодействие

### 4.1 Открытие/закрытие

- **Клавиша I** — открыть/закрыть инвентарь
- **Клавиша Tab** — альтернативное открытие
- **Клавиша E** — контекстное действие (подобрать предмет, открыть сундук)
- При открытии — игра переходит в состояние `GameState.Inventory`
- Курсор разблокируется (Cursor.lockState = None)

### 4.2 Drag & Drop

- Захват предмета: клик левой кнопкой
- Перемещение: удержание + движение мыши
- Отпускание: если валидная позиция — предмет перемещается
- Если целевой слот занят — предметы меняются местами (swap)
- Если предмет бросить мимо панели — он выпадает на землю

### 4.3 Контекстное меню (правый клик)

| Действие | Условие |
|----------|---------|
| Экипировать | Предмет экипируемый, не надет |
| Снять | Предмет надет на персонаже |
| Использовать | Предмет расходуемый |
| Разделить стек | Стек > 1 |
| Выбросить | Любой предмет |
| Подробнее | Открыть tooltip |

### 4.4 Tooltip

При наведении мыши на предмет показывается карточка:

## 6. Связь с бэкендом

### 6.1 InventoryController (уже существует)

Вызовы из UI:
- `AddItem()` — добавить предмет
- `RemoveItem()` / `RemoveItemById()` — удалить
- `MoveItem()` — переместить в сетке
- `SwapSlots()` — поменять местами
- `FindSlotById()` — найти слот
- `HasItem()` / `CountItem()` — проверить наличие
- `GetSaveData()` / `LoadSaveData()` — сохранение

События для UI:
- `OnItemAdded` — обновить визуал
- `OnItemRemoved` — обновить визуал
- `OnItemStackChanged` — обновить счётчик
- `OnWeightChanged` — обновить панель веса
- `OnInventoryFull` — показать предупреждение

### 6.2 EquipmentController (уже существует)

- `EquipItem()` — экипировать
- `UnequipItem()` — снять
- `GetEquippedItem(slotId)` — получить надетый предмет
- `IsSlotOccupied(slotId)` — занят ли слот

### 6.3 Цикл обновления

1. Игрок нажимает I → UI открывает панель
2. UI читает `InventoryController.Slots` → создаёт визуальные слоты
3. UI читает `EquipmentController` → заполняет куклу
4. Игрок перетаскивает предмет → UI вызывает `MoveItem()` или `EquipItem()`
5. Событие контроллера → UI обновляет визуал

## 9. Порядок реализации

### Этап 1: Базовая панель
- InventoryCanvas + InventoryPanel
- Сетка 8×6 с пустыми слотами
- Открытие/закрытие по клавише I
- Чтение данных из InventoryController

### Этап 2: Отображение предметов
- Визуальные слоты с иконками и счётчиками
- Рамки по редкости
- Tooltip при наведении

### Этап 3: Drag & Drop
- Перетаскивание предметов между слотами
- Swap при помещении на занятый слот
- Подсветка валидных/невалидных позиций

### Этап 4: Кукла тела
- BodyDollPanel со слотами экипировки
- Экипировка/снятие через drag & drop
- Правый клик → экипировать/снять

### Этап 5: Пояс и дух. хранилище
- BeltPanel (4 слота, горячие клавиши)
- SpiritStoragePanel (вкладка, 20 слотов)

### Этап 6: Контекстное меню и доработка
- Правый клик → контекстное меню
- Разделение стека
- Выброс предметов
- Анимации
