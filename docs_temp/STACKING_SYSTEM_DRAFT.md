## 1. Принципы стекирования

### 1.1 Что может стекироваться

Предмет **можно объединить в стак** если выполняются **ВСЕ** условия:

| Условие | Проверка | Примечание |
|---------|----------|------------|
| `itemData.stackable == true` | Флаг предмета | Расходники, материалы — да; экипировка — нет |
| `slot.HasDurability == false` | Нет системы прочности | `durability < 0` (обычно -1) |
| Предметы **идентичны** | `itemId` совпадает | Один и тот же ItemData |
| Нет бонусов/модификаторов | `grade == Common` | Только базовые, не крафтовые |
| Прочность 100% | `durability == -1` ИЛИ `durability == maxDurability` | Не повреждённые |

### 1.2 Что НЕ может стекироваться

| Категория | Причина |
|-----------|---------|
| Экипировка (Weapon, Armor, Accessory) | `stackable = false` по умолчанию |
| Предметы с прочностью < 100% | Разная степень износа = разные предметы |
| Крафтовые предметы с грейдом | `grade != Common` — уникальные модификаторы |
| Квестовые предметы | `Quest` категория — каждый отслеживается отдельно |
| Органика с порчей | Будущая система: время существования → разные предметы |

### 1.3 Примеры

| Предмет | Стекируется? | Причина |
|---------|:---:|---------|
| Зелье лечения ×10 | ✅ | stackable=true, нет прочности, нет грейда |
| Железная руда ×50 | ✅ | stackable=true, нет прочности |
| Меч (Common, 100% пр.) | ❌ | `stackable=false` для экипировки |
| Меч (Common, 80% пр.) | ❌ | Экипировка + повреждён |
| Пилюля культивации ×5 | ✅ | stackable=true, нет прочности |
| Пилюля (Refined грейд) | ❌ | `grade != Common` — уникальный эффект |

## 2. Отображение в инвентаре

### 2.1 Строка списка

```
[Иконка] Зелье лечения  ×10  1.0кг  5.0л
```

- `×N` — количество в стаке (показывается только если N > 1)
- Вес = `itemData.weight × count`
- Объём = `itemData.volume × count`

### 2.2 Цветовая индикация

- Рамка строки — цвет редкости предмета
- Количество `×N` — белое для обычного стака, жёлтое если接近 maxStack

## 3. Управление переносом предметов

### 3.1 Режимы переноса

| Действие | Кнопка | Количество | Поведение |
|----------|--------|:----------:|-----------|
| Перенос одного | ЛКМ drag | 1 | Отщепляет 1 предмет от стака |
| Перенос всего стака | Shift + ЛКМ drag | Все | Перемещает весь стак целиком |
| Перенос половины | ПКМ drag | ⌊N/2⌋ | Половина стака, округление вниз |
| Конкретное количество | Shift + ПКМ | Диалог ввода | Ввод числа от 1 до N |

### 3.2 Подробная логика

#### ЛКМ — перенос одного предмета

```
Исходный стак: ×10
Действие: ЛКМ drag
Результат:
  - Из стака убирается 1 предмет
  - Создаётся новый стак ×1 на курсоре
  - Если исходный стак = ×1, переносится целиком
```

#### Shift + ЛКМ — перенос всего стака

```
Исходный стак: ×10
Действие: Shift + ЛКМ drag
Результат:
  - Весь стак ×10 перемещается на курсор
  - Исходная строка удаляется
```

#### ПКМ — перенос половины стака

```
Исходный стак: ×10
Действие: ПКМ drag
Результат:
  - 10 / 2 = 5 предметов на курсор
  - В исходном стаке остаётся ×5

Исходный стак: ×7
Действие: ПКМ drag
Результат:
  - ⌊7 / 2⌋ = 3 предмета на курсор
  - В исходном стаке остаётся ×4
```

#### Shift + ПКМ — ввод конкретного количества

```
Исходный стак: ×20
Действие: Shift + ПКМ drag
Результат:
  - Появляется диалог ввода: "Перенести (1-20):"
  - Пользователь вводит число, например 7
  - 7 предметов на курсор
  - В исходном стаке остаётся ×13
```

### 3.3 Контекстное меню (правый клик без drag)

| Пункт меню | Условие | Действие |
|------------|---------|----------|
| Использовать | Consumable | Использовать 1 шт. |
| Экипировать | Equipment | Надеть на куклу |
| Разделить | stackable && count > 1 | Разделить пополам |
| В кольцо хранения | Кольцо активно | Переместить весь стак |
| В дух. хранилище | Хранилище открыто | Переместить весь стак |
| Выбросить | Всегда | Выбросить весь стак |

## 5. Что нужно реализовать

### 5.1 InventorySlotUI — определение режима переноса

```
Необходимые поля:
  - dragMode: DragMode enum (One, All, Half, Custom)
  - dragAmount: int (для режима Custom)

Определение режима в OnBeginDrag:
  - ЛКМ без Shift → DragMode.One
  - ЛКМ + Shift → DragMode.All
  - ПКМ без Shift → DragMode.Half
  - ПКМ + Shift → DragMode.Custom (показать диалог)
```

### 5.2 DragDropHandler — обработка режимов

```
BeginDrag(slotUI, eventData):
  1. Определить dragMode из нажатых кнопок
  2. Вычислить dragAmount:
     - One: min(1, slot.Count)
     - All: slot.Count
     - Half: floor(slot.Count / 2)
     - Custom: показать диалог → ввести число
  3. Если dragAmount == slot.Count → перенос всего стака
  4. Если dragAmount < slot.Count → отщепить от стака
```

### 5.3 InventoryController — новые методы

```
// Отщепить указанное количество от стака, вернуть новый слот
SplitFromSlot(int slotId, int count) → InventorySlot

// Проверка: можно ли стекировать два предмета
CanStackWith(InventorySlot source, InventorySlot target) → bool

// Объединить два стака (если возможно)
MergeSlots(int sourceSlotId, int targetSlotId) → bool
```

### 5.4 Условия стекирования при drop

```
Drop на существующий стак:
  1. Проверить CanStackWith(source, target)
  2. Если да — добавить к целевому стаку (не превышая maxStack)
  3. Остаток — новый стак

Drop на пустое место:
  1. Создать новый стак с dragAmount предметов
```

## 6. Диалог ввода количества (Shift + ПКМ)

### Минимальная реализация

```
[Panel: "Перенести предметов"]
  "Зелье лечения (×20)"
  
  [−] [___7___] [+]
  Ползунок: 1 ────── 20
  
  [Отмена] [Перенести]
```

- Ползунок (Slider): min=1, max=slot.Count
- Поле ввода: синхронизировано с ползунком
- Кнопки ±: шаг 1, Shift±: шаг 5, Ctrl±: шаг 10

## 7. Совместимость с системой порчи (будущее)

Когда будет реализована система порчи органики:

| Изменение | Влияние на стекирование |
|-----------|------------------------|
| `ItemData.hasSpoilage` | Новый флаг |
| `InventorySlot.spoilageTimer` | Время до порчи |
| Предметы с разным таймером порчи | Разные стакы (как разная прочность) |
| Испорченный предмет | Новый itemId или флаг `isSpoiled` |

**Принцип:** Предметы с **разным оставшимся временем жизни** — разные стакы.
