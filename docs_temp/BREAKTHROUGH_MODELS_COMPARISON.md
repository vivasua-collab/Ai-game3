## 📋 Обзор

Документ сравнивает три модели системы прорыва и рассчитывает игровое время, необходимое для прохождения с L1 на L9 при разных методах восполнения Ци.

## 🔷 Модель А: Растущий резервуар

### Принцип
- **Ёмкость ядра растёт** с каждым уровнем
- **Требуемое количество Ци для прорыва** = coreCapacity × multiplier
- Плотность Ци — отдельный параметр качества (влияет на урон техник)

### Формула ёмкости
```
coreCapacity = 1000 × 1.1^totalSubLevels
```

### Требования для прорыва
- **Малый уровень:** coreCapacity × 10 Ци
- **Большой уровень:** coreCapacity × 100 Ци

---

## 🔶 Модель Б: Сжатие в постоянном объёме

### Принцип
- **Ёмкость ядра фиксирована** (например, 1000 единиц объёма)
- **Плотность Ци растёт** с уровнем: qiDensity = 2^(level-1)
- Прорыв требует заполнения ядра Ци текущего уровня плотности

### Формула "количества Ци" для прорыва
```
effectiveQiRequired = baseQi × qiDensity
```

### Интерпретация
- Ядро вмещает фиксированный объём, но Ци становится "гуще"
- Для накопления 1000 единиц Ци плотности 4 (L3) нужно поглотить в 4 раза больше "сырой Ци" из окружения

---

## 🟢 Модель В: Растущий резервуар + Сжатие Ци (ЛОР)

### Принцип
- **Ёмкость ядра растёт** с каждым уровнем (как в А)
- **Плотность Ци растёт** с уровнем (как в Б)
- **Комбинация:** Эффективное количество Ци = Объём × Плотность

### Формулы
```
coreCapacity = 1000 × 1.1^totalSubLevels  // Растущий объём (А)
qiDensity = 2^(level-1)                     // Сжатие Ци (Б)
effectiveQi = coreCapacity × qiDensity      // Эффективное Ци в ядре
conductivity = coreCapacity / 360           // Проводимость (ед/сек)
```

### Требования для прорыва
**Прорыв = расширение ядра + сжатие Ци до новой плотности**

```
Требуемая сырая Ци = coreCapacity(nextLevel) × qiDensity(nextLevel)
```

### Интерпретация
- Практик накапливает Ци в ядре текущей плотности
- При прорыве ядро расширяется И Ци сжимается до новой плотности
- После прорыва currentQi = 0 (нужно накапливать заново)

### Пример
- **L1.0:** Ёмкость 1000, Плотность 1 → Эффективное Ци: 1000×1 = 1000
- **L2.0:** Ёмкость 2594, Плотность 2 → Эффективное Ци: 2594×2 = 5188
- **L3.0:** Ёмкость 6727, Плотность 4 → Эффективное Ци: 6727×4 = 26,908

**Для прорыва L1→L2** нужно накопить сырой Ци = 2594 × 2 = 5,188
**Для прорыва L2→L3** нужно накопить сырой Ци = 6727 × 4 = 26,908

## 📈 Вариант 2: Медитация 8 часов/день

### Модель поглощения Ци из окружения (из ЛОРА)

**Правильно из лора (без environmentMult!):**
- `ρ_окр` — плотность Ци в воздухе (ед/м³) — это ЗАПАС, не множитель!
- Практик собирает ВСЮ Ци в зоне радиуса R
- Скорость поглощения = проводимость практика (ед/сек)
- Ци в зоне ИСЧЕЗАЕТ после поглощения
- Восстановление из недр — ОЧЕНЬ МЕДЛЕННОЕ

**Плотность Ци в зонах (из лора):**
| Зона | ρ_окр (ед/м³) |
|------|--------------|
| Внутреннее кольцо | 500-1000+ |
| Среднее кольцо | 200-500 |
| Внешнее кольцо | 100-200 |
| Приграничные | 20-100 |

### Условия расчёта
- Стартовая локация: внешнее кольцо, ρ_окр = 150 ед/м³ (среднее)
- Радиус зоны поглощения R = 10м (базовый)
- Объём зоны = 4/3 × π × R³ ≈ 4,189 м³
- Запас Ци в зоне = 150 × 4189 ≈ 628,350 ед
- Медитация: 8 часов/день = 28,800 сек
- После прорыва currentQi = 0

### Формула поглощения
```
qiFromMeditation = meditationTime × conductivity
Ограничение: qiFromMeditation ≤ qiAvailableInZone
```

**Важно:** Запас Ци в зоне (628,350) больше требуемого для большинства прорывов в Модели А и Б, но становится лимитирующим фактором для высоких уровней в Модели В.

## 🎯 Анализ и выводы

### Модель А: Растущий резервуар

**Плюсы:**
- Чувство прогресса (ёмкость растёт)
- Баланс с зарядниками — нужен апгрейд снаряжения
- Больше игрового времени = больше контента

**Минусы:**
- Лимит запаса Ци в зоне на L4+
- Зарядник становится узким местом на L5+
- Требует ~130 Кристаллов для быстрого прогресса

### Модель Б: Сжатие в постоянном объёме

**Плюсы:**
- Быстрый прогресс
- Проводимость практика фиксирована — нет проблем с зарядником
- Меньше расход камней Ци

**Минусы:**
- Нет ощущения роста ёмкости
- Может казаться "слишком легко"
- Не соответствует ЛОРУ

### Модель В: Растущий резервуар + Сжатие (ЛОР)

**Плюсы:**
- Соответствует ЛОРУ
- Прогрессия: каждый прорыв в 2× сложнее предыдущего
- Реалистичная экономика ресурсов

**Минусы:**
- Самое долгое время прогресса
- Требует много камней Ци на высоких уровнях
- Зарядник — узкое место на L5+
- Запас Ци в зоне — лимитирующий фактор

**Ключевой вывод:** Модель В создаёт реалистичную прогрессию культивации, где прорыв на каждый следующий уровень требует в 2 раза больше ресурсов.

## 🔧 Рекомендации для Модели В

### Для баланса с зарядниками

1. **Увеличить проводимость зарядников:**
   - Common: 10 ед/сек (вместо 5)
   - Refined: 25 ед/сек (вместо 10)
   - Perfect: 50 ед/сек (вместо 25)
   - Transcendent: 100 ед/сек (вместо 50)

2. **Или добавить перки на проводимость**

### Для баланса запаса Ци в зоне

1. **Радиус зоны поглощения растёт с уровнем:**
   ```
   R = 10 × (1 + 0.1 × level) метров
   L1: 11м, L5: 15м, L9: 19м
   ```

2. **Или "глубокое дыхание" — поглощение из недр:**
   - Позволяет собирать Ци ниже поверхности
   - Доступно на L4+
