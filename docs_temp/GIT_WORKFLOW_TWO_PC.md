## ⚠️ Важно

> Unity проект напрямую подключён к GitHub.

> Следуйте этим инструкциям для избежания конфликтов!

## 📁 Структура репозитория

```
Ai-game3/
├── UnityProject/           # ← Unity проект (подключён напрямую)
│   ├── Assets/             # ✅ В Git (вместе с .meta)
│   ├── ProjectSettings/    # ✅ В Git (настройки проекта)
│   ├── Packages/           # ✅ В Git (manifest.json)
│   ├── Library/            # ❌ НЕ в Git (кэш Unity)
│   └── ...
├── docs/                   # Документация
└── README.md
```

## 🔄 Рабочий процесс

### На ПК 1 (Основной)

```
1. Открыть Unity
2. Дождаться импорта (если были изменения)
3. Работать над проектом
4. Закрыть Unity (рекомендуется)
5. Git → Push изменений
```

### На ПК 2 (Второй)

```
1. Git → Pull изменений
2. Открыть Unity
3. Дождаться импорта
4. Работать над проектом
5. Закрыть Unity
6. Git → Push изменений
```

---

## 📋 Подробные инструкции

### Перед началом работы (на каждом ПК)

```bash
# 1. Получить последние изменения
git pull origin main

# 2. Открыть Unity и дождаться импорта
# Unity автоматически обработает изменения
```

### После завершения работы

```bash
# 1. Закрыть Unity (важно для избежания конфликтов .meta)

# 2. Проверить статус
git status

# 3. Добавить все изменения
git add -A

# 4. Сделать коммит
git commit -m "описание изменений"

# 5. Получить изменения (если были пуши с другого ПК)
git pull origin main --rebase

# 6. Запушить
git push origin main
```

---

## ⚠️ Правила для избежания конфликтов

### 1. Всегда Pull перед работой

```bash
git pull origin main
```

### 2. Закрывайте Unity перед Push

Unity блокирует файлы при работе, что может вызвать:
- Невозможность записи .meta файлов
- Конфликты при Push
- Повреждение файлов проекта

### 3. Не редактируйте одни файлы одновременно

Если оба ПК редактируют один файл → конфликт.

**Решение:**
- Разделите зоны ответственности
- Или работайте последовательно

### 4. Проверяйте .meta файлы

Если создаёте новый файл в Assets:
- Unity создаёт .meta файл
- Оба файла (файл + .meta) должны быть в Git

## 🔧 Работа из Unity Editor

### Вариант A: Встроенная поддержка Git (рекомендуется)

Unity 2021+ имеет встроенную поддержку VCS:

1. `Edit → Preferences → Version Control`
2. Включить `Visible Meta Files`
3. `Mode: Perforce` или `Plastic SCM`

## 🐛 Решение проблем

### Проблема: Конфликт .meta файлов

**Причина:** Оба ПК создали .meta для одного файла.

**Решение:**
```bash
# Удалить дубликаты .meta
git status
# Решить конфликт вручную
git add .
git commit -m "resolve meta conflicts"
```

### Проблема: Library повреждена

**Причина:** Push во время работы Unity.

**Решение:**
```bash
# Закрыть Unity
# Удалить Library
rm -rf Library/
# Открыть Unity (пересоздаст Library)
```

### Проблема: Сцена повреждена

**Причина:** Конфликт в .unity файле.

**Решение:**
- Использовать Git LFS для .unity файлов
- Или работать со сценами по очереди

### Проблема: Разные версии Unity

**Причина:** Разные версии Unity на ПК.

**Решение:**
- Установить ОДИНАКОВУЮ версию Unity на обоих ПК
- Проверить `ProjectSettings/ProjectVersion.txt`

## 📦 Git LFS (опционально)

Для больших бинарных файлов (аудио, текстуры):

```bash
# Установить Git LFS
git lfs install

# Настроить отслеживание
git lfs track "*.psd"
git lfs track "*.mp3"
git lfs track "*.wav"

# Добавить .gitattributes
git add .gitattributes
git commit -m "setup Git LFS"
```

## 🔗 Полезные команды

```bash
# История коммитов
git log --oneline -10

# Разница с remote
git diff origin/main

# Отменить локальные изменения (ОСТОРОЖНО!)
git checkout -- .

# Временное сохранение изменений
git stash
git stash pop

# Посмотреть статус файлов
git status -s
```

## 📞 Если что-то сломалось

1. **Не паникуйте!**
2. Проверьте `git status`
3. Если есть незакоммиченные изменения — закоммитьте
4. `git pull --rebase` для получения изменений
5. При конфликтах — решите вручную
6. Если совсем сломалось — клонируйте репозиторий заново
