## 🎯 Решение: Иерархия источников истины

### Принцип

> **Каждая сущность/параметр должен быть определён только в ОДНОМ документе.**
> **Все остальные документы должны только ссылаться на него.**

## ✅ План реализации

### Шаг 1: Архивация устаревших документов

| Файл | Действие |
|------|----------|
| WORLD_SYSTEM.md | Переименовать в `_ARCHIVED_WORLD_SYSTEM.md` или добавить заголовок "⚠️ УСТАРЕЛО — см. WORLD_MAP_SYSTEM.md" |

### Шаг 2: Исправление LOCATION_MAP_SYSTEM.md

Добавить в начало файла:

```markdown
## ⚠️ Важно

> **Размерность мира:** См. [WORLD_MAP_SYSTEM.md](./WORLD_MAP_SYSTEM.md) — §0.1, §0.2
> - **Сектор = 10×10 км** (не 320×320 м!)
> - **Тайл = 2×2 м**
> - **Размер локации = переменный (100 м ... 10 км)**
```

Удалить/исправить устаревшие размеры в документе.

### Шаг 3: Обновление шаблона документов

Создать стандартный заголовок для всех документов:

```markdown
**Версия:** X.X
**Дата:** YYYY-MM-DD
**Статус:** [Статус]
**Источники:** [Список документов-источников истины]

## ⚠️ Важно

> **Ссылки на источники истины:**
> - Размерность мира: [WORLD_MAP_SYSTEM.md](./WORLD_MAP_SYSTEM.md)
> - Формулы: [ALGORITHMS.md](./ALGORITHMS.md)
> - Сущности: [ENTITY_TYPES.md](./ENTITY_TYPES.md)
```

### Шаг 4: Проверка связанности

Убедиться, что каждый документ:
1. Имеет ссылки на источники истины
2. НЕ дублирует данные из источников
3. Содержит только свою специфику
