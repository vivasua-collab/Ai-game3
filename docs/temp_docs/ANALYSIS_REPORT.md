## 1️⃣ Система проводимости меридиан

### ✅ ПЕРЕНЕСЕНА И РЕАЛИЗОВАНА

**Старая документация (start_lore.md):**
```
проводимость = объём ядра / 360 сек
буфер меридиан = проводимость × 5 секунд
```

**Новая документация (QI_SYSTEM.md):**
```
conductivity = coreVolume / 360 секунд
bufferVolume = conductivity × 5 секунд
```

**Код (QiController.cs:94):**
```csharp
conductivity = maxQiCapacity / 360f;
```

**Использование в коде:**
- Медитация: `baseGain = conductivity * qiDensity * durationTicks;`
- Вывод в UI: `"Проводимость: {conductivity:F1}"`

**Вывод:** Система полностью перенесена и реализована.

## 4️⃣ FORMATION_SYSTEM.md

### ⚠️ ТРЕБУЕТ ДОРАБОТКИ

**Проблема:** Новая документация сильно упрощена и не соответствует старой.

#### Сравнение:

| Аспект | Старая (formation_unified.md) | Новая (FORMATION_SYSTEM.md) |
|--------|-------------------------------|------------------------------|
| **Ядра формаций** | Дисковые (L1-L6), Алтарные (L5-L9) | Не упомянуты |
| **Проводимость ядра** | 5-100 ед/сек | Не упомянута |
| **Ёмкость формации** | contourQi × sizeMultiplier | qiCost (упрощено) |
| **Наполнение** | Несколько практиков | Один практик |
| **Утечка Ци** | Дискретная, зависит от уровня | Не упомянута |
| **Типы формаций** | Barrier, Trap, Amplification, Gathering | Offensive, Defensive, Support, Control, Special |
| **Слоты усиления** | Камни Ци в ядро | Предметы в слоты |

#### Критические противоречия:

**1. Ядра формаций (ОТСУТСТВУЕТ):**

Старая документация:
```typescript
const DISK_CORE_TYPES = {
  stone_disk: { baseConductivity: 5, maxCapacity: 10000 },
  jade_disk: { baseConductivity: 10, maxCapacity: 50000 },
  iron_disk: { baseConductivity: 15, maxCapacity: 200000 },
  // ...
};

const ALTAR_CORE_TYPES = {
  jade_altar: { baseConductivity: 40, maxCapacity: 5000000 },
  // ...
  dragon_bone_altar: { baseConductivity: 100, maxCapacity: 200000000 },
};
```

Новая документация: **Отсутствует**

**2. Наполнение несколькими практиками (ОТСУТСТВУЕТ):**

Старая документация:
```typescript
// totalFillRate = Σ (conductivity × qiDensity) всех участников
function calculateFillRate(filling: FormationFilling): number {
  return filling.participants.reduce((sum, p) => 
    sum + (p.conductivity * p.qiDensity), 0
  );
}
```

Новая документация: **Тратит QiCost практика напрямую**

**3. Естественная утечка Ци (ОТСУТСТВУЕТ):**

Старая документация:
```typescript
// L1: 60 тиков = 1 час, 1 Ци/час
// L5: 20 тиков = 20 минут, 90 Ци/час
// L9: 5 тиков = 5 минут, 1200 Ци/час
```

Новая документация: **Только duration**

#### Рекомендации:

1. **Добавить систему ядер формаций:**
   - Дисковые ядра (переносные)
   - Алтарные ядра (стационарные)
   - Проводимость ядра

2. **Добавить систему наполнения:**
   - Участие нескольких практиков
   - Формула скорости наполнения
   - Требования к уровню помощников

3. **Добавить естественную утечку:**
   - Интервал утечки по уровню
   - Количество Ци за раз
   - Время жизни без подпитки

4. **Уточнить типы формаций:**
   - Barrier (поглощение урона)
   - Trap (активация при входе)
   - Amplification (баффы)
   - Gathering (сбор Ци)
