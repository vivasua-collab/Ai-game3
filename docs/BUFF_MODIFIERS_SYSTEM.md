### Что НЕЛЬЗЯ менять баффами:

|----------|---------|--------------|

### Что МОЖНО менять баффами:

| Параметр | Ограничения | Примеры |
|----------|-------------|---------|
| **Текущее Ци** | Только регенерация/расход | Восстановление Ци |
| **Проводимость** | Только временно, с ОТКАТОМ | См. ЧАСТЬ A §"Проводимость" |
| **Вторичные характеристики** | Через мягкие капы | Attack, Defense, Speed |
| **HP частей тела** | Регенерация, щиты | Лечение, поглощение урона |
| **Скорости** | Через мягкие капы | Скорость атаки, передвижения |

# ЧАСТЬ A: БАФФЫ И ДЕБАФФЫ

## Обзор баффов

Система баффов и дебаффов управляет **временными эффектами**, влияющими на характеристики персонажа. Баффы — положительные эффекты, дебаффы — отрицательные.

**Ключевой принцип:** Баффы не дают «бесплатных» постоянных улучшений — они временные и имеют свою цену.

## Особый случай: Проводимость (Conductivity)

### Правила изменения проводимости

> **Ключевой принцип:** Любой бонус к проводимости от **баффа** имеет ОТКАТ — понижение на большее время. Перк влияет на **базовую** проводимость — это развитие персонажа, НЕ модификатор.

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    СИСТЕМА ПРОВОДИМОСТИ                                      │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│   Проводимость (Conductivity)                                               │
│   ├── Формула при пробуждении: coreCapacity / 360 секунд                    │
│   ├── Влияет на: скорость поглощения Ци, скорость вывода для техник        │
│   └── Влияет на: ёмкость рабочего буфера меридиан                          │
│                                                                              │
│   ══════════════════════════════════════════════════════════════════════   │
│   ПРАВИЛО ОТКАТА (для баффов):                                              │
│   ══════════════════════════════════════════════════════════════════════   │
│                                                                              │
│   bonusDuration      = X секунд (длительность бонуса)                       │
│   penaltyDuration    = X × 3 секунд (длительность штрафа)                   │
│   penaltyValue       = bonusValue × 0.5 (величина штрафа)                   │
│                                                                              │
│   Пример:                                                                    │
│   ├── Бонус: +20% проводимости на 30 секунд                                │
│   ├── Откат: -10% проводимости на 90 секунд                                │
│   └── Итого: 30 сек усиления, затем 90 сек ослабления                      │
│                                                                              │
│   ══════════════════════════════════════════════════════════════════════   │
│   ПЕРКИ (без отката):                                                        │
│   ══════════════════════════════════════════════════════════════════════   │
│                                                                              │
│   Перк влияет на базовую проводимость меридиан:                             │
│   ├── «Эластичность меридиан» — innate свойство                             │
│   └── «Увеличенный диаметр» — acquired свойство                             │
│   Это развитие персонажа, НЕ модификатор. Баффы — временно, перки — навсегда│
│                                                                              │
│   ══════════════════════════════════════════════════════════════════════   │
│   ПРИЧИНА ОТКАТА: Меридианы "растягиваются" при усилении,                   │
│   затем должны "восстановиться" в сжатом состоянии.                         │
│   ══════════════════════════════════════════════════════════════════════   │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘
```

### Структура ConductivityModifier

> **Реализация:** [examples/BuffSystem_Examples.md](./examples/BuffSystem_Examples.md#1-conductivitymodifier)

Класс содержит:
- **Бонус:** `bonusValue`, `bonusRemainingTime`
- **Откат:** `penaltyValue` (= bonusValue × 0.5), `penaltyRemainingTime`
- **Состояние:** `isBonusActive`, `isPenaltyActive`
- **Метод:** `GetMultiplier()` — возвращает итоговый множитель проводимости

| Категория | Типы |

### BuffApplication (Enum)

| Значение | Описание |
|----------|----------|
| Instant | Мгновенный эффект (лечение, урон) |
| Duration | Длительный эффект |
| Permanent | Постоянный (пока не снят) |
| Stacking | Накапливающийся |
| Refreshing | Обновляет длительность |

### BuffStacking (Enum)

| Значение | Описание |
|----------|----------|
| Replace | Заменить |
| Refresh | Обновить таймер |
| Stack | Добавить стек |
| Ignore | Игнорировать |

### BuffData (ScriptableObject)

Ключевые поля:
- **Identity:** id, displayName, description, icon
- **Type:** type, isDebuff, element
- **Effect:** value, isPercentage
- **Duration:** application, duration, maxStacks, stackingBehavior
- **Conductivity Special:** hasConductivityPenalty, penaltyMultiplier (×3), penaltyValueMultiplier (×0.5)
- **Tick Effect:** hasTickEffect, tickInterval, tickDamage, tickHealing
- **Visual/Sound:** effectPrefab, buffColor, debuffColor, applySound, removeSound

### ActiveBuff (Runtime)

Поля:
- data (BuffData), remainingDuration, currentStacks, tickTimer
- source (GameObject), isActive
- conductivityModifier (для ConductivityBoost)

Свойства:
- `Progress` — прогресс длительности (0-1)
- `TotalValue` — итоговое значение с учётом стеков

# ЧАСТЬ B: МОДИФИКАТОРЫ И ПЕРКИ

## Обзор модификаторов

Система модификаторов — единая система бонусов и штрафов, где **одни и те же переменные** используются для расчёта:
- **Бонусы** → положительные значения переменных
- **Штрафы** → отрицательные значения переменных

### Ключевые принципы

1. **Единые переменные** — бонус и штраф это знак (+/-) значения одной переменной
2. **Суммирование перед применением** — сначала сумма всех модификаторов, потом применение
3. **Мягкие капы** — защита от чрезмерного разгона
4. **Порядок расчёта** — плоские → сумма → проценты

> **Различие баффов и перков:** Баффы — временные эффекты с возможным откатом (ЧАСТЬ A). Перки — постоянные пассивные способности, влияющие на базовые характеристики (развитие персонажа). См. PERK_SYSTEM.md.

---

## Формула расчёта

### Базовый алгоритм

```
┌─────────────────────────────────────────────────────────────────────┐
│                    АЛГОРИТМ РАСЧЁТА                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  ШАГ 1: Сбор модификаторов                                          │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │  flat_modifiers = [+10, +5, -3, +8]                         │    │
│  │  percent_modifiers = [+10%, +5%, -15%, +20%]                │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                      │
│  ШАГ 2: Суммирование                                                │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │  flat_sum = 10 + 5 + (-3) + 8 = 20                          │    │
│  │  percent_sum = 10 + 5 + (-15) + 20 = 20%                    │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                      │
│  ШАГ 3: Применение                                                  │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │  result = (base_value + flat_sum) × (1 + percent_sum/100)   │    │
│  │  result = (100 + 20) × 1.20 = 144                           │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                      │
│  ШАГ 4: Мягкий кап                                                  │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │  result = apply_soft_cap(result, base_value, variable_id)   │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Порядок применения

**Правило:** Плоские → Проценты

**Почему:** Проценты должны усиливать ВСЮ базу, включая плоские бонусы.

**Пример:**
```
База: 100
Плоские: +30
Проценты: +50%

result = (100 + 30) × 1.5 = 195
```

## Мягкие капы (Soft Caps)

### Зачем нужны

Без капов можно получить:
- +200% скорости → персонаж движется в 3 раза быстрее
- +150% урона → убивает врагов с одного удара
- -80% стоимости Ци → техники почти бесплатны

Мягкие капы позволяют получать бонусы сверх лимита, но с **убывающей эффективностью**.

### Формула диминишинга

> **Источник истины:** [ALGORITHMS.md](./ALGORITHMS.md) §6 «Мягкие капы»

**Экспоненциальное затухание:**

```
effectiveBonus = cap × (1 - e^(-bonus / (cap × decayRate)))
```

Где `decayRate` — индивидуальный параметр для каждой переменной (см. таблицу ниже).

| Бонус | Кап | Эффективность |
|-------|-----|---------------|
| = кап | 50% | ~63% (31.5%) |
| = 2× кап | 50% | ~86% (43%) |
| = 3× кап | 50% | ~95% (47.5%) |

### Конфигурация капов

| Переменная | Тип капа | Значение | Decay |
|------------|----------|----------|-------|
| **Скорость** | | | |
| speed | absolute | ±50% | 1.5 |
| attack_speed | positive | +75% | 1.2 |
| **Урон** | | | |
| damage | positive | +100% | 1.0 |
| crit_chance | positive | +50% | 0.8 |
| crit_damage | positive | +150% | 1.0 |
| **Защита** | | | |
| defense | positive | +80% | 1.2 |
| armor | positive | +200 | 1.5 |
| **Ци** | | | |
| qi_cost | negative | -50% | 1.0 |
| qi_efficiency | positive | +50% | 1.0 |
| **Утилити** | | | |
| cooldown | negative | -60% | 1.2 |
| life_steal | positive | +30% | 0.8 |

```

Эффективность
    │
100%├─────────────────────────────────────────────────────
    │                    ╭──────────────────────

    │               ╭─╯

    │           ╭─╯

    │       ╭─╯

    │   ╭─╯
```

## Типы модификаторов

### Плоские (Flat)

Добавляют фиксированное значение к базе.

**Примеры:**
- +10 к урону
- +50 HP

### Процентные (Percent)

Умножают итоговое значение.

**Примеры:**
- +20% к урону
- +15% скорости
- -10% стоимости Ци

### Аддитивное сложение процентов

**Правило:** Проценты суммируются, а не перемножаются!

**Пример:**
```
+20%, +20%, +20% = +60% (НЕ +72.8%)

result = 100 × 1.6 = 160
```

|----|----------|------------|

### Необратимые переменные

Развиваются только естественным путём, НЕ модифицируются бонусами.

| ID | Описание | Как развивается |
|----|----------|-----------------|
| strength | Сила | Тренировка и закалка мышц |
| agility | Ловкость | Тренировка связок и суставов |
| intelligence | Интеллект | Изучение и осознание |
| vitality | Живучесть | Развитие организма и культивация |
| coreCapacity | Ёмкость ядра | Прорывы уровней культивации |
| qiDensity | Плотность Ци | Уровень культивации (2^(level-1)) |

**Исключение:** qi_conductivity может быть временно модифицирована:
- Повреждение меридиан → снижение
- Техники «очищения каналов» → временное повышение

> **Проводимость и перки:** Перк влияет на базовую проводимость меридиан (врождённое или приобретённое свойство — «эластичность меридиан» или «увеличенный диаметр»). Это развитие персонажа, НЕ модификатор. Баффы могут временно модифицировать проводимость с откатом, но перки — навсегда.

## Примеры расчётов

### Тяжёлая броня

```
Экипировка: "Тяжёлая латная броня"
Материал: Утяжелённая сталь (тир 3)
Грейд: Refined

=== Бонусы ===
От материала:
  • defense: +15% (тяжёлый металл прочнее)

От грейда:
  • defense: +30% (refined = ×1.3 к защите)

=== Штрафы ===
От веса (25 кг, база 10 кг):
  • speed: -15% (перегруз 15 кг × 1%)
  • stamina_cost: +30% (перегруз 15 кг × 2%)

От материала (плохая проводимость Ци):
  • qi_efficiency: -10%

=== Итоговые модификаторы ===
defense: +15% + 30% = +45%
speed: -15%
stamina_cost: +30%
qi_efficiency: -10%

=== Расчёт защиты ===
Базовая защита: 100
percent_sum: +45%

result = 100 × 1.45 = 145

Мягкий кап для defense: +80%
145 < 180 (кап), применяется полностью

Итоговая защита: 145

=== Расчёт скорости ===
Базовая скорость: 100
percent_sum: -15%

result = 100 × 0.85 = 85

Мягкий кап для speed: ±50%
85 в пределах капа

Итоговая скорость: 85%
```

### Полный сет с проклятием

```
Персонаж: Культиватор 5 уровня

=== Экипировка ===
1. Меч Дракона (refined, мифрил)
   • damage: +20% (material)
   • damage: +30% (grade)
   • crit_chance: +10% (enchant)

2. Броня Титана (perfect, адамантий)
   • defense: +50% (material)
   • defense: +70% (grade: perfect ×1.6)
   • speed: -20% (вес)
   • qi_efficiency: -15% (плохая проводимость)

3. Сапоги Ветра (refined, лёгкий металл)
   • speed: +25%
   • attack_speed: +15%

4. Амулет Жадности (проклятый)
   • qi_cost: -20%
   • luck: -30% (curse)
   • life_steal: +10%

=== Суммирование модификаторов ===
damage:
  • +20% (меч, material)
  • +30% (меч, grade)
  → percent_sum: +50%

defense:
  • +50% (броня, material)
  • +70% (броня, grade)
  → percent_sum: +120%

speed:
  • -20% (броня, weight)
  • +25% (сапоги)
  → percent_sum: +5%

attack_speed:
  • +15% (сапоги)
  → percent_sum: +15%

qi_cost:
  • -20% (амулет)
  → percent_sum: -20%

crit_chance:
  • +10% (меч)
  → percent_sum: +10%

luck:
  • -30% (амулет, curse)
  → percent_sum: -30%

life_steal:
  • +10% (амулет)
  → percent_sum: +10%

=== Расчёт defense с диминишингом ===
База: 100
percent_sum: +120%

result = 100 × 2.20 = 220
Мягкий кап: +80%

Применяем диминишинг:
cap = 80, decayRate = 1.2
effective = 80 × (1 - e^(-120/96)) ≈ 80 × 0.714 ≈ 57%

Итоговая защита: 157% (а не 220%)
```

# ИНТЕГРАЦИЯ

## С техниками

При применении техники с баффами:
1. Получить `BuffManager` цели
2. Для каждого `buffId` в технике:
   - Загрузить `BuffData` из базы
   - Проверить на запрещённые типы (в редакторе)
   - Вызвать `ApplyBuff(buffData, source)`

## С предметами

При использовании расходуемого предмета:
1. Получить `BuffManager` пользователя
2. Применить все баффы из списка предмета

## С экипировкой

Модификаторы от:
- Материала предмета
- Грейда предмета
- Зачарований
- Сетовых бонусов
- Проклятий

## С техниками (модификаторы)

Модификаторы от:
- Активных техник (баффы)
- Пассивных техник
- Культивационных методов

## С состояниями

Модификаторы от:
- Отравлений
- Проклятий
- Благословений
- Окружения

## С окружением

Модификаторы от:
- Местности (болото → -speed)
- Погоды (дождь → -fire_damage)
- Времени суток (ночь → +stealth)
