## 📋 Обзор

Система времени управляет течением игрового времени, отличающегося от реального. Все действия измеряются в игровых минутах.

---

## 🎯 Фундаментальное правило

**1 TICK = 1 СЕКУНДА РЕАЛЬНОГО ВРЕМЕНИ**

Это правило **НЕИЗМЕННО**. Скорость игры регулируется через переменную `minutesPerTick`.

**Важно:** При появлении многопользовательского режима все игроки будут жить в одном времени со статичной скоростью тиков. Сейчас игра однопользовательская.

## 🚀 Система масштабирования времени (Time Scaling)

### Принцип: "Время как ресурс"

**Концепция:** Все действия измеряются в **игровых минутах**, а не в реальных секундах.

```
Действие = "занимает X игровых минут"
Реальная длительность = X / minutesPerTick секунд
```

### Примеры

**Атака (занимает 1 игровую минуту):**
- При normal (5 мин/тик): 1/5 = 0.2 секунды (быстро!)
- При slow (1 мин/тик): 1/1 = 1 секунда
- При superSlow (0.5): 1/0.5 = 2 секунды

**Путешествие 100px (занимает 0.5 игровой минуты):**
- При normal (5 мин/тик): 0.5/5 = 0.1 секунды
- При slow (1 мин/тик): 0.5/1 = 0.5 секунды
- При superSlow (0.5): 0.5/0.5 = 1 секунда

### Плюсы подхода

- Время = ресурс (культивация тратит время)
- Баланс: быстрое время = быстро "тратится"
- Подходит для RPG культивации
- "Тайм-менеджмент" как игровая механика

## 🔄 Автоматическое переключение скорости

### Решения пользователя

**Разная скорость для разных действий?** — ДА

| Действие | Режим времени | Обоснование |
|----------|--------------|-------------|
| Путешествие (выход с локации) | `fast` / `ultra` | Ускорение перемещения |
| Исследование локации | `normal` | Стандартная скорость |
| Бой | `superSuperSlow` | Тактический контроль |
| Медитация | `ultra` | Максимальное ускорение |

**Медитация (ultra) делает всё мгновенным?** — ДА

- В режиме медитации время ускоряется до `ultra` (60 мин/тик)
- Все действия масштабируются соответственно
- Игрок "проживает" часы за секунды

**Боевой режим времени?** — ДА

- Автоматическое переключение на `superSuperSlow` при начале боя
- Возврат к предыдущей скорости после боя
- Возможность ручного переключения во время боя

## 📂 Менеджер активностей (Activity Manager)

### Назначение

Автоматическое переключение скорости времени в зависимости от текущей активности игрока.

### Профили активностей

| Профиль | Скорость | Триггеры |
|---------|----------|----------|
| Combat | superSuperSlow | Начало боя, атака |
| Exploration | normal | Движение по локации |
| Travel | fast | Путешествие между локациями |
| Meditation | ultra | Медитация, отдых |
| Dialogue | slow | Разговор с NPC |

### Поток переключения

```
1. Игрок начинает медитацию
   └── ActivityManager.SetActivity('meditation')
       └── TimeManager.SetSpeed('ultra')

2. Игрок заканчивает медитацию
   └── ActivityManager.SetActivity('exploration')
       └── TimeManager.SetSpeed('normal')

3. Игрок вступает в бой
   └── ActivityManager.SetActivity('combat')
       └── TimeManager.SetSpeed('superSuperSlow')
```

## ⚡ Batch обработка

### Оптимизация производительности

Вместо обновления всех систем каждый тик:

1. **Накопление тиков** в памяти
2. **Отправка каждые 10 тиков** на обработку
3. **Применение изменений** один раз

### QiTickProcessor

Обрабатывает:
- Пассивное накопление Ци
- Рассеивание Ци
- Регенерацию HP

### Сохранение

- Автосохранение каждые 60 тиков (1 минута реального времени)
- Принудительное сохранение при паузе
- Критические события сохраняются немедленно

## 🔧 Известные проблемы и решения

### Проблема 1: timer:start НЕ обрабатывается

**Симптом:** При движении игрока время не начинается

**Решение:** Добавить слушатель события OnTimerStart

### Проблема 2: isPaused не синхронизирован

**Симптом:** Движение заблокировано после старта

**Решение:** Проверять состояние напрямую из TimeManager

### Проблема 3: Разные источники времени

**Симптом:** UI показывает 06:00, игра показывает 07:00

**Решение:** Единый источник истины — TimeManager
