## 1. Обзор Проблемы

### 1.1 Масштаб мира сянси

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                      МАСШТАБ МИРА КУЛЬТИВАЦИИ                                 │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│   Обитаемая суша: 200,000 × 200,000 км (40 млрд км²)                        │
│                                                                              │
│   Для сравнения:                                                             │
│   • Земля (площадь суши): ~148,940,000 км²                                  │
│   • Наш мир: ~268× больше Земли                                              │
│                                                                              │
│   Реалии миров сянси:                                                        │
│   • Практически нереально посетить все локации                              │
│   • Многие области остаются "чёрными" и "серыми"                            │
│   • Огромные расстояния между ключевыми точками                             │
│   • Путешествия занимают годы игрового времени                              │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 1.2 Проблема сохранения

| Проблема | Описание |
|----------|----------|
| **Размер данных** | Полное сохранение всего мира невозможно |
| **Неисследованные зоны** | Нет смысла сохранять то, что не посещено |
| **Производительность** | Быстрая загрузка только нужных областей |
| **Память** | Нельзя держать весь мир в памяти |

```csharp
// === ЛОКАЦИЯ ===
class LocationData
{
    string locationId;           // Уникальный ID
    string locationName;         // "Город Туманный"
    
    // Размеры (ПЕРЕМЕННЫЕ)
    int widthMeters;             // Ширина в метрах
    int heightMeters;            // Высота в метрах
    
    // Позиция в секторе
    string sectorId;             // ID родительского сектора
    int sectorOffsetX;           // Смещение от начала сектора (м)
    int sectorOffsetY;           // Смещение от начала сектора (м)
    
    // Тип и свойства
    LocationType type;           // Megapolis, City, Village, Temple, etc.
    int dangerLevel;             // Уровень опасности
    int qiDensity;               // Плотность Ци
    
    // Точки входа/выхода
    List<EntrancePoint> entrances;
    
    // Связи с другими локациями
    List<LocationConnection> connections;
    
    // Unity сцены (для больших локаций)
    List<string> sceneNames;     // Имена Unity сцен
    bool isMultiScene;           // Требует ли несколько сцен
}

// === ТОЧКА ВХОДА ===
class EntrancePoint
{
    string entranceId;           // "north_gate", "south_road"
    string displayName;          // "Северные ворота"
    int tileX, tileY;            // Позиция в тайлах (1×1 м)
    List<string> connectsTo;     // Куда ведёт (ID локаций или "wilderness")
}

// === СВЯЗЬ МЕЖДУ ЛОКАЦИЯМИ ===
class LocationConnection
{
    string fromLocationId;
    string fromEntranceId;
    string toLocationId;
    string toEntranceId;
    
    // Параметры пути
    float distanceKm;            // Расстояние в км
    int travelTimeMinutes;       // Время пути в игровых минутах
    TravelType travelType;       // Walk, Ride, Fly, Teleport
    float dangerModifier;        // Модификатор случайных встреч
    RoadQuality roadQuality;     // Плохая, Обычная, Хорошая, Торговая
}
```

### 6.2 Сохранение локации

```csharp
// === СОХРАНЕНИЕ ЛОКАЦИИ ===
class LocationSaveData
{
    string locationId;
    long lastVisited;
    
    // Тип локации определяет объём сохранения
    LocationType type;
    
    // Для СЛУЧАЙНЫХ локаций (упрощённое)
    int seed;                    // Seed для генерации
    List<ModifiedObject> changes; // Только изменённые объекты
    
    // Для СЮЖЕТНЫХ локаций (полное)
    List<ObjectSaveData> objects;   // Все объекты
    List<NPCSaveData> npcs;         // Все NPC
    List<ContainerSaveData> containers; // Сундуки, шкафы
    List<DoorSaveData> doors;       // Двери, проходы
    Dictionary<string, bool> flags; // Локальные флаги
}

// === ИЗМЕНЁННЫЙ ОБЪЕКТ (для случайных локаций) ===
class ModifiedObject
{
    string objectId;
    string modificationType;      // "destroyed" | "looted" | "changed"
    Dictionary<string, object> changes;
}
```

### 7.1 Размеры записей

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                      РАЗМЕРЫ ЗАПИСЕЙ                                         │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│   ЧАНК (минимум):                    ~25 байт                               │
│   СЕКТОР (метаданные):               ~50 байт                               │
│   ЛОКАЦИЯ (метаданные):              ~100 байт                              │
│                                                                              │
│   ЛОКАЦИЯ СЛУЧАЙНАЯ (сохранение):    ~100-500 байт                          │
│   ЛОКАЦИЯ СЮЖЕТНАЯ (сохранение):     ~3-10 КБ                               │
│                                                                              │
│   ТАЙЛЫ НЕ СОХРАНЯЮТСЯ индивидуально!                                       │
│   Генерируются процедурно по seed + изменения                               │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 7.2 Сценарии использования

| Время игры | Посещено локаций | Размер сохранения |
|------------|------------------|-------------------|
| 10 часов | 10-20 | ~10-30 КБ |
| 100 часов | 50-100 | ~50-150 КБ |
| 1000 часов | 300-500 | ~200-500 КБ |
| Экстремум | 2000 | ~1-2 МБ |

### 7.3 Оптимизации

| Оптимизация | Уменьшение | Итог (100 ч) |
|-------------|------------|--------------|
| Без оптимизации | — | ~100 КБ |
| Бинарный формат | ×3-4 | ~30 КБ |
| Сжатие GZIP | ×2-3 | ~15 КБ |
| Дельта-сохранение | ×2-5 | ~5-8 КБ |
