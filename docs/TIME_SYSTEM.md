## 📋 Обзор

Система времени управляет течением игрового времени, отличающегося от реального. Все действия измеряются в тиках (= игровых минутах).

---

## 🎯 Фундаментальное правило

**1 тик = 1 МИНУТА ИГРОВОГО ВРЕМЕНИ**

Скорость тиков в реальных секундах настраивается игроком. Чем больше тиков в секунду, тем быстрее течёт игровое время.

**Важно:** При появлении многопользовательского режима все игроки будут жить в одном времени со статичной скоростью тиков. Сейчас игра однопользовательская.

## 🚀 Система масштабирования времени (Time Scaling)

### Принцип: "Время как ресурс"

**Концепция:** Все действия измеряются в **тиках (= игровых минутах)**, а не в реальных секундах.

```
Действие = "занимает X тиков (= X игровых минут)"
Реальная длительность = X / ticksPerSecond секунд
```

### Примеры

**Атака (занимает 1 тик = 1 игровую минуту):**
- При normal (1 тик/сек): 1/1 = 1 секунда
- При fast (5 тиков/сек): 1/5 = 0.2 секунды
- При quick (15 тиков/сек): 1/15 ≈ 0.07 секунды

**Перемещение на 1 клетку (занимает 1 тик = 1 игровую минуту):**
- При normal (1 тик/сек): 1/1 = 1 секунда
- При fast (5 тиков/сек): 1/5 = 0.2 секунды
- При quick (15 тиков/сек): 1/15 ≈ 0.07 секунды

**Медитация 1 час (занимает 60 тиков = 60 игровых минут):**
- При normal (1 тик/сек): 60/1 = 60 секунд
- При fast (5 тиков/сек): 60/5 = 12 секунд
- При quick (15 тиков/сек): 60/15 = 4 секунды

### Плюсы подхода

- Время = ресурс (культивация тратит время)
- Баланс: быстрое время = быстро "тратится"
- Подходит для RPG культивации
- "Тайм-менеджмент" как игровая механика

### Движение

| Действие | Тиков (= игровых минут) | Примечание |

| Микро-шаг (внутри клетки) | 0.1 | Подгонка позиции без смены клетки |

## 📂 Менеджер активностей (Activity Manager)

### Назначение

Автоматическое переключение скорости времени в зависимости от текущей активности игрока.

### Профили активностей

| Профиль | Скорость | Триггеры |
|---------|----------|----------|
| Combat | normal | Начало боя, атака |
| Exploration | normal | Движение по локации |
| Travel | fast | Путешествие между локациями |
| Meditation | quick | Медитация, отдых |
| Dialogue | normal | Разговор с NPC |

### Поток переключения

```
1. Игрок начинает медитацию
   └── ActivityManager.SetActivity('meditation')
       └── TimeManager.SetSpeed('quick')

2. Игрок заканчивает медитацию
   └── ActivityManager.SetActivity('exploration')
       └── TimeManager.SetSpeed('normal')

3. Игрок вступает в бой
   └── ActivityManager.SetActivity('combat')
       └── TimeManager.SetSpeed('normal')
```

## ⚡ Batch обработка

### Оптимизация производительности

Вместо обновления всех систем каждый тик:

1. **Накопление тиков** в памяти
2. **Отправка каждые 10 тиков** на обработку
3. **Применение изменений** один раз

### QiTickProcessor

Обрабатывает:
- Пассивное накопление Ци
- Рассеивание Ци
- Регенерацию HP

### Сохранение

- Автосохранение каждые 60 тиков (= 60 игровых минут = 1 игровой час)
- При normal (1 тик/сек): каждые 60 реальных секунд
- При fast (5 тиков/сек): каждые 12 реальных секунд
- Принудительное сохранение при паузе
- Критические события сохраняются немедленно

## ⚔️ Стоимость действий во времени

| Действие | Time Cost | Примечание |
|----------|-----------|------------|
| Move 1 tile | 1 tick (= 1 game-min) | Ловкость, местность |
| Attack (melee) | 1 tick (= 1 game-min) | Скорость оружия |
| Attack (ranged) | 1-2 ticks | Оружие, прицеливание |
| Cast technique | 2-5 ticks | Сложность, навык |
| Meditate 1 hour | 60 ticks | Локация, уровень |
| Rest 1 hour | 60 ticks | Локация |

> **Согласование (M-05/M-06):** 1 tick = 1 игровая минута. «Базовая атака = 1 game-min» и «Attack = 1 tick» — одно и то же. «Микро-шаг = 0.1 game-min» — подгонка внутри клетки, не перемещение на клетку.

## 🔧 Известные проблемы и решения

### Проблема 1: timer:start НЕ обрабатывается

**Симптом:** При движении игрока время не начинается

**Решение:** Добавить слушатель события OnTimerStart

### Проблема 2: isPaused не синхронизирован

**Симптом:** Движение заблокировано после старта

**Решение:** Проверять состояние напрямую из TimeManager

### Проблема 3: Разные источники времени

**Симптом:** UI показывает 06:00, игра показывает 07:00

**Решение:** Единый источник истины — TimeManager
