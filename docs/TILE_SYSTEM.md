## 📋 Обзор

Тайловая система — основа локальных карт, на которых происходит основной геймплей. Каждый тайл представляет собой ячейку **2×2 м**, содержащую данные о проходимости, объектах, субъектах и параметрах окружения.

> **Размер тайла = 2×2 м** (см. [WORLD_MAP_SYSTEM.md](./WORLD_MAP_SYSTEM.md) §0.2)

#### Диапазон высот

| Уровень Z | Описание | Примеры |
|-----------|----------|---------|
| **+5 и выше** | Экстремальная высота | Пики гор, небесные острова |
| **+3..+4** | Высокая точка | Вершины холмов, башни |
| **+1..+2** | Возвышенность | Холмы, уступы |
| **0** | Базовый уровень | Равнина, пол, поверхность |
| **-1..-2** | Понижение | Ямы, овраги, подвалы |
| **-3 и ниже** | Глубина | Пропасти, глубокие пещеры |

#### Влияние высоты на геймплей

| Разница Z | Эффект | Механика |
|-----------|--------|----------|
| **+2 и выше** | Падение | Урон = (Z_diff - 1) × 20 |
| **+1** | Преимущество | +10% к урону, +5% к уклонению |
| **0** | Нейтрально | Без бонусов |
| **-1** | Невыгодно | -5% к урону, +5% получить крит |
| **-2 и ниже** | Серьёзный минус | -15% урон, -10% уклонение |

#### Расчёт плотности Ци на тайле

```
finalQiDensity = baseQiDensity
               × terrainModifier
               × climateModifier
               × localSourceModifier
               × formationModifier
               × eventModifier
```

## 💾 Сохранение тайлов

> **Подробнее о системе сохранения см. [WORLD_SAVE_SYSTEM.md](./WORLD_SAVE_SYSTEM.md)**

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                      СОХРАНЕНИЕ КАРТЫ ТАЙЛОВ                                  │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│   ПРИНЦИПЫ:                                                                 │
│   ├── Изменённые тайлы (delta)                                             │
│   ├── Сжатие повторяющихся паттернов                                       │
│   ├── Стандартные тайлы (генерируются из seed)                             │
│   └── Только уникальные данные                                             │
│                                                                              │
│   ФОРМАТ:                                                                   │
│   {                                                                         │
│     "locationId": "loc_001",                                               │
│     "seed": 123456789,                                                      │
│     "modifiedTiles": [                                                      │
│       { "x": 5, "y": 3, "changes": { "terrain": "dirt" } },                │
│       { "x": 7, "y": 2, "changes": { "qiDensity": 150 } }                  │
│     ]                                                                        │
│   }                                                                         │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘
```

### Техники оптимизации

| Техника | Описание | Применение |
|---------|----------|------------|
| **Delta Compression** | Сохранение только изменений | Уменьшение размера |
| **LOD (Level of Detail)** | Упрощение далёких тайлов | Визуализация |
| **Chunking** | Группировка тайлов в чанки | Загрузка/выгрузка |
| **Pooling** | Переиспользование объектов | Производительность |
