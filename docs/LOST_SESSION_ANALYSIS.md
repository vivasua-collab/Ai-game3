### 1.2 Анализ "пулемёта" техник
**Вопрос:** Как избежать rapid-fire и нужно ли?

**Решение:** НЕ избегать!
- L9 практик — полубог, ДОЛЖЕН "косить траву" быстро
- Естественные ограничители: запас Ци, проводимость, мастерство
- Прогрессия: L1-L3 тактический → L7-L9 "аниме-бой"

### 1.3 Удаление regenerationMultiplier
**Проблема:** Множитель регенерации ×1.1 → ×1000 не имеет лорного обоснования.

**Решение:** УДАЛИТЬ из:
- `CultivationLevelData.cs`
- `cultivation_levels.json`
- `AssetGenerator.cs`

**Результат:**
```
Регенерация = 10% ядра/сутки для ВСЕХ уровней (константа!)
L1: 0.0012 Ци/сек
L9: 2.37 Ци/сек
Время восстановления: 10 суток (одинаково для всех!)
```

**Обоснование:** "Дар мира" — скорость выработки микроядром ПОСТОЯННА и НЕ МОЖЕТ быть увеличена искусственно!

### 1.4 Два способа быстрого восстановления Ци
1. **Медитация** — поглощение Ци из окружения (вне боя)
2. **Зарядник + Камни Ци** — экстренное восстановление (в бою)

## 2. 🔴 КРИТИЧЕСКИЕ РАСХОЖДЕНИЯ

### 2.1 Плотность Ци vs Ёмкость ядра

**LORE (правильно):**
```
"Когда повышается уровень L, Ци запрессовывается в 2 раза"
1000 Ци практика L2 = 2000 Ци практики L1 (в базовых единицах)
```

**Текущая реализация:**
```
coreVolume = 1000 × 1.1^totalSubLevels  // Растущий резервуар
qiDensity = 2^(level-1)                  // Дополнительный множитель
```

**Проблема:** Формула ёмкости НЕ учитывает сжатие правильно!

### 2.2 Две конкурирующие модели

| Аспект | Модель А (текущая) | Модель Б (LORE) |
|--------|-------------------|-----------------|
| Резервуар | Растёт экспоненциально | Постоянный объём |
| Плотность | Множитель урона | Качество Ци |
| Базовое Ци | Не используется | = объём × плотность |
| Прорыв | Ци тратится | Ци сжимается в 2× |

**Критическое открытие:** Обе модели дают ОДИНАКОВЫЕ цифры в интерфейсе!

Разница только в:
- Поведении при прорыве
- Передаче Ци между практиками разного уровня

### 2.3 Правило прорыва
**LORE:** После прорыва Ци = 0 (ядро пустое)

**Текущий код (ОШИБКА):**
```csharp
SpendQi((int)required);  // Тратится только required
```

**Правильно:**
```csharp
currentQi = 0;  // ВСЁ Ци тратится на прорыв
```

## 3. Время прорывов

### 3.1 Фундаментальные формулы
```
Проводимость = coreVolume / 360 ИГРОВЫХ секунд
Требование малого прорыва = coreVolume × 10
Требование большого прорыва = coreVolume × 100
```

**Ключевой вывод:** Время прорыва ОДИНАКОВО для всех уровней!
```
Время = requiredQi / conductivity = (coreVolume × 100) / (coreVolume / 360) = 36,000 игровых секунд = 10 игровых часов
```

### 3.2 Время наблюдения игрока (ultra режим)
| Действие | Реальное время |
|----------|---------------|
| Заполнить ядро | 0.1 сек |
| Малый прорыв | 8 сек |
| Большой прорыв | ~1.3 мин |
| L1→L9 полный путь | ~24 минуты |

### 3.3 Плотность Ци окружения (из LORE)
| Зона | ρ_окр (ед/м³) |
|------|--------------|
| Внутреннее кольцо | 500-1000+ |
| Среднее кольцо | 200-500 |
| Внешнее кольцо | 100-200 |
| Приграничные | 20-100 |

## 4. 🔴 Галлюцинация ИИ: environmentMult

**Проблема:** Предыдущий агент ввёл `environmentMult` — множитель скорости поглощения, которого НЕТ в лоре!

**Что есть в лоре:**
- `ρ_окр` — плотность Ци в воздухе (ед/м³)
- `G` — скорость истечения Ци местности (ед/ч·м²)

**Реальная модель поглощения:**
```
1. Практик собирает ВСЮ Ци в зоне радиуса R
2. Ци в зоне ИСЧЕЗАЕТ (ρ_окр → 0)
3. Восстановление из недр — ОЧЕНЬ МЕДЛЕННОЕ

ρ_окр НЕ влияет на скорость поглощения!
Практик просто собирает Ци из объёма, затем ждёт восполнения.
```
