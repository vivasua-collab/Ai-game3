## 📦 Модели данных

### Item (Предмет)

| Поле | Тип | Описание |
|------|-----|----------|
| id | string | Уникальный ID |
| name | string | Название |
| nameId | string | ID для поиска пресета |
| description | string | Описание |
| type | ItemType | Тип предмета |
| category | ItemCategory | Категория |
| rarity | Rarity | Редкость |
| icon | string | Эмодзи или путь |
| quantity | int | Количество |
| maxStack | int | Макс. в стеке |
| stackable | bool | Можно стакать |
| sizeWidth | int | Ширина (1-2) |
| sizeHeight | int | Высота (1-3) |
| weight | float | Вес |
| value | int | Стоимость |

### Типы предметов (11 подтипов)

| Тип | Описание |
|-----|----------|
| weapon | Оружие |
| armor | Броня |
| jewelry | Украшения |
| charger | Зарядник Ци |
| tool | Инструмент |
| consumable | Расходник |
| artifact | Артефакт |
| implant | Имплант |
| material | Материал |
| qi_stone | Камень Ци |
| technique_scroll | Свиток техники |

> **Примечание (M-23):** ARCHITECTURE.md определяет 5 базовых классов предметов: **Consumable**, **Equipment**, **Material**, **Qi Stone**, **Technique Scroll**. Перечисленные выше 11 типов — это подкатегории внутри этих базовых классов:
> - **Consumable** → consumable
> - **Equipment** → weapon, armor, jewelry, charger, tool, artifact, implant
> - **Material** → material
> - **Qi Stone** → qi_stone
> - **Technique Scroll** → technique_scroll

## ⚔️ Система экипировки

### Архитектура "Матрёшка"

Три независимых слоя:

1. **Базовый класс** (неизменен)
   - Тип экипировки
   - Уровень предмета
   - Материал
   - Требования

2. **Материал** (неизменен)
   - Тир материала
   - Базовые свойства
   - Особые эффекты

3. **Grade** (изменяем)
   - Качество: Damaged → Common → Refined → Perfect → Transcendent
   - Множители параметров
   - Дополнительные бонусы

### Grade множители (5 грейдов)

| Grade | Efficiency | Durability |
|-------|------------|------------|
| Damaged | ×0.5 | ×0.5 |
| Common | ×1.0 | ×1.0 |
| Refined | ×1.3 | ×1.5 |
| Perfect | ×1.6 | ×2.5 |
| Transcendent | ×2.0 | ×4.0 |

> **Источник истины:** [EQUIPMENT_SYSTEM.md](./EQUIPMENT_SYSTEM.md) §2 "Система грейдов"

Подробная таблица грейдов и распределение — в EQUIPMENT_SYSTEM.md.

## 🎯 Слоты экипировки

### EquipmentSlot enum (канонический)

| Слот | Описание | Типы предметов |
|------|----------|---------------|
| `Head` | Голова | Шлем, корона, шапка |
| `Torso` | Торс | Нагрудник, кираса, роба |
| `Belt` | Пояс | Пояс, кушак |
| `Legs` | Ноги | Поножи, штаны |
| `Feet` | Ступни | Сабатоны, сапоги |
| `WeaponMain` | Основная рука | Оружие, щит |
| `WeaponOff` | Вторичная рука | Оружие, щит |
| `Amulet` | Амулет | Ожерелье, колье |
| `RingLeft1` | Кольцо лев. рука 1 | Кольцо |
| `RingLeft2` | Кольцо лев. рука 2 | Кольцо |
| `RingRight1` | Кольцо прав. рука 1 | Кольцо |
| `RingRight2` | Кольцо прав. рука 2 | Кольцо |
| `Charger` | Зарядник Ци | Пояс-накопитель, браслет и др. |
| `Hands` | Кисти | Перчатки, боевые перчатки |
| `Back` | Спина | Плащ, ранец |

> **Двуручное оружие:** занимает слоты `WeaponMain` + `WeaponOff` одновременно.

### Правила экипировки

1. **Каждый слот** — один предмет (кроме двуручного оружия)
2. **Двуручное оружие** — занимает WeaponMain + WeaponOff
3. **Зарядник** — один на персонажа (слот `Charger`)
4. **Кольца** — по 2 на каждую руку, максимум 4 (RingLeft1/2, RingRight1/2)
5. **Амулет** — только 1 слот (`Amulet`)

## 📈 Бонусы экипировки

### Типы бонусов

| Категория | Бонусы |
|-----------|--------|
| Combat | Урон, крит, пробитие |
| Defense | Броня, уклонение, HP |
| Qi | Регенерация, проводимость |
| Elemental | Сопротивления стихиям |
| Special | Вампиризм, удача |
| Utility | Скорость, скрытность |

### Структура бонуса

| Поле | Описание |
|------|----------|
| type | Тип бонуса |
| value | Значение |
| isPercent | % или абсолютный |
| source | Источник (base, grade, material, set) |

## 🎭 Сетовые бонусы

### Структура сета

| Поле | Описание |
|------|----------|
| id | ID сета |
| name | Название |
| pieces | ID предметов сета |
| setBonuses | Бонусы за количество |

### Пример

```
Fire Lord Set:
  Pieces: 5
  2 pieces: +20% сопротивление огню
  3 pieces: +5 интеллект, техника fire_aura
  5 pieces: +15 интеллект, +80% сопротивление огню
```
