## ⚔️ Система экипировки

### Архитектура "Матрёшка"

Три независимых слоя:

1. **Базовый класс** (неизменен)
   - Тип экипировки
   - Уровень предмета
   - Материал
   - Требования

2. **Материал** (неизменен)
   - Тир материала
   - Базовые свойства
   - Особые эффекты

3. **Grade** (изменяем)
   - Качество: Damaged → Common → Refined → Perfect → Transcendent
   - Множители параметров
   - Дополнительные бонусы

4. **Зачарование** (наложение)
   - Магический эффект на предмет
   - Независимый модификатор поверх Grade
   - Подробнее: [EQUIPMENT_SYSTEM.md](./EQUIPMENT_SYSTEM.md) §5.5 «Система зачарования»

> **Источники бонусов предмета:** base, grade, material, set, **enchant** (5 источников).

| `Hands` | Кисти 🔒 | Перчатки, боевые перчатки | **ЗАГЛУШКА** — будущее расширение |
| `Back` | Спина 🔒 | Плащ, ранец | **ЗАГЛУШКА** — будущее расширение |

> **Двуручное оружие:** занимает слоты `WeaponMain` + `WeaponOff` одновременно.

### Правила экипировки

1. **Каждый слот** — один предмет (кроме двуручного оружия)
2. **Двуручное оружие** — занимает WeaponMain + WeaponOff
3. **Зарядник** — один на персонажа (слот `Charger`)
4. **Кольца** — по 2 на каждую руку, максимум 4 (RingLeft1/2, RingRight1/2)
5. **Амулет** — только 1 слот (`Amulet`)

## 🔧 Будущая разработка: Сеточная модель инвентаря

> ⚠️ Сеточная модель (grid-based) **не удалена**, а отложена для возможной реализации в будущем.
> Текущая реализация использует строчную модель (список + масса/объём).

### Описание сеточной модели

- Предметы размещаются в сетке по координатам **posX / posY** (0-6)
- Каждый предмет занимает **sizeWidth × sizeHeight** ячеек (например, 1×1, 2×3)
- Рюкзак определяет размер сетки: **gridWidth × gridHeight** (например, 3×4 = 12 слотов)
- Предметы нельзя разместить, если они не помещаются в свободные ячейки
- SwapSlots() — обмен предметов между слотами

### Поля (удалены из текущего кода)

| Поле | Тип | Описание |
|------|-----|----------|
| posX | int | X-позиция в сетке (0-6) |
| posY | int | Y-позиция в сетке (0-6) |
| sizeWidth | int | Ширина предмета (1-2) |
| sizeHeight | int | Высота предмета (1-3) |
| gridWidth | int | Ширина сетки рюкзака |
| gridHeight | int | Высота сетки рюкзака |

### Почему отложена

Сеточная модель добавляет сложность размещения предметов (packing algorithm) без существенного геймплея для текущего этапа. Строчная модель с ограничениями по массе и объёму проще в реализации и обеспечивает достаточную глубину управления инвентарём.
