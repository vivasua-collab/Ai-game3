## 1. Обзор Системы

### 1.1 Что такое Перки?

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                      ЧТО ТАКОЕ ПЕРКИ?                                        │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│   Перк (Perk/Feature/Trait) — постоянная пассивная способность персонажа.   │
│                                                                              │
│   Отличия от других систем:                                                 │
│   ┌───────────────────────────────────────────────────────────────────────┐ │
│   │                        │                    │                         │ │
│   │   ПЕРКИ                │   БАФФЫ            │   НАВЫКИ                │ │
│   ├───────────────────────────────────────────────────────────────────────┤ │
│   │ Длительность:          │                    │                         │ │
│   │   Постоянные           │   Временные        │   Развиваются           │ │
│   │   (до снятия)          │   (секунды/минуты) │   (тренировка)          │ │
│   ├───────────────────────────────────────────────────────────────────────┤ │
│   │ Получение:             │                    │                         │ │
│   │   События, выбор,      │   Техники,         │   Повторяющиеся         │ │
│   │   достижения, рождение │   предметы         │   действия              │ │
│   ├───────────────────────────────────────────────────────────────────────┤ │
│   │ Эффект:                │                    │                         │ │
│   │   Уникальные           │   Стандартные      │   Прогрессивные         │ │
│   │   особенности          │   бонусы           │   улучшения             │ │
│   └───────────────────────────────────────────────────────────────────────┘ │
│                                                                              │
│   Перк = "Твоя уникальная особенность"                                      │
│   Бафф = "Временное усиление"                                               │
│   Навык = "Развитая способность"                                            │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 1.2 Ключевые принципы

| Принцип | Описание |
|---------|----------|
| **Перманентность** | Перк действует постоянно, пока не удалён |
| **Уникальность** | Каждый перк — особенный эффект, не просто +X% |
| **Ограниченность** | Ограниченное количество слотов для перков |
| **1 перк = 1 слот** | Каждый перк занимает ровно один слот |
| **Получение через игру** | Зарабатываются действиями, событиями, выборами |

## 3. Типы Перков

### 3.1 Категории по происхождению

| Категория | Описание | Слоты |
|-----------|----------|-------|
| **🌟 Врождённые** | При создании персонажа, неизменяемы | Вне системы |
| **🎯 Приобретённые** | Через игру, можно заменить | Основные слоты |
| **💀 Проклятые** | Навязанные, имеют +/- | Отдельные слоты (до 3) |

### 3.2 Категории по эффекту

| Категория | Описание | Примеры |
|-----------|----------|---------|
| **Боевые** | Влияют на бой | +урон, +крит, новые атаки |
| **Выживание** | Защита и регенерация | HP, сопротивления, иммунитеты |
| **Культивация** | Ци и развитие | Поглощение Ци, прорывы |
| **Социальные** | Взаимодействие с NPC | Торговля, репутация, диалоги |
| **Исследование** | Мир и путешествия | Скорость, видение, поиск |
| **Крафт** | Создание предметов | Качество, скорость, рецепты |

### 4.2 Проводимость меридиан

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    ПРОВОДИМОСТЬ МЕРИДИАН                                     │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│   Базовая формула (при становлении практиком):                              │
│   ┌───────────────────────────────────────────────────────────────────────┐ │
│   │                                                                        │ │
│   │   conductivity = coreCapacity / 360 секунд                            │ │
│   │                                                                        │ │
│   │   Пример L1.0: 1000 / 360 ≈ 2.78 Ци/сек                               │ │
│   │   Пример L5.0: 45260 / 360 ≈ 125.7 Ци/сек                             │ │
│   │                                                                        │ │
│   └───────────────────────────────────────────────────────────────────────┘ │
│                                                                              │
│   Что определяет проводимость:                                              │
│   ┌───────────────────────────────────────────────────────────────────────┐ │
│   │                                                                        │ │
│   │   1. Скорость ПОГЛОЩЕНИЯ Ци из окружающей среды (медитация)           │ │
│   │      absorbedQi = meditationTime × conductivity × environmentMult     │ │
│   │                                                                        │ │
│   │   2. Скорость ВЫВОДА Ци для техник                                    │ │
│   │      outputRate = conductivity × qiDensity                            │ │
│   │                                                                        │ │
│   │   3. Ёмкость БУФЕРА меридиан                                          │ │
│   │      bufferVolume = conductivity × 5 секунд                           │ │
│   │                                                                        │ │
│   └───────────────────────────────────────────────────────────────────────┘ │
│                                                                              │
│   ╔═════════════════════════════════════════════════════════════════════╗   │
│   ║  РАЗВИТИЕ ПРОВОДИМОСТИ — ВОЗМОЖНО ЧЕРЕЗ ПЕРКИ!                      ║   ║
│   ╠═════════════════════════════════════════════════════════════════════╣   ║
│   ║                                                                      ║   ║
│   ║  Бонус проводимости (conductivityBonus):                             ║   ║
│   ║                                                                      ║   ║
│   ║  finalConductivity = baseConductivity × (1 + conductivityBonus)     ║   ║
│   ║                                                                      ║   ║
│   ║  Источники бонуса:                                                   ║   ║
│   ║  • Врождённый перк "Золотое качество тела" — +30%                   ║   ║
│   ║  • Приобретённый перк "Закалка меридиан" — +15%                     ║   ║
│   ║  • Техника "Небесные каналы" (временный) — +50%                     ║   ║
│   ║  • Формация "Круг поглощения" (локальный) — +25%                    ║   ║
│   ║                                                                      ║   ║
│   ║  Ограничение: Поглощение НЕ может превышать проводимость!           ║   ║
│   ║                                                                      ║   ║
│   ║  absorbedQi = min(                                                  ║   ║
│   ║      meditationTime × conductivity × environmentMult,               ║   ║
│   ║      availableQiInEnvironment                                       ║   ║
│   ║  )                                                                  ║   ║
│   ║                                                                      ║   ║
│   ╚═════════════════════════════════════════════════════════════════════╝   │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 4.3 Примеры перков для проводимости

| Перк | Тип | Эффект |
|------|-----|--------|
| **Золотое качество тела** | Врождённый | +30% проводимость меридиан |
| **Закалка меридиан** | Приобретённый | +15% проводимость (требует 100 часов медитации) |

## 7. Интеграция с кодом

### 7.1 Структуры данных (будущее)

```csharp
public enum PerkCategory
{
    Innate,      // Врождённый — вне системы слотов
    Acquired,    // Приобретённый — основные слоты
    Cursed       // Проклятый — отдельные слоты
}

public class PerkData
{
    public string id;
    public string displayName;
    public PerkCategory category;
    public PerkEffect[] effects;
    public bool isPermanent;  // Врождённые и проклятые = true
}

public class PerkController
{
    // Врождённые (фиксированы, 2 слота)
    private List<PerkData> innatePerks = new List<PerkData>();
    
    // Приобретённые (5 + cultivationLevel слотов)
    private List<PerkData> acquiredPerks = new List<PerkData>();
    public int MaxAcquiredSlots => 5 + cultivationLevel;
    
    // Проклятые (до 3 слотов)
    private List<PerkData> cursedPerks = new List<PerkData>();
    public const int MaxCursedSlots = 3;
    
    // Бонус проводимости от перков
    public float GetConductivityBonus()
    {
        float bonus = 0f;
        
        // Врождённые
        foreach (var perk in innatePerks)
            bonus += GetEffectValue(perk, "conductivity_bonus");
        
        // Приобретённые
        foreach (var perk in acquiredPerks)
            bonus += GetEffectValue(perk, "conductivity_bonus");
        
        return bonus;
    }
}
```
