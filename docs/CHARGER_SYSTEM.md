## 📋 Обзор

**Зарядник** — специальный тип экипировки, предназначенный для:
1. **Хранения камней Ци** в упорядоченном виде
2. **Контролируемого поглощения** Ци из камней
3. **Буферизации Ци** для быстрого использования в бою

### Ключевая концепция

│                     ПРИНЦИП РАБОТЫ ЗАРЯДНИКА                            │

## 2️⃣ ТЕХНИЧЕСКИЕ ХАРАКТЕРИСТИКИ

### 2.1 Слоты для камней

| Характеристика | Описание |
|----------------|----------|
| Количество слотов | 1-15 (зависит от форм-фактора) |
| Совместимость | minQuality, maxSize, types |
| Состояние | isActive, isSealed, damageLevel |
| Модификаторы | absorptionBonus, qiRetention |

### 2.2 Буфер Ци

| Параметр | Описание | Диапазон |
|----------|----------|----------|
| capacity | Ёмкость буфера | 50-2000 |
| currentQi | Текущее Ци | 0-max |
| inputRate | Скорость входящего потока | 5-50 ед/сек |
| outputRate | Скорость исходящего потока | 5-50 ед/сек |
| conductivity | Проводимость (ключевой параметр) | 5-100 ед/сек |

### 2.3 Проводимость зарядника

**Ключевой параметр — ограничивает скорость передачи Ци!**

> **Важно:** Зарядник — это испаритель камней Ци с фиксированной проводимостью.
> Практик должен ПОДБИРАТЬ зарядник под свою проводимость меридиан.
> Зарядник НЕ подстраивается под практика!

**Формула итоговой скорости:**

```
effectiveRate = min(
  stone.releaseRate,         // Камень может выдать
  charger.conductivity,       // Зарядник может передать (ФИКСИРОВАНО)
  practitioner.conductivity   // Практик может принять
)
```

**Узкие места:**
- Если `charger.conductivity < practitioner.conductivity` → зарядник узкое место
- Если `charger.conductivity > practitioner.conductivity` → практик узкое место
- **Оптимально:** `charger.conductivity ≈ practitioner.conductivity`

**Примеры проводимости:**

| Качество | Проводимость | Скорость/час | Рекомендуемый практик |
|----------|--------------|--------------|----------------------|
| Damaged | 2.5 ед/сек | 9,000 | L1 (cond: 1-5) |
| Common | 5 ед/сек | 18,000 | L1-L2 (cond: 1-10) |
| Refined | 10 ед/сек | 36,000 | L2-L3 (cond: 5-25) |
| Perfect | 25 ед/сек | 90,000 | L4-L6 (cond: 20-80) |
| Transcendent | 50 ед/сек | 180,000 | L7-L9 (cond: 50-200) |

**Проводимость практика (формула):**
```
practitioner.conductivity = coreCapacity / 360 секунд
```

| Уровень | coreCapacity | Проводимость |
|---------|------------|--------------|
| L1.0 | 1,000 | 2.8 ед/сек |
| L3.0 | 6,727 | 18.7 ед/сек |
| L5.0 | 45,260 | 125.7 ед/сек |
| L9.0 | 2,048,400 | 5,690 ед/сек |

**Вывод:** Практик L5 с проводимостью 126 ед/сек не сможет эффективно использовать Common зарядник (5 ед/сек). Ему нужен минимум Perfect (25 ед/сек), а лучше Transcendent (50 ед/сек).

## 3️⃣ МЕХАНИКА РАБОТЫ

### 3.1 Поглощение из камней

**Поток:**
1. Камни Ци → высвобождают Ци (releaseRate)
2. Зарядник → буферизирует Ци (проводимость)
3. Практик → принимает Ци (conductivity меридиан)

**Узкое место:**
- Проводимость зарядника — главный ограничитель
- Проводимость практика — вторичный ограничитель

### 3.2 Режимы работы

**Только два режима:**

| Режим | Описание | Скорость | Потери |
|-------|----------|----------|--------|
| `off` | Выключен | 0 | 0% |
| `on` | Включен | ×1.0 | 10% |

> **Упрощение:** Убраны сложные режимы (trickle, burst, combat). 
> Зарядник либо работает, либо нет. Потери фиксированы на уровне 10%.

### 3.3 Расчёт скорости

```
effectiveRate = min(
  totalStoneRate × modeMultiplier,
  charger.conductivity,
  practitioner.conductivity
) × (1 - efficiencyLoss)
```

## 4️⃣ ИСПОЛЬЗОВАНИЕ В БОЮ

### 4.1 Проблема бесконечных боёв

> **Проблема:** Если зарядник даёт бесконечную Ци пока есть камни — бой бесконечен.

**Решение: Ограничение через ТРИ фактора:**

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    ОГРАНИЧЕНИЯ ЗАРЯДНИКА В БОЮ                              │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│   1. БУФЕР (мгновенный запас)                                               │
│      ├── capacity: 50-2000 Ци                                               │
│      ├── Можно использовать мгновенно                                       │
│      └── После использования — пуст, требует пополнения                     │
│                                                                              │
│   2. ПОПОЛНЕНИЕ БУФЕРА (скорость)                                           │
│      ├── Проводимость зарядника: 5-50 Ци/сек                               │
│      ├── Буфер не пополняется мгновенно!                                    │
│      └── Пример: буфер 500, проводимость 10 → 50 сек до полного            │
│                                                                              │
│   3. НАГРЕВ (тепловой баланс)                                               │
│      ├── Каждое использование → +heat                                       │
│      ├── Рассеивание: -heat/сек (пассивно)                                 │
│      └── При 100% heat → блокировка на 30 сек                              │
│                                                                              │
│   Итог: Можно сделать несколько мощных техник, потом — ждать или биться    │
│         без Ци. Это создаёт тактический выбор.                              │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 4.2 Механика использования

**Порядок источников Ци для техники:**
1. **Ядро практика** — первичный источник (нет потерь)
2. **Буфер зарядника** — вторичный источник (10% потерь)

**Условия использования:**
| Условие | Источник Ци |
|---------|-------------|
| `currentQi >= qiCost` | Только ядро |
| `currentQi < qiCost` && `buffer >= remaining` | Ядро + зарядник |
| `currentQi = 0` && `buffer >= qiCost` | Только зарядник |
| `currentQi = 0` && `buffer < qiCost` | Техника недоступна |

### 4.3 Тепловой баланс

**Накопление тепла:**
```
heatGain = qiUsed × 0.05  // 5% от использованной Ци = тепло
```

**Примеры:**
| Техника | Ци | Тепло | Накопится за 20 использований |
|---------|-----|-------|-------------------------------|
| melee_strike L1 | 10 | 0.5 | 10% |
| fireball L3 | 50 | 2.5 | 50% |
| ultimate L5 | 250 | 12.5 | 250% → перегрев на 8-й |

**Рассеивание:**
- Пассивно: -1%/сек
- В бою (режим активности): -0.5%/сек

**Перегрев (100%):**
- Зарядник блокируется на 30 секунд
- Камни не высвобождают Ци
- После остывания: 0% тепла, работа возобновляется

### 4.4 Тактический пример

**Зарядник:** belt, буфер 500, проводимость 10

| Ход | Действие | Ци из ядра | Ци из буфера | Буфер после | Тепло |
|-----|----------|------------|--------------|-------------|-------|
| 1 | fireball (50) | 50 | 0 | 500 | 0% |
| 2 | fireball (50) | 0 | 50 | 450 | 2.5% |
| 3 | fireball (50) | 0 | 50 | 400 | 5% |
| ... | ... | ... | ... | ... | ... |
| 10 | fireball (50) | 0 | 50 | 50 | 25% |
| 11 | fireball (50) | 0 | 50 | 0 | 27.5% |
| 12 | Попытка fireball | **Нет Ци!** | — | 0 | — |

**Вывод:** За 10 ходов израсходован буфер. Нужно 50 секунд для пополнения, или сражаться без Ци из зарядника.

### 4.5 Поглощение Ци из окружения

**В бою НЕ работает:**
- Поглощение Ци из окружения требует концентрации (медитация)
- В бою практик фокусируется на противнике
- Только микроядро даёт пассивную регенерацию (10% ядра/день)

**После боя:**
- Можно включить зарядник для быстрого восстановления
- Скорость ограничена проводимостью
- Тепло рассеивается быстрее (-2%/сек вне боя)

### 4.6 Итоговые ограничения

| Ограничение | Значение | Цель |
|-------------|----------|------|
| Буфер | 50-2000 Ци | Ограничивает мгновенный запас |
| Пополнение буфера | 5-50 Ци/сек | Требует времени между техниками |
| Перегрев | 100% → 30 сек блок | Наказание за спам техник |
| Потери | 10% | Штраф за использование зарядника |
| В бою | Нет поглощения из окружения | Только микроядро |

## 5️⃣ ИСПОЛЬЗОВАНИЕ ДЛЯ МЕДИТАЦИИ

### 5.1 Ускорение медитации

**Базовая скорость** = qiFromEnvironment + qiFromMicroCore

**Дополнительно от зарядника** = chargerRate × efficiency

**Итоговая скорость** = baseRate + chargerBoost

### 5.2 Пример расчёта

```
Базовая скорость: 0.5 ед/сек = 1 800 ед/час
Зарядник (проводимость 10, режим on): 
  10 × 0.9 = 9 ед/сек (с учётом 10% потерь)

Итого: 0.5 + 9 = 9.5 ед/сек
Ускорение: ~19x
```

### 5.3 Рекомендации по использованию

**Использовать зарядник когда:**
- Ядро < 90%
- Срочная необходимость в Ци
- Низкая плотность Ци в месте медитации

**НЕ использовать когда:**
- Ядро > 95%
- Нет запаса камней
- Высокая температура зарядника

### 5.4 Оптимальный режим

| Заполненность ядра | Рекомендуемый режим |
|--------------------|---------------------|
| < 95% | on (зарядник работает, Ци поступает) |
| > 95% | off (только базовое накопление из среды) |

> **Примечание:** Расширенные режимы (burst / normal / trickle) рассматриваются как планируемое расширение на будущее. В текущей реализации зарядник работает в режиме on/off.

## 8️⃣ ИНТЕГРАЦИЯ С СИСТЕМОЙ

### 8.1 Как часть экипировки

Зарядник — подтип экипировки:
- equipmentType: 'charger'
- subType: 'charger_belt' / 'charger_bracelet' / etc.

### 8.2 Связь с камнями Ци

Зарядник содержит камни Ци в слотах.
Камни расходуются при использовании.

### 8.3 Связь с техниками

Техники могут использовать Ци из зарядника:
- Проверка источника Ци
- Применение штрафов при использовании зарядника
- Расчёт потерь
