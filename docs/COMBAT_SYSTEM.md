## 💥 Расчёт урона

### Авторитетная формула урона

> **Источник истины:** [ALGORITHMS.md](./ALGORITHMS.md)

**Формула:**
```
finalDamage = capacity × gradeMultiplier × ultimateMultiplier
```

Где:
- `capacity` = baseCapacity × 2^(techniqueLevel-1) × (1 + mastery/100 × 0.5)
- `gradeMultiplier` = множитель Grade (common:1.0, refined:1.3, perfect:1.6, transcendent:2.0)
- `ultimateMultiplier` = isUltimate ? 2.0 : 1.0

> ⚠️ **Устаревшие формулы удалены:** формулы с `qiDensity × statMultiplier × masteryBonus` или `qiInvested` как базой — НЕВЕРНЫ. Единственная правильная формула использует `capacity`.

## 📊 Формула Qi Density (Качество Ци)

> **Источник истины:** [ALGORITHMS.md](./ALGORITHMS.md) §3.3 "Плотность Ци"

**Принцип:** Качество Ци = qiDensity = 2^(уровень-1)

Геометрический рост ×2 за каждый уровень культивации. Подробная таблица — в ALGORITHMS.md.

> **Примечание:** qiDensity встроена в формулу capacity как `2^(techniqueLevel-1)` и не используется отдельно в расчёте урона.

## 📊 Подавление уровнем (Level Suppression)

> **Источник истины:** [ALGORITHMS.md](./ALGORITHMS.md) §1 "Система подавления уровнем"

**Проблема:** Qi Buffer с поглощением 90% позволяет практике L1 наносить урон практике L8.

**Решение:** Множитель подавления на основе разницы уровней.

**Кратко:**
| Разница | Normal | Technique | Ultimate |
|---------|--------|-----------|----------|
| 0 | ×1.0 | ×1.0 | ×1.0 |
| 1 | ×0.5 | ×0.75 | ×1.0 |
| 2 | ×0.1 | ×0.25 | ×0.5 |
| 3 | ×0.0 | ×0.05 | ×0.25 |
| 4 | ×0.0 | ×0.0 | ×0.1 |
| 5+ | ×0.0 | ×0.0 | ×0.0 |

Полная таблица и формулы — в ALGORITHMS.md.

## 🏃 Скорость атаки

### Базовые длительности

| Тип атаки | Игровые минуты |

| Базовая атака | 1 |
| Быстрая техника | 0.5 |

### Влияние ловкости

```
actualDuration = baseDuration / (1 + agility × 0.01)
```

> **Примечание:** Данная формула применяется только к **базовым атакам**. Время каста техник определяется проводимостью (conductivity), уровнем культивации и бонусами мастерства (per ALGORITHMS.md §12).

## 🩸 Применение урона к телу

### Поток повреждения

Урон распределяется **одновременно** (не последовательно!):

```
redHP -= damage × 0.7     // 70% в функциональную HP
blackHP -= damage × 0.3   // 30% в структурную HP
```

- При redHP ≤ 0 → паралич части тела
- При blackHP ≤ 0 → отрубание части тела

> **Источник истины:** [ALGORITHMS.md](./ALGORITHMS.md) — модель одновременного split 0.7/0.3, НЕ последовательного переполнения.

### Кровотечение

При значительном уроне начинается кровотечение:

| Степень | HP/тик |
|---------|--------|
| Лёгкое | 1 |
| Среднее | 3 |
| Тяжёлое | 5 |
| Критическое | 10 |
