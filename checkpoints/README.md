# 📋 Система чекпоинтов разработки

**Цель:** Фиксация этапов разработки проекта CultivationWorldSimulator

---

## 🗂️ Формат имени файла

```
ММ_ДД_цель.md
```

- **ММ** — месяц (01-12)
- **ДД** — день (01-31)
- **цель** — краткое описание этапа или номер фазы

**Примеры:**
- `03_30_phase1_complete.md` — завершена фаза 1
- `03_30_combat_core.md` — чекпоинт боевого ядра
- `04_15_save_system.md` — система сохранений

---

## 📝 Содержимое чекпоинта

Каждый файл содержит:

```markdown
# Чекпоинт: [Название]

**Дата:** YYYY-MM-DD HH:MM
**Фаза:** X
**Статус:** [in_progress | complete | blocked]

## Выполненные задачи
- [x] Задача 1
- [x] Задача 2

## Текущие задачи
- [ ] Задача 3

## Проблемы
- Описание проблем (если есть)

## Следующие шаги
- Что делать дальше

## Изменённые файлы
- путь/к/файлу.cs
```

---

## 🔄 Логика создания чекпоинта

1. **Получить системное время** — `DateTime.Now`
2. **Сформировать имя** — `{MM}_{DD}_{цель}.md`
3. **Записать состояние** — список выполненных задач
4. **Зафиксировать файлы** — какие файлы изменены/созданы
