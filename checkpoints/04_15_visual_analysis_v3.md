## ПРОБЛЕМА 1: Белый фон спрайта персонажа

### Симптом
Спрайт персонажа отображается с белым фоном (непрозрачный прямоугольник).

### Корневая причина
Файл `Assets/Sprites/Characters/Player/player_variant1_cultivator.png`:
- **Размер:** 1024×1024 пикселей
- **Формат:** **RGB** (3 канала, НЕТ альфа-канала!)
- Без альфа-канала Unity интерпретирует все пиксели как непрозрачные → белый фон виден

### Код-путь
1. `PlayerVisual.cs:LoadPlayerSprite()` → `AssetDatabase.LoadAssetAtPath<Sprite>(path)`
2. Unity загружает PNG → создаёт Sprite без прозрачности (RGB → все пиксели opaque)
3. `mainSprite.color = Color.white` → не тонирует (правильно для AI-спрайта), но фон белый
4. Спрайт масштабируется `size=0.4` → `localScale = (0.4, 0.4, 1)` → 0.4 × (1024/32) = 12.8 юнита — **ОГРОМНЫЙ!**

### Дополнительная проблема
PPU спрайта персонажа = 32 (дефолт Unity для sprite import). При 1024×1024 и PPU=32:
- World size = 1024/32 = **32 юнита** × 0.4 scale = **12.8 юнита**
- Это в 6.4 раза больше ячейки тайла (2×2 юнита)!

### Решение
1. **Обработать player_variant1_cultivator.png** → конвертировать RGB→RGBA, удалить белый фон (flood fill от углов)
2. **Уменьшить размер** 1024→128 пикселей (достаточно для персонажа)
3. **Установить PPU=64** → 128/64 = 2.0 юнита × 0.4 scale = 0.8 юнита (разумный размер)
4. Сохранить в `Assets/Sprites/Characters/Player/` (перезаписать исходный)

## ПРОБЛЕМА 2: Белая сетка между тайлами

### Симптом
Белые линии (сетка) между terrain-тайлами. При процедурной генерации спрайтов — сетки нет.

### Корневая причина — МНОГОФАКТОРНАЯ

#### Фактор A: Точный размер спрайта = размер ячейки
- Обработанные AI terrain спрайты в `Tiles/`: **64×64 RGBA**
- PPU=31 → world size = 64/31 = **2.065 юнита**
- Grid cellSize = (2.0, 2.0, 1.0)
- Перекрытие = 0.0325 юнита с каждой стороны

**Но:** В Tilemap каждый тайл рендерится в пределах своей ячейки. Спрайт шире ячейки — это не гарантирует перекрытие, т.к. TilemapRenderer в режиме Chunk может обрезать по границам чанка.

#### Фактор B: TilemapRenderer Mode
Код устанавливает `terrainRenderer.mode = TilemapRenderer.Mode.Individual`. Это было сделано для устранения артефактов чанков, но Individual режим рендерит каждый тайл отдельно → каждый Sprite имеет свой draw call → между ними могут быть субпиксельные зазоры.

#### Фактор C: FilterMode.Point на спрайтах
`filterMode = FilterMode.Point` — нет билинейной интерполяции. При субпиксельном рендеринге между двумя соседними тайлами может быть 1-пиксельный зазор, который Point фильтрация не заполняет.

#### Фактор D: Terrain спрайты с RGBA прозрачными краями
AI-спрайты были обработаны Python-скриптом `preprocess_ai_sprites.py`. Скрипт мог добавить полупрозрачные пиксели по краям при downscale (1024→64 через bilinear), что при Point фильтрации выглядит как белый зазор.

### Решение (комплексное)
1. **Отказаться от AI terrain-спрайтов** — использовать только процедурные (проверено: без сетки)
2. **Процедурные terrain-спрайты 68×68 при PPU=32** = 2.125 юнита → 0.0625 юнита перекрытие с каждой стороны
3. **TilemapRenderer.mode = Chunk** — рендерит чанк целиком, нет зазоров между тайлами
4. **FilterMode.Bilinear** для terrain спрайтов — сглаживает субпиксельные границы
5. Удалить AI terrain спрайты из Tiles/ (obj_* оставить — они с прозрачностью)

## ПРОБЛЕМА 3: Отсутствуют спрайты ресурсов + нет системы логов

### Симптом
Ресурсы (руда, деревья, растения) не видны на карте. Логи показывают заспавненные, но визуально — ничего.

### Корневая причина — АНАЛИЗ КОДА

#### ResourceSpawner.cs: данные vs визуал
1. `TestLocationGameController.SetupResourceSpawner()` создаёт ResourceSpawner runtime
2. `SetDefaultResourceTypes()` заполняет 12 типов ресурсов
3. `SpawnAllResources()` спавнит объекты — логи: "113 ресурсов заспавнено"
4. Каждый ресурс получает: SpriteRenderer + CircleCollider2D + ResourcePickup

#### Потенциальные причины невидимости:
1. **AI-спрайт не загружается** — `LoadResourceSprite()` ищет в `Tiles/{spriteName}.png`. Если файл есть, но import settings неправильные → Sprite = null → fallback на программный кристалл
2. **PPU спрайта ресурса из Tiles/** — установлен TileSpriteGenerator'ом: obj_* → PPU=160. При 64×64 → 0.4 юнита. При spriteScale=1.0 → 0.4 юнита — **ОЧЕНЬ МАЛЕНЬКИЙ**
3. **Sorting layer/order** — sr.sortingOrder=5, но terrain tilemap=0, objects=1. Должно быть видно
4. **Unlit шейдер** — используется корректно (виден без Light2D)
5. **Позиция Z** — не установлена явно, может быть 0 (как у terrain) → перекрытие

#### Главная вероятная причина: РАЗМЕР
При PPU=160: 64×64 → 0.4 юнита. Tilemap cell = 2×2 юнита. Ресурс = 1/5 тайла. На карте 100×80 тайлов это микроскопическая точка. Камера orthographicSize=8 → видит ~16×12 юнит → 8×6 тайлов. Ресурс 0.4 юнита в этой области — **размером с 1 пиксель на экране**.

### Решение
1. **Увеличить размер ресурсов** — spriteScale=3.0 или PPU=32 (64/32=2.0 юнита × 1.0 = 2.0 юнита)
2. **Добавить систему логов** `[ResourceLogger]` для детальной диагностики:
   - Логирование загрузки спрайта (AI vs fallback, размер, PPU)
   - Логирование world size спрайта (sprite.bounds.size)
   - Логирование позиции и scale ресурса
   - Логирование видимости (renderer.enabled, renderer.isVisible)
3. **Sprite-Scale = 2.0** — чтобы ресурсы были заметны (0.8 юнита при PPU=160)
