## 1. Проблема: Спрайт персонажа

### Причина
PNG-файлы спрайтов игрока не имели .meta файлов. При первом импорте Unity создаёт .meta
с `textureType=Default` (а не Sprite). `PlayerVisual.LoadPlayerSprite()` вызывал
`EnsurePlayerSpritePPU()` только если `LoadAssetAtPath<Sprite>()` уже вернул не-null —
но при Default-типе спрайт равен null → `EnsurePlayerSpritePPU` никогда не вызывался →
спрайт навсегда оставался не-спрайтом.

### Исправление (FIX SPRITE-01)
Теперь `EnsurePlayerSpritePPU()` вызывается **до** `LoadAssetAtPath<Sprite>()`, 
а не после. Также добавлена проверка `File.Exists()` чтобы не пытаться исправить
настройки для несуществующих файлов.

### Как проверить
1. Открыть проект в Unity
2. Найти `Assets/Sprites/Characters/Player/player_variant1_cultivator.png` в Project окне
3. Кликнуть → Inspector должен показать Texture Type = **Sprite (2D and UI)**
4. Если тип = Default → Play mode → `EnsurePlayerSpritePPU` исправит автоматически
5. После Play → спрайт должен отображаться в Scene View

## 2. Инструмент: EquipmentDebugPanel

### Файл
`Assets/Scripts/Debug/EquipmentDebugPanel.cs`

### Как добавить на Player
- **Автоматически**: Phase06 Player создаст компонент при повторной сборке сцены
- **Вручную**: Выбрать Player в Hierarchy → Add Component → `EquipmentDebugPanel`

### Управление
| Клавиша | Действие |
|---------|----------|
| **F2** | Переключить видимость отладочной панели |

### Функции панели

#### Секция «ГЕНЕРАТОР ЭКИПИРОВКИ»
- Поле «Уровень» (1-10) — влияет на itemLevel, materialTier, grade
- Поле «Кол-во» (1-50) — количество предметов для массовой генерации
- **Оружие x1** — сгенерировать 1 случайное оружие
- **Броня x1** — сгенерировать 1 случайную броню
- **Случайный x1** — 50/50 оружие или броня
- **Оружие/Броня/Смешанный xN** — массовая генерация

#### Секция «ИНВЕНТАРЬ»
- Отображает: кол-во предметов, вес/объём (с прогресс-барами)
- Предупреждения: ПЕРЕГРУЗ / ПЕРЕПОЛНЕНИЕ
- Список всех предметов с [Grade] T[Tier] [Slot] [Урон/Защ]
- Кнопка «X» — удалить предмет
- «Очистить инвентарь» — удалить всё
- «Сортировать» — сортировка по категории → имени → редкости

#### Секция «ЭКИПИРОВКА (КУКЛА)»
- 7 видимых слотов: Head, Torso, Belt, Legs, Feet, WeaponMain, WeaponOff
- Для каждого: название, грейд, тир, урон/защита, прочность
- Кнопка «Снять» — снимает в инвентарь
- Суммарные статы: Урон, Защита, Снижение, Уклонение

#### Секция «СТАТИСТИКА ГЕНЕРАТОРОВ»
- Seed, статус инициализации
- Размер кэшей: NPC, Техники, Экипировка
- «Сбросить кэш» — очистить все кэши
- «Новый сид» — реинициализировать с новым seed

## 3. Пошаговая инструкция тестирования

### Тест 1: Спрайт персонажа
```
1. Play mode
2. В Scene View найти Player
3. Проверить: виден спрайт культиватора (не белый квадрат, не пустота)
4. Console: "[PlayerVisual] AI-спрайт загружен: ...PPU=64..."
5. Если виден программный гуманоид (белая фигура) → FIX SPRITE-01 не сработал
```

### Тест 2: Генерация одного предмета
```
1. Play mode
2. Нажать F2 → отладочная панель
3. Установить Уровень = 1
4. Нажать «Оружие x1»
5. Проверить: в секции «ИНВЕНТАРЬ» появился 1 предмет
6. Проверить: предмет имеет [Common] T1, есть Урон
7. Нажать «Броня x1»
8. Проверить: 2 предмета, броня имеет Защиту
```

### Тест 3: Массовая генерация
```
1. Установить Уровень = 3, Кол-во = 10
2. Нажать «Смешанный x10»
3. Проверить: ~10 предметов в инвентаре
4. Проверить: есть и оружие и броня (перемешаны)
5. Проверить: грейды [Common]/[Refined] (уровень 3)
6. Проверить: T1-T3 материалы
```

### Тест 4: Перегруз инвентаря
```
1. Установить Кол-во = 50
2. Нажать «Смешанный x50» несколько раз
3. Дождаться предупреждения ПЕРЕГРУЗ или ПЕРЕПОЛНЕНИЕ
4. Проверить: не все предметы добавились (корректно)
5. В Console нет ошибок NullReferenceException
```

### Тест 5: Кукла экипировки
```
1. Сгенерировать оружие и броню (Тест 2-3)
2. Нажать I → открыть инвентарь
3. Перетащить оружие на WeaponMain
4. Перетащить броню на Torso/Head/Legs/Feet
5. Закрыть инвентарь (ESC)
6. F2 → секция «ЭКИПИРОВКА» → проверить слоты заполнены
7. Нажать «Снять» рядом с оружием
8. Проверить: оружие вернулось в инвентарь
```

### Тест 6: Двуручное оружие
```
1. Сгенерировать оружие пока не появится:
   - Greatsword, Spear, Bow, Staff или Crossbow
2. Экипировать на WeaponMain
3. Проверить: WeaponOff заблокирован (зачёркнут/серый)
4. Снять двуручное → WeaponOff разблокирован
```

### Тест 7: Генератор — детерминированность
```
1. F2 → «Новый сид» → записать seed
2. «Оружие x3» → записать названия
3. «Сбросить кэш»
4. Установить тот же seed вручную (через код)
5. «Оружие x3» → названия должны совпасть
```

### Тест 8: Иконки и цвета
```
1. Сгенерировать предметы разных грейдов (Уровень 5+)
2. Открыть инвентарь (I)
3. Проверить: иконки имеют цветные фоны по грейду
   - [Damaged] = коричневый фон
   - [Common] = серый фон
   - [Refined] = зелёный фон
   - [Perfect] = синий фон
4. Проверить: рамки по Rarity (серый/зелёный/синий/фиолетовый/золотой)
5. Проверить: Tier-индикатор 4x4 в углу иконки
```

### Тест 9: Очистка и сортировка
```
1. Сгенерировать 15+ предметов
2. Нажать «Сортировать» → предметы упорядочены
3. Нажать «Очистить инвентарь» → 0 предметов
4. Проверить: вес = 0, объём = 0
```

## 5. Известные ограничения

1. **equippedSprite = null** — все предметы генерируются без equippedSprite (нет оверлея на персонаже)
2. **Нет Resources/ папки** — runtime-спрайт игрока не загрузится в Build (только Editor)
3. **ConsumableSOFactory не реализован** — расходники не генерируются
4. **Нет LootTable SO** — лут всегда случайный, без таблиц дропа
5. **Стартовый рюкзак не назначается** — нужно вручную или через Phase18

## 6. Pipeline: DTO → SO → Inventory → Equip (полная цепочка)

```
WeaponGenerationParams                    ArmorGenerationParams
         ↓                                          ↓
WeaponGenerator.Generate()              ArmorGenerator.Generate()
         ↓                                          ↓
GeneratedWeapon (DTO)                   GeneratedArmor (DTO)
         ↓                                          ↓
EquipmentSOFactory.CreateRuntimeFromWeapon()   CreateRuntimeFromArmor()
         ↓                                          ↓
EquipmentData (ScriptableObject, в памяти) ←─GradeColors + ProceduralIcon
         ↓
LootGenerator.GenerateLoot() / GeneratorRegistry.GenerateEquipmentLoot()
         ↓
InventoryController.AddItem(equipmentData, 1)
         ↓
InventorySlot (rowIndex, weight, volume)
         ↓  (drag-drop через InventoryScreen)
InventoryController.EquipFromInventory(slotId)
         ↓
EquipmentController.Equip(equipmentData, grade, durability)
         ↓
EquipmentInstance (slot → equipmentData, grade, durability)
         ↓
EquipmentStats (totalDamage, totalDefense, bonuses)
```
