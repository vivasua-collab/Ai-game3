**Дата:** 2026-04-27

## 🎯 Цель

Перевести систему инвентаря с сеточной (7×7) на строчную модель, где ограничивающими факторами являются **масса** и **объём**, а не фиксированная сетка слотов.

## 📋 Принятые решения

1. **Отказ от сетки** — вместо координатной сетки (posX/posY 0-6) использовать строчный список предметов
2. **Масштабируемый инвентарь** — ёмкость зависит от сумки/рюкзака/контейнера
3. **Два ограничителя:** масса (weight) и объём (volume)
4. **Нет необходимости рисовать отдельную сетку** — строчный интерфейс проще

## 📐 Предварительная модель данных

```
BackpackData:
  id: string
  name: string
  maxWeight: float        // Максимальная масса (кг)
  maxVolume: float        // Максимальный объём (литры)
  currentWeight: float    // Текущая масса
  currentVolume: float    // Текущий объём
  items: InventoryItem[]  // Список предметов (не сетка!)

InventoryItem:
  id: string
  itemId: string          // Ссылка на ItemData
  quantity: int
  weight: float           // Масса одной единицы
  volume: float           // Объём одной единицы
  
ItemData (дополнить):
  weight: float           // Базовая масса
  volume: float           // Базовый объём
  stackable: bool         // Можно ли складывать
  maxStack: int           // Максимум в стопке
```
